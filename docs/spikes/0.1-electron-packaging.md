# Spike 0.1 — Electron shell / packaging (status: GO)

> OQ-001 · Spec anchors: `ARCHITECTURE.md` §13 (Deployment/Install), §14 (ADR-001 Electron shell), §16 (worker supervision & lifecycle). Plan task 0.1 + downstream 11.6.
> Date run: 2026-06-29 · Host: macOS 26.4.1 (build 25E253), arm64 (Apple Silicon) · Node v22.13.0 · pnpm 11.5.2.

## Question / unknown

Can we ship V1 as an **unsigned build-from-source** Electron app (owner decision, §13 — matches the open-source install-from-GitHub goal; signed+notarized DMG is explicitly V1.1, §15) that **spawns + supervises** the three long-running control-plane subprocesses —

1. the Node control-plane **worker**,
2. the local **Temporal dev-server** (`temporal server start-dev --db-filename …`), and
3. one **GBrain sidecar per workspace** (single-owner per PGLite file) —

with the §16 supervision contract (**restart-on-crash + bounded backoff + crash-loop threshold → "worker down" state instead of looping**), and what **hardened-runtime entitlements** does such a build need?

**NO-GO ⇒ reopen the Electron shell ADR** (§14 / DECISIONS ADR-001 → Tauri).

## What I did

Throwaway spike in `…/scratchpad/phase0/0.1/` (NOT in the repo). Real toolchain, real artifacts:

1. **Authored an extractable supervisor** (`src/supervisor.js`, pure `node:child_process` + `node:events`) implementing the §16 contract: restart-on-crash, bounded exponential backoff (min/max/factor), a crash-loop guard (N crashes within a rolling window → terminal `down` state, **stops respawning**), and a stable-uptime backoff reset. The **same module** runs inside Electron main and in a headless harness — that is how the load-bearing behaviour is validated without a display.
2. **Electron main** (`src/main.js`) that, on `app.whenReady()`, boots one `Supervisor` for the worker, one for the Temporal dev-server, and **one per workspace** (`employer`, `personal`, `side-projects`) for GBrain — wiring each supervisor's state events into a `health` map (the System-Health/OBS projection source). Children are resolved from `process.resourcesPath/children` when packaged, `__dirname/children` in dev.
3. **Three child stubs** (`worker.js`, `temporal-stub.js`, `gbrain-stub.js`) — env-driven so the harness can script healthy / flap / crash-loop. (The real Temporal binary + real GBrain are the live-validation gate below.)
4. **Headless supervision harness** (`harness/run-supervisor-standalone.js`) — spawns **real** node child processes and asserts three scenarios.
5. **electron-builder config** (in `package.json`) → **unsigned** mac `dir` target, `identity: null` + `CSC_IDENTITY_AUTO_DISCOVERY=false`, `asar:true`, child scripts via `extraResources`.
6. **Hardened-runtime entitlements** (`build/entitlements.mac.plist`).
7. **Ran** the harness (plain node), **built** the unsigned `.app`, **inspected** it (Info.plist, `codesign`, `spctl`), **ad-hoc re-signed** it with hardened runtime + entitlements to prove the plist is valid, and **booted the real Electron main headlessly** — both dev source and the packaged hardened-runtime `.app`.

Commands (all hard-timeout-wrapped): `pnpm install` (electron 42.5.1 + electron-builder 26.15.3) · `electron-builder --mac dir --arm64 --config.mac.identity=null` · `node harness/run-supervisor-standalone.js` · `codesign -dv` / `codesign --force --deep -s - --options runtime --entitlements …` / `codesign -d --entitlements` · `spctl --assess` · headless `electron .` and `SoWSpike.app/Contents/MacOS/SoWSpike` under `SPIKE_HEADLESS_SELFTEST=1`.

**(4) arm64 signature reality + entitlements validity.**
```
unsigned build:           CodeDirectory flags=0x20002(adhoc,linker-signed)  Signature=adhoc  TeamIdentifier=not set
                          → Apple Silicon REQUIRES at least this ad-hoc/linker signature to execute; hardened runtime OFF.
after `codesign --options runtime --entitlements build/entitlements.mac.plist`:
                          CodeDirectory flags=0x10002(adhoc,runtime)         ← hardened runtime ON

                          · allow-dyld-environment-variables · files.user-selected.read-write   (all 5 present)

```

**(5) Gatekeeper / first-launch.** `spctl --assess --type execute` → **rejected** (rc=3) — expected for unsigned/not-notarized. **Key nuance:** a *build-from-source* install (git clone + local `pnpm build`) never receives the `com.apple.quarantine` xattr (that flag is applied by the **downloading** app — browser/unzip — not by the compiler), so the locally built `.app` launches **without** the Gatekeeper prompt. Documented fallback for any quarantined copy: `xattr -dr com.apple.quarantine <App>.app` (or right-click → Open).

## Decision: GO

Unsigned build-from-source packaging on Electron is **proven end-to-end on this stack**: electron-builder produced a runnable `.app` with `identity:null` (no signing-identity dependency, §13/§15), and the **§16 supervision contract** — restart-on-crash, bounded exponential backoff, and a crash-loop threshold that surfaces a terminal "worker down" state instead of looping — is validated both as isolated logic and **inside the real (packaged, hardened-runtime) Electron main**, spawning + supervising the worker, the Temporal dev-server, and **one GBrain sidecar per workspace**. The hardened-runtime entitlement set is authored, valid, and confirmed to permit child-process spawning. ADR-001 (Electron) stands; **no reopen**.

Rationale: every load-bearing claim of OQ-001 was exercised with real artifacts; the only unproven legs are the *real* Temporal/GBrain binaries (substituted by faithful long-running stubs) and one visible-GUI foreground launch — both low-risk and captured as live-validation gates, neither touching the packaging/supervision mechanism itself.

## No-go branch (made concrete — NOT triggered)

Had packaging or supervision failed, §14/ADR-001 says **reopen the Electron shell ADR → Tauri**. Concretely: move the shell to **Tauri v2** (Rust core), bundle the worker/Temporal/GBrain as **sidecars** declared in `tauri.conf.json > bundle.externalBin` and spawned/supervised via `tauri-plugin-shell` (`Command::sidecar(...).spawn()`), porting `supervisor.js`'s restart/backoff/crash-loop logic into the Rust core. Cost: a Rust↔Node seam in the control plane (the exact thing ADR-001 chose Electron to avoid). Not needed.

4. **V1.1 signed + notarized DMG** with a real Developer ID + `notarytool` — deferred to V1.1 by §15; entitlements plist already proven valid against `--options runtime`.
