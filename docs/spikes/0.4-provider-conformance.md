## Decision: GO-WITH-CONDITIONS

The deliverable for OQ-003 is a **recorded, committed baseline**: providers enumerated, default model per capability chosen, cost-estimation method + price sources recorded, default cap set, and `config/providers.defaults.json` written as real usable JSON shaped to the Appendix-A contracts. That is done and committed.

It is **GO-WITH-CONDITIONS**, not unconditional GO, because conformance *is the contract* (§7) and **no pair was live-verified** — no key in Keychain and neither local endpoint was up. Every `pinnedModels[*].conformanceStatus` is honestly `unknown`; the broker must treat `unknown` as **ineligible** until the §12 conformance harness (Phase-5 task 5.10) flips a pair to `passing`. The structured-output/tool-use *capability* claims are research-pinned from current docs and safe to build the matrix shape against; the per-model *fidelity* numbers are the part that needs a live run.

This satisfies the Phase-0 acceptance bullet ("provider/model pairs + default caps recorded") without overclaiming verification.

## No-go branch

Made concrete from §7 + the safety rules:
- **A cloud pair fails conformance** → that `provider × capability × model` pair is disabled (`conformanceStatus: failing`); the matrix falls back to another conformant pair for the capability. The config already carries alternates per capability.
- **No cloud pair is conformant for `meeting.close`** → DoD cannot be certified for that workspace (§7 requires ≥1 conformant provider/runtime for `meeting.close`); the build blocks until one passes. Claude Agent SDK is the primary; Hermes is the secondary (Spike 0.3).
- **Employer-Work raw content + egress ack OFF and no conformant local provider** → the job **fails closed** (§5 veto), never a cloud fallback. The `localZeroEgress` route per capability exists precisely so a pulled local model can satisfy this.
- **Local-only is unviable** (no Ollama/LM Studio reachable, as at spike time) → cloud pairs carry every non-Employer-raw capability; local is an optional zero-egress path, not a release gate (§7).
