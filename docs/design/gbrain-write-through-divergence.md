# GBrain Write-Through & Divergence/Parity Layer — V1 Design Spec

- **Status:** APPROVED (owner sign-off 2026-06-29) — basis for the `ARCHITECTURE.md` + `IMPLEMENTATION_PLAN.md` amendment.
- **Provenance:** 8-agent design workflow (ground against real gbrain 0.35.1.0 source → 3 independent designs → 3 adversarial safety critics → synthesis). Grounding inputs preserved in the session task output.
- **Decision context:** Owner chose to ship GBrain **write-through in V1** (reversing the 0.2 read-only/index-only deferral), made safe by a mandatory divergence/parity layer, and to use gbrain's generative features as a **gated proposal source** (DB-canonical reversal considered + rejected — Markdown stays canonical). See memory `system-of-work-prd`.

---

## 1. Decision & the load-bearing insight

GBrain is **DB-first natively** (`gbrain put`/MCP `put_page` write the DB, never canonical Markdown; `dream`/`synthesize`/`patterns`/Minions write new DB pages with no Markdown backing). SoW **inverts this by policy**, and the inversion is enforced by one decisive design move:

> **Serve answer bytes *from committed Markdown*, never from the DB row.** The GBrain DB is a **pointer/ranking index only**. And **derive the canonical "what should exist" set with a SoW-owned, gbrain-INDEPENDENT Markdown parser** — gbrain is never asked what the Markdown contains, so it sits *outside its own checker's trust base*.

This collapses every adversarial attack the critics found (borrowed stamp, tampered DB row, stale allow-set, lost-update window, classification-downgrade, common-mode-malicious gbrain) from "serves a false answer" into "withholds + raises a HealthItem."

**Basis:** reconciler-first (trust spine: continuous bidirectional parity + default-deny ServingGuard) + rebuild-equivalence (gbrain rebuild oracle, demoted to a *corroborating cross-check* behind the SoW parser) + provenance-first (the stamp, upgraded from a forgeable checksum to an **HMAC signature** with serve-time content rebinding).

## 2. Three independent safety legs

Safety rule 1 (Markdown is the only canonical truth; KnowledgeWriter its only autonomous writer) holds via three independently-sufficient structural legs:

1. **Bytes-from-Markdown serving.** The `MarkdownRehydrationServingGate` never returns DB-row bytes. The DB is a pointer/ranking index; every served fact's content is re-hydrated from committed Markdown at serve time and hash-matched to the SoW deriver's `mdContentSha` at the current revision. A DB-only fact has no bytes to serve → structurally un-serveable.
2. **gbrain-independent allow-set + unforgeable provenance.** The allow-set is the `CanonicalFactSet` derived by SoW's own Markdown parser (the gbrain rebuild oracle is only a corroborating cross-check; its disagreement is a *defect*, never a calibration target). Admission additionally requires a valid **HMAC `SignedProvenanceStamp`** minted with a SecretsPort key the generative / DB-write / runtime paths never hold.
3. **OS-level one-writer.** KnowledgeWriter is the sole OS principal that can write canonical `.md` (filesystem ACL + read-only mounts / immutable snapshots for *every* gbrain process + sole PGLite advisory lock + positive fs-watch attribution). gbrain physically cannot write the vault; an fs-write during an index job is a hard alarm by construction.

**Fails closed (default-deny):** a fact is served only if `Markdown-rehydrated AND signature-valid AND in current-revision allow-set AND not quarantined`. On *any* divergence/uncertainty (dirty/incomplete `ParityReport`, oracle-build failure, pin mismatch / `PENDING` sentinel, missing embedding key, crashed reconciler, out-of-order apply, signature failure) → the workspace degrades to Markdown-provenanced-only (deepest degrade: direct-committed-Markdown) serving. Worst observable outcome = a withheld fact + a pending-review HealthItem. If containment can't be proven for a workspace, `writeThroughEnabled` stays **OFF** → GBrain runs read-only/index-only (the §6 named fallback; still DoD-satisfying, REQ-D-001).

## 3. Components

| Component | Purpose | Mechanism (grounded in real gbrain 0.35.1.0) |
|---|---|---|
| **CanonicalFactDeriver** (`packages/knowledge/src/gbrain/derive/`) | The single TRUSTED "what should be in the brain" set; takes gbrain out of its own checker | SoW-owned, gbrain-INDEPENDENT parser: vault Markdown @ pinned revision → normalized `SemanticFact[]` (pages, links/edges, timeline, tags, frontmatter values), each with content-independent `factIdentity` + a SoW-computed `mdContentSha`. `gbrain extract --source fs --dry-run --json` is used ONLY as a divergence cross-check oracle, never as the canonical source. |
| **SignedProvenanceStamper** (inside KnowledgeWriter) | Make "Markdown-provenanced" an unforgeable property, not a copyable label | At KW atomic commit writes a `SignedProvenanceStamp` into frontmatter: `{kwRevision, originPath, mdContentSha, writerActor:'KnowledgeWriter', sourceEventRef, committedAt, sig}` where `sig = HMAC(workspaceId, factIdentity, originPath, mdContentSha, kwRevision)` via a SecretsPort/Keychain key the generative/DB-write/runtime never hold. Survives `gbrain import` into `pages.frontmatter` JSONB (gbrain strips only `slug:`). |
| **MarkdownRehydrationServingGate** | Guarantee no DB-only/tampered byte is ever an answer (THE load-bearing fix) | Wraps the read surface. DB used ONLY for retrieval/ranking/pointers (slug + span + score), never as a byte source. Re-hydrates every candidate's bytes from committed Markdown at serve time; admits only if rehydrated-hash == `CanonicalFactDeriver.mdContentSha` @ current revision AND `sig` verifies AND `factIdentity ∈` current-revision allow-set AND not quarantined. DEFAULT-DENY; degraded coverage → direct-committed-Markdown retrieval only. |
| **ContainedSynthesisGate** | A free-text generative read (`think`/Copilot synthesis) can't be policed by a factIdentity filter | Serve-time synthesis runs ONLY over the ServingGate-filtered, Markdown-rehydrated, signature-verified context passed in as input — never the raw PGLite store/embeddings. `GbrainReadGrant.allowedOps` excludes store-wide `think`; Copilot Q&A (§9 flow 13) consumes only the contained context. |
| **ParityReconciler + DivergenceClassifier** | Continuously re-prove every DB fact is derivable from Markdown; classify into a closed action lattice (GO #3) | SoW-owned Temporal-scheduled pass (NOT gbrain cron) diffs `CanonicalFactSet` (SoW deriver) vs `DbProjection` (read-only HTTP token) keyed by `factIdentity + mdContentSha`, with the `gbrain import`-into-scratch RebuildOracle as a SECOND corroborating check. Fires on post-commit index / fs-watch / schedule (LIFE-2 collapse=MAX revision) / on-demand. Closed enum: `db_only | unstamped` (HARD floor — never auto-downgraded/backfilled on a gbrain-supplied hash) `| content_mismatch` (Markdown-wins resync ONLY) `| md_only` (benign re-index) `| edge_db_only | edge_md_only | stale_revision`. Emits a revision-scoped `ParityReport` with `cleanForServing` + `coverageComplete`. |
| **QuarantineLedger + revision-scoped allow-set** | The do-not-serve enforcement gbrain has no concept of; modeled so a quarantine can't regress or resurrect | Allow-set IS `CanonicalFactSet @ current revision` — a SET, not an accumulating union; applied by MONOTONIC compare-and-set on `revisionId` (pointer never regresses). Quarantine = ABSENCE-from-allow-set; `QuarantineRecord` keyed on stable content-independent `factIdentity` (one-byte change → same identity, so purge can't be evaded by re-introduction). On a KW delete/edit commit, old `factIdentity` admissibility is atomically retracted at the commit (durability point). |
| **RemediationRouter** | Drive every defect to a terminal state without laundering DB content into Markdown | `db_only` with no Markdown counterpart → materialize-via-plan (re-validated through the full pipeline) OR purge via a **DELETE/PURGE-ONLY token** (never a general `put_page`/`add_link` capability, even local). `content_mismatch` → resync-FROM-Markdown ONLY (Markdown wins; DB body never proposed as canonical). Ambiguous → owner review; auto-purge requires positive proof of non-derivability, not merely an absent stamp. |
| **GenerativeProposalIntake** | The ONLY path a generative output reaches canonical state — propose-only, non-circular | Generation = a SoW worker ModelProviderPort/AgentJob call over read-only, ServingGate-filtered context — NEVER gbrain in-engine `synthesize`/`dream`/`autopilot`. Output → `GBrainProposedFact` candidate → JSON-Schema gate + no-inference validator (REQ-F-017) → `KnowledgeMutationPlan` (`provenanceOrigin='gbrain_proposal'`, `requiresApproval` default-true) → KnowledgeWriter → Markdown → commit → import/sync. Evidence MUST cite already-canonical Markdown / a genuinely-ingested SourceEnvelope span; the proposal's own scratch origin is INADMISSIBLE as support. Scratch brain = physically separate PGLite file + vault dir. |
| **GbrainWriteFence + OS one-writer lockdown** | Make GO #1 PREVENTIVE not detective; the write-trust point gbrain lacks natively | SoW worker is sole issuer of GBrain credentials: a read-ONLY HTTP OAuth token (`GbrainReadGrant`) to the runtime, never stdio `gbrain serve` (no scope gate), never a write/admin token. Worker holds the sole PGLite lock AND a filesystem ACL makes it the only OS principal with write access to the vault directory. EVERY gbrain process (import/sync/extract/oracle/doctor/lint) runs against a READ-ONLY mount / immutable revision snapshot. CONTINUOUS scan rejects any stray `stdio-serve`/`sync --install-cron`/`autopilot`/`jobs-work` bound to the canonical brain; the generative DB-writing cycle is hard-disabled in V1. |
| **FsWatchPositiveAttribution + ConcurrencySerializer** | Close the §6 out-of-band-writer hole (GO #1 read side) + no lost update (GO #2) | fs-watch uses POSITIVE KW attribution (`kw_writer_sig` + write-journal). Any working-tree mutation that is neither a verified KW write nor a human-REGION edit — including a NEW assistant-domain file — becomes a conflict-review HealthItem, NEVER an auto base-revision advance. Pipeline: KW-validate→write→commit→snapshot→sync; index keyed by `(workspaceId, revisionId)`; sync reads an IMMUTABLE snapshot (no torn reads); monotonic compare-and-set apply; trigger collapse = MAX(revision); single-owner PGLite lock + content_hash idempotency. |
| **GbrainPinGate + WriteThroughEnableFlag + CrashRecoveryReconciler** | Block write-through until validated vs installed 0.35.1.0; keep read-only as default/kill-switch; recover safely | Re-captures `config/gbrain.pin` vs installed 0.35.1.0 (`gbrain --version` + `doctor --json` schema_version; retires v0.18.2/PENDING sentinel). `writeThroughEnabled` (per-workspace, default FALSE) flips ON only when all 4 GO conditions are green LIVE + read-token-rejects-write conformance passes against the ACTUAL pinned SHA + embedding key GREEN + no cron/autopilot installed. A dirty/failed `ParityReport` auto-reverts to Markdown-provenanced-only. On restart, the allow-set is rebuilt from `CanonicalFactDeriver(current Markdown)` — gbrain-independent, so a crash never strands a true fact un-served nor resurrects a quarantined one. |

## 5. The 4 GO conditions as V1 acceptance tests

All run against **real gbrain 0.35.1.0 + a real embedding key** (never a `noEmbed`-degraded index), no mocks.

- **GO #1 (one-writer / no hidden brain)** — full generative-cycle + index-sync + oracle-rebuild against a vault mounted READ-ONLY for every gbrain process. Assert: zero non-KW canonical-`.md` mutation events; a stray `frontmatter --fix`/`writeBrainPage` at the vault FAILS + raises conflict-review; a NEW assistant-domain out-of-band `.md` → conflict-review (not auto-advance); a human-REGION edit still clean-advances.
- **GO #2 (no lost update)** — `gbrain sync` index job vs an immutable snapshot of revision N while KW commits N+1, triggers delivered OUT OF ORDER. Assert: final allow-set == `CanonicalFactDeriver(N+1)`; the N apply is a no-op (no regression); collapse=MAX; ServingGate withheld every not-yet-proven fact during the window; single-owner lock + content_hash idempotency held.
- **GO #3 (parity catches DB-only facts)** — inject 4 DB-only facts: (a) manual `gbrain put`; (b) synthesize/dream-style page no Markdown backing; (c) **borrowed-stamp** page (copies a real signed stamp onto fabricated bytes); (d) **forged-content_hash** collision. Assert each → `db_only`/`unstamped` (HARD floor), absent from allow-set, quarantined, `parity_defect` HealthItem, served by NEITHER the keyed gate NOR contained-synthesis. (c) rejected by signature + serve-time rehydration-hash mismatch; (d) rejected because the gbrain-independent deriver doesn't derive it from Markdown.
- **GO #4 (round-trip lossless)** — KW-write → commit → `import/sync` → DB → rebuild oracle → compare. Assert SEMANTIC-field equality (title/body/frontmatter-values/region-markers/links/timeline) using the SoW parser as the REFERENCE side (not gbrain on both sides; slug-strip + frontmatter reformat tolerated, byte-level not required); `gbrain lint` clean; doctor `frontmatter_integrity` pass; `markdown_body_completeness ≥` threshold; `embeddings`/`embedding_provider` GREEN.
- **ENABLEMENT GATE** — `writeThroughEnabled` stays FALSE (read-only/index-only fallback) until GO #1–#4 green LIVE vs 0.35.1.0 AND `config/gbrain.pin` promoted out of `PENDING_LIVE_VALIDATION` AND read-token-rejects-`put_page`/`add_link` conformance passes against the actual pinned SHA AND no cron/autopilot installed.

## 7. Residual risks (named)

1. **Performance** — serve-time Markdown rehydration + gbrain-independent re-derivation + the full rebuild-oracle cadence add serving/reconcile latency beyond the index leg. KW→GBrain ≤60s / KW→read-model ≤10s cover the index leg only; rehydration + oracle cadence need their own measured SLO rows. A too-rare oracle can still surface a STALE POINTER (mis-ranked but Markdown-true), though tampered bytes stay un-serveable.
2. **Parser maintenance** — the SoW `CanonicalFactDeriver` must track gbrain's link/timeline/edge derivation closely enough to avoid false-positive divergence floods while staying independent; re-validate at every pin bump.
3. **HMAC key management** — the stamp-signing key (Keychain/SecretsPort) must never be reachable by generative/DB-write/runtime paths; rotation invalidates historical stamps → re-stamp-on-rotation migration or multi-key verify window.
4. **ContainedSynthesisGate** — depends on the runtime always routing synthesis through the filtered-context path; enforce structurally (`GbrainReadGrant.allowedOps` excludes store-wide `think`) + a conformance test, not by convention.
5. **Version skew to 0.35.1.0** — `content_hash` semantics, fs-vs-db extract edge-resolution, frontmatter-VALUE survival through `import` (a JSONB coercion would break sig/hash compare), OAuth scope-gate behavior must all be re-validated against the ACTUAL pinned binary before enablement.
6. **Quarantine-loop / re-synthesis resurrection** — keyed on claim/node identity, itself a judgment surface; a rephrased re-synthesis could evade dedup. Owner-review-default for ambiguous remediation bounds, doesn't eliminate.
7. **OS-level vault ACL + read-only mounts are macOS-specific** operational invariants the install doctor must verify; a misconfigured mount (gbrain pointed at the live vault with `--fix`) re-opens GO #1. fs-watch positive-attribution is the detective backstop when the ACL/mount is absent. A monitoring rate-alarm on `coverageComplete=false`/oracle-build failures (treat as possible attack/DoS) is needed.
