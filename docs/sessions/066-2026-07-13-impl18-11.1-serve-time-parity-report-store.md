## Step-8 mandatory dual review + findings addressed

Both reviewers ran on the slice diff (safety-relevant). **security-reviewer: CLEAR to ship dormant** (verified rule-2 candidate re-gate on read-back, rule-7 redaction, §16 fail-closed both directions/both dialects, first-write-wins, WS-8/injection, dual-dialect parity). **code-quality-reviewer: SHIP.**

Findings fixed in the same review pass (all re-verified green):
1. **[security MEDIUM + cq LOW — CONVERGENT] Non-deterministic "latest" tiebreak.** `getLatestForRevision` ordered `desc(recordedAt)` with no secondary key → two distinct reports at the same `recordedAt` for one `(ws, rev)` resolved to an arbitrary physical row (SQLite vs Postgres could disagree — forbidden-pattern #2; a clean report could shadow a dirty one → trust-gate defeat). **Fix:** added `desc(reportId)` secondary sort to both adapters (mirrors the in-file `healthItems` precedent) + a comment that the B3 write-path caller must supply canonical ISO-8601. New contract test pins a stable, both-dialect-equal tie winner.
2. **[cq MEDIUM] Newest-wins test non-discriminating.** The test's insertion order coincided with `recordedAt` order, so a `desc(rowid)` impl would also pass. **Fix:** inverted the insertion order so only true `desc(recordedAt)` passes.
3. **[cq LOW] db test filename.** **Fix:** renamed `parity-report-store.test.ts` → `parity-report-durability.test.ts` (matches the `knowledge-revision-durability` precedent; disambiguates from the worker-layer `parityReportStore.test.ts`).
4. **[security LOW → orch ruled ADD NOW] Read-back identity gate (WS-8, safety rule 4).** The re-gate validated payload SHAPE but not IDENTITY vs the query key — a directly-tampered row (columns ≡ query-key, payload claims a different ws/rev) would surface cross-workspace. Orch ruled this belongs closed in the store slice, not a later wiring slice. **Fix:** `toParityReport` now also asserts the parsed `workspaceId`/`reconciledAtRevision` equal the query args, else throws → typed err (fault, not absence). New durability test inserts a tampered row and pins the fail-closed err.

Finding accepted as-is (no change): **[cq LOW, informational]** the pg corrupt-payload/fault path is exercised on sqlite only — consistent with the `knowledge-revision-durability` precedent; the both-dialect contract suite covers the happy paths and the pg fault path is structurally identical (same `run`/`toDbError` wrapper).
