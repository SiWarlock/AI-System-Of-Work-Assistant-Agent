## What was built

### P2.3 — the real synthesis adapter (`47d57cb`)

**Files created:**
- `apps/worker/src/api/procedures/copilotClaudeSynthesis.ts` — `createClaudeCopilotSynthesis(client, opts?)` implements `CopilotSynthesisPort` over the `ClaudeSubscriptionCompletion` client (`@sow/providers`, from P2.2). Exports: the governed `COPILOT_SYSTEM_PROMPT` (grounded-only, cite-by-citationId, no-invention/REQ-F-017), `buildCopilotUserPrompt` (question + citationId-tagged passages, paired to blocks by index), `COPILOT_OUTPUT_SCHEMA` (`{answer,citations}` JSON Schema), `mapCompletionToCandidate` (the grounding reconciler), `foldCompletionError` (redaction-safe error fold), `DEFAULT_COPILOT_MAX_COST_USD`.
- `apps/worker/test/api/procedures/copilotClaudeSynthesis.test.ts` — the TDD suite (34 tests at P2.3; grew to 45 with P2.4).

**Load-bearing properties:**
- **Grounding enforced in CODE, not just the prompt.** `mapCompletionToCandidate` reconciles the model's returned citations against the RETRIEVED set: a citationId not retrieved is dropped (the model can't invent a source), the AUTHORITATIVE retrieved title is used (the model's echoed title is discarded before it can reach the UI), duplicates collapse, model order preserved. The downstream `toUiSafeCopilotAnswer` gate re-validates the whole shape — fail-closed at two layers.
- **Redaction-safe by construction.** `foldCompletionError` DROPS the SDK-origin `CompletionError.message` (which may carry prompt/content fragments) and carries only the enum `kind`, a stable UPPER_SNAKE cause code, and `retryable` — all survive §16 redaction; a free-form message would be collapsed to `REDACTED_RAW` anyway. Discharges the P2.2 §16 carry-forward.
- **Fail-closed route guard (rule-5 defense-in-depth).** The subscription client always egresses to Anthropic, so the adapter serves ONLY a Claude PROVIDER route (`"provider" in route && route.provider === "claude"`), else a typed err BEFORE any egress. This ties the adapter's true destination (Anthropic) to the route's declared processor, so a wiring slip can't silently egress AND the upstream notice can't name a processor the content never reached. Binds to the veto-cleared `route.model` (P2.1); never re-selects (discharges the P1.2b carry-forward).

**Load-bearing properties:**

The security review traced the full flag-ON employer-work path → `decideCopilotEgress` ALLOWs the cloud route WITH `egressProcessor: "claude"` (the notice), never silent.
- **The flag branch is unit-tested.** `buildCopilotDeps` is the single place the real-vs-interim decision lives; the real SDK client is constructed ONLY on the real path (a fake completion factory proves the OFF path never calls it). A flipped ternary can't ship silently.
- **OFF path byte-equivalent** to the prior interim (stub + local route + fail-closed `localWorkspacePosture`). Flag defaults to undefined/false.

## Decisions made

- **The abandoned `{runtime:"claude-agent-sdk"}` route (from the stale plan) is SUPERSEDED** by a Claude PROVIDER route — consistent with the P2.2 dedicated-completion-client pivot, and the P2.3 guard enforces it. If P2.4 had used a runtime route, the guard would have rejected it loudly (the right fail-closed behavior).
- **The `copilotRealModel` flag IS the interim consent gesture** — flipping it to true acks employer-work cloud egress (with the notice), implementing the owner's stated posture ("fine with Employer-Work going to a cloud model, I just want a notice"). Interim until the authoritative per-workspace `WorkspaceConfigRepository` posture lands (the ack becomes a real per-workspace setting then).
- **The model is config-threadable** (`copilotModel` → `createClaudeCloudRouteSelector`), fixing a doc that promised "overridable" without a mechanism.
- **Grounding reconciliation belongs in the adapter** (deterministic), not the prompt — the model can only cite what it was given, and titles come from trusted retrieval.

## How to use what was built

Set `copilotRealModel: true` (and optionally `copilotModel`) where `BootConfig` is assembled (the Electron main supervisor). An Employer-Work Copilot ask then egresses to Anthropic via the ambient local `claude` login, with the visible "Answered using claude — a cloud model — on Employer-Work content" banner. Leaving the flag off keeps the deterministic interim stub (nothing egresses). Retrieval is still the fixture stub until P3, so answers will honestly report finding nothing until real GBrain retrieval lands.
