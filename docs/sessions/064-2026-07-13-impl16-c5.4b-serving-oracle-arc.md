## Decisions made

- **Slice 1 / Q4 — first-resolved-fact (page) content, NOT concatenation** for multi-fact citations (matches the production page-only resolver; avoids incoherent/duplicated bytes). Orch-endorsed.
- **Slice 1 / block rebuild = POSITIONAL** (`sources.map(s => admitted.get(s.citationId)?.content ?? "")`), NOT compacted. This was a Step-7.5 mid-slice correction: the mandatory security review caught that a *compacted* rebuild breaks `buildCopilotUserPrompt`'s positional `blocks[i]↔sources[i]` pairing → misattributes a proven excerpt to the wrong (untrusted) citation on partial admission. Option B fixes it in-scope, drops unverified bytes, and keeps alignment. Orch approved reversing the Step-2.5 compaction call.
- **Slice 2 / Q1 — sync-or-async union** for `CommittedVaultReader` (not pure-async), so the real async reader fits without rippling to 6 existing sync test fakes.
- **Slice 2 / file-fault policy** — a listed-but-unreadable `.md` (read → undefined) is SKIPPED (readable subset kept); all-unreadable/throw → undefined. Fail-safe (an absent page can only WITHHOLD). Pinned by an orch-required RED test.
- **Slice 3 / Q1 — single-served-workspace vault mapping** — map ONLY `copilotGbrainWorkspaceId` → the one dev vault; unset → empty map → all degrade. A refinement on the brief's default (mapping multiple served workspaces to the shared dev vault would be a cross-workspace read). Orch-endorsed.
- **Slice 3 / pin-probe DEVIATION** — the brief said "reuse the gbrain-pin probe (boot.ts:~1155)", but that probe is async fire-and-forget with no value at the selector call site (:869). Injected `pin`/`resolveRunning` via the go-live bundle instead (honest: coverage degrades regardless of `pinValid`). Orch approved (do NOT hoist an async pin-load — zero dormant benefit).
- **Slice 3 / helper extraction** — pure `servingOracleAssembly.ts` so the go-live logic is unit-testable without the async `bootWorker`.
