# /tdd brief — inbox_reroute_action (15.8 desktop leg)

## Feature
Add a **reroute / assign-project action** to the Ingestion Inbox so a low-confidence parked item can be **human-resolved to an explicit workspace/project**, not merely accept/reject — the UI half of REQ-F-017's "route to clarification." Today the Inbox offers only accept/reject (`apps/desktop/renderer/surfaces/ingestion-inbox/index.tsx:42-46`). This adds a reroute control with a **workspace/project picker sourced from the 14.6 typed-Project registry (read path)** and dispatches the **extended triage command** (the shared contract pinned in brief 105). Desktop-track (the **desktop leg** of 15.8). **Pure-build, NO hard line** — the renderer only builds a command payload + calls the existing triage IPC/tRPC; the worker leg validates + drives. Closes the desktop half of **G60**.
