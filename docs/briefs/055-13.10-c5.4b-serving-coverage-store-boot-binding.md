# /tdd brief ‚Äî serving_coverage_store_boot_binding

## Feature
The composition-root **CLOSING slice** for the serve-time parity read chain (the waiver-holder pattern, worker Lesson 20): boot binds the durable `ParityReportStore` read-adapter into `createServingCoverageReader`, **closing the B2 reachability waiver** so the reader‚Üístore‚Üírepo chain is reachable (behind the existing triple-lock), plus a **write‚Üíread round-trip e2e** proving a clean revision-matched `ParityReport` flows through the bound store to green the two **PARITY legs** (`cleanForServing` + `coverageComplete`).

**HONEST SCOPE ‚Äî this is NOT yet a full GREEN admission.** Serving coverage is AND-composed of 4 legs (`deriveServingCoverage`: `cleanForServing`, `coverageComplete`, `pinValid`, `oracleBuildOk`). B2 left **`oracleBuildOk` (the rebuild-oracle build-status leg) hardwired `false`** (deferred), so the overall verdict still **degrades** even with a clean report ‚Äî the e2e asserts this HONEST no-false-green. A full green admission awaits the **rebuild-oracle leg** (a named follow-on) + the arming. This slice greens ONLY the legs it wires (worker Lesson 13, "ship one leg at a time"). **Propose stays structurally OFF**; nothing is armed.
