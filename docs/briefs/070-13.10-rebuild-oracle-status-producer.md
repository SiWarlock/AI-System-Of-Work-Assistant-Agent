## Feature
Build the **pure worker-side rebuild-oracle STATUS producer** — the "build-first" deliverable the runbook's Phase 4 (`docs/runbooks/turn-on-and-smoke-test-runbook.md`, §"Serving oracle go-live", Activation step 1) names as the thing that must exist before `copilotServingOracleGoLive` can be armed. Today `oracleBuildOk` is effectively hardwired `false` because the coverage reader's `resolveOracleBuild` dep is **UNBOUND** (`servingContextBootReaders.ts:139` → `deps.resolveOracleBuild?.() ?? false`), so even a clean+complete `ParityReport` + valid pin STILL degrades the serving-coverage gate. This slice builds the producer that computes `oracleBuildOk` by **rebuilding the index from committed Markdown into a scratch space and comparing** — the "rebuild oracle."

**This slice = piece A of a 3-piece dormant arc** (mirrors the reconcile-TRIGGER arc's A → F1 → F2 shape):
- **A (this slice)** — the pure `probeRebuildOracle` producer composition over injected collaborators → a fail-closed `oracleBuildOk` status. Reachability-waivered.
- **B (next)** — the default-OFF `gateRebuildOracle` boot-gate helper (byte-equivalent BY CONSTRUCTION, no `bootWorker` edit; Lesson 23 split).
- **C (last)** — the `bootWorker` binding: run the producer once at boot behind the gate, cache the boolean, expose `resolveOracleBuild: () => cachedBool`, bind it into `createServingCoverageReader`. Real client UNBOUND ⇒ byte-equivalent; closes reachability.

**The hard line is NOT crossed by this arc.** `rebuildIndexFromMarkdown` (`@sow/knowledge`, already built) does its real gbrain scratch-import through an **injected `IndexRebuildClient` seam**. In this arc that real client stays **UNBOUND** (exactly like the reconcile arc's `makeDbAdapter → undefined`): tests inject a fake; boot leaves it unbound ⇒ the producer never runs ⇒ `oracleBuildOk` stays `false` ⇒ shipped default byte-equivalent. Binding a *real* `IndexRebuildClient` (real gbrain I/O) is the owner-gated crossing — NOT this arc.

## Things to flag at Step 2.5
1. **Outcome shape — richer typed status vs bare boolean.** Options: (a) a discriminated `RebuildOracleStatus` = `corroborated{oracleBuildOk:true, oracleSet} | absent{false} | diverged{false, healthItem} | faulted{false}`; (b) a flat `{ oracleBuildOk: boolean; oracleSet?; healthItem? }`. **Default vote (a)** — a typed status distinguishes a benign ABSENCE (empty vault — sound but inert) from a real DIVERGENCE-defect (health-worthy) from a FAULT, so piece C can route health correctly (mirrors reconcile piece C/D's 3-way outcome, Lesson 20/21). The value the coverage leg ultimately consumes is still just the boolean.
2. **Pin the rebuild to the head revision we read (`expectedRevisionId`).** `rebuildIndexFromMarkdown` accepts an optional `expectedRevisionId` that fail-closes BEFORE any client call on a stale snapshot (LIFE-6). **Default vote: PIN it to `snapshot.revisionId`** — closes a TOCTOU where the vault moves under us between the read and the rebuild; the snapshot IS the head we just read, so a mismatch means a race ⇒ degrade. Flag if the reader's revision and the snapshot's are not the same view.
3. **`RebuildOracleSet.factIdentities` extraction.** `DerivedFact` carries `factIdentity` (`canonical-fact-deriver.ts`), but it's a discriminated union — confirm `factIdentity` is uniformly present across variants (`facts.map(f => f.factIdentity)`) vs needing the existing identity accessor. **Default vote:** `facts.map(f => f.factIdentity)`; flag if the union makes it non-uniform. (`RebuildSuccess.facts` is the derived set the rebuild reconstructed.)
4. **Per-workspace probe vs boot-global fold.** `resolveOracleBuild: () => boolean` is a coarse boot-GLOBAL accessor (no ws arg, `servingContextBootReaders.ts:139`), but this probe is per-workspace (it reads one vault). **Default vote: slice A stays per-workspace** (`probeRebuildOracle(workspaceId, …)`); the boot gate (piece C) decides the fold (single served workspace ⇒ its result IS the global boolean; multi-ws ⇒ all-served-must-be-OK). Do NOT fold here.
