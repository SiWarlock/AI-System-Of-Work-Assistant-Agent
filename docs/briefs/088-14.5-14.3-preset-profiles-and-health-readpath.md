# /tdd brief — preset_profiles + system_health_readpath (14.5 + 14.3 worker legs, BUNDLED)

## Feature
**A bundle of two independent low-risk worker legs** (owner bundling directive):
- **Part A (14.5):** `presetProfiles` — map each onboarding preset (Simple/Professional/Founder/Advanced) to a concrete, DISTINCT provisioning profile `{ connectors, workflows, schedules, policyDefaults }`, so the four tiers are differentiated not cosmetic — with EVERY arming flip default-OFF regardless of preset.
- **Part B (14.3 worker leg):** verify-and-complete the System-Health read path — a minted degraded `HealthItem` (e.g. `temporal-unavailable` `worker_down`) is retained in the durable `HealthSurface` and round-trips through the read procedure **redaction-safe** (secrets/raw content stripped before the sink). Most of this is already built (§below); the worker leg confirms it end-to-end + fills any gap. The renderer panel is a later desktop-impl slice.

## Acceptance criteria (what "done" means)
**Part A — 14.5 preset profiles:**
- [ ] `presetProfiles(preset) → ProvisioningProfile` (`apps/worker/src/composition/presetProfiles.ts`) maps each of the 4 presets to a profile `{ connectors: string[], workflows: string[], schedules: [...], policyDefaults: {...} }`.
- [ ] **Divergence:** the 4 presets yield DISTINCT profiles (different default connector sets / enabled-workflow sets) — proven by a test asserting pairwise divergence (e.g. Simple = minimal spine; Founder/Advanced surface more connectors + the cross-calendar workflow).
- [ ] **No profile arms a hard line (safety):** EVERY arming flip in EVERY profile is default-OFF — a test asserts no profile enables a real-spend / real-write / propose / real-transport flip (the profile sets defaults only; arming stays owner-confirmed regardless of preset).
- [ ] `policyDefaults` build on the fail-closed safe posture (egress CLOSED by default — never a preset that pre-acknowledges Employer-Work egress).

**Part B — 14.3 System-Health read path (worker leg):**
- [ ] A minted degraded `HealthItem` (drive `temporal-unavailable` `worker_down`, or another named producer) is RETAINED in the durable `HealthSurface` and round-trips through the read procedure as a `UiSafeHealthItem` — proven by a test that mints → reads it back.
- [ ] **Redaction (safety rule 7):** the read path strips secrets + raw content before the sink — the retained item surfaces ref-only (no `message`/`auditRef`/`factIdentity`); a test pins a HealthItem whose `message` contains a marker-secret does NOT leak it through the read.
- [ ] Any named producer (`worker_down`, connector `auth_locked`, `parity_defect`, coverage-degrade, `keychain-locked`) NOT already routed through the durable surface is wired (or its non-routing is documented as a named follow-up if out of worker scope). The renderer panel (`surfaces/system-health/`) + the retained-but-UNREAD round-trip (24.3) are explicitly deferred.

**Both:**
- [ ] `/preflight` clean (worker + repo-wide).
