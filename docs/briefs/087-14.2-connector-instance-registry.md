# /tdd brief — connector_instance_registry (14.2 worker leg)

## Feature
The **per-workspace connector-instance config registry**: a `@sow/db` connector-instance table + dual-dialect migration + repository, and the `connectorConfig` tRPC procedure (register a connector bound to a workspace, record its **tokenRef reference**, enable/pause, set cadence) — the durable config record the Phase-16 connector composition + Phase-23 arming later consume. **NO real credential is provisioned — only a tokenRef *reference*; no live vendor call.** Worker/db-track; the connectors settings UI is a later desktop-impl slice.

## Use case + traceability
- **Task ID:** 14.2 (worker leg)
- **Architecture sections it implements:** `ARCHITECTURE.md §19.1` (primary), `§8` (connector config surface / Connector Gateway registry), `§4` (operational storage), `§11` (settings surface — UI later). **Safety rule 7** (secrets via SecretsPort/Keychain — the record holds a tokenRef reference, never the credential).
- **Related context:**
  - **Dependency:** 14.1 workspace registry (`e1e4cfd`) — a connector-instance binds to a REGISTERED workspace; reuse the fail-closed `resolveKnownWorkspace` membership gate.
  - **Pattern to mirror (just built):** the operational-config-record pattern from 14.1 `WorkspaceConfigRepository` + 14.6 `ProjectRegistryRepository` (`b9dc03b`/`21ee906`) — a db-owned Row DTO (contracts primitives) + repo-contract suite on both dialects + WS-8 membership gate + the **workspace-binding immutability guard** + a boot-wired create/config procedure.
  - **tokenRef precedent:** `GbrainReadGrant.tokenRef` (an opaque *reference* to a Keychain secret — never the secret bytes). The record's tokenRef is the same kind of reference.
  - **Existing cadence/enablement surfaces (confirm at Step 1):** `connectorSyncHealth` / `instanceLease` / `EnablementConditions` — reuse a cadence/schedule type if one fits.

## Acceptance criteria (what "done" means)
- [ ] A `@sow/db` connector-instance table + dual-dialect (SQLite + Postgres) migration + a `ConnectorInstanceRepository` (`upsert`, `get`, `listByWorkspace`, `setState`, `setCadence`) passing the ONE repo-contract suite on BOTH dialects.
- [ ] The record shape round-trips: `{ instanceId, connectorId/vendor, workspaceId, tokenRef (opaque REFERENCE), state (enabled|paused), cadence }`. A db-owned Row DTO (contracts primitives), mapped at the worker boundary if a port type is warranted.
- [ ] **tokenRef reference-only (security-reviewer=invariant, safety rule 7):** the record persists an opaque tokenRef REFERENCE (a Keychain key identifier) ONLY — never credential/token bytes. A test pins that the procedure neither accepts nor persists raw secret material, and NO live vendor call happens (the record is config only). Binding a real credential/transport is Phase 17/23.
- [ ] **WS-8 (security-reviewer=invariant):** a connector-instance binds only to a workspace KNOWN in the 14.1 registry — an instance for an unregistered workspace is rejected (fail-closed); the caller's workspaceId is validated against the registry, never a bypass.
- [ ] **Workspace-binding immutability (consistent with 14.1/14.6 — pre-baked):** re-registering an existing `instanceId` with a CHANGED `workspaceId` ⇒ rejected (typed `connector_instance_workspace_immutable`); same-workspace re-register/update is idempotent; a fault on the existence check fails closed (no upsert).
- [ ] The `connectorConfig` tRPC procedure (register / enable / pause / set-cadence / bind-tokenRef) is boot-wired, WS-8 membership-gated, and redaction-safe (typed Results; never echoes the tokenRef target or any secret — §16 / rule 7).
- [ ] pause/enable + cadence toggles persist and round-trip from the durable registry.
- [ ] `/preflight` clean (worker + repo-wide; repo-contract green on both dialects).

## Things to flag at Step 2.5
1. **Record identity/key — a synthetic `instanceId`, or a `(vendor, workspaceId)` composite key?** My default vote: **synthetic `instanceId`** — a workspace could legitimately have >1 instance of the same vendor (e.g. two Google accounts); a composite key forecloses that. Flag if the model is strictly one-per-vendor-per-workspace.
2. **tokenRef shape — reuse the `GbrainReadGrant.tokenRef` opaque-reference convention?** My default vote: **yes** — an opaque ref string to a Keychain entry, validated non-secret-shaped, never the credential. [security-reviewer=invariant]
3. **Workspace-binding immutability — apply the same guard as 14.1/14.6?** My default vote: **YES** (pre-baked into acceptance #5) — the workspace binding is the isolation anchor; a silent rebind moves the connector (and its future reads) across the boundary. Same `get-before-upsert` guard.
4. **cadence type — reuse an existing cadence/schedule type or a new field?** My default vote: **reuse** an existing cadence/enablement type if one fits (`connectorSyncHealth`/`EnablementConditions`); else a simple interval/schedule value. Confirm the surface at Step 1.
