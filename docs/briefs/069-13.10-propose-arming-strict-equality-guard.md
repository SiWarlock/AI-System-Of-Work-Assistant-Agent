# /tdd brief — propose_arming_strict_equality_guard

## Feature
Add a **regression guard test** (test-only; zero production change) pinning that the propose / serving-oracle **false-arming** checks are strict-equality (`=== true`) and can never silently weaken to a truthy coercion. The audit named three arming sites; a future refactor that changed any `=== true` to a truthy check (`if (config.X)`) would let a non-boolean truthy config value (`1`, `"false"`, `{}`) silently ARM the go-live path. This slice makes that regression fail a test.

- **Task ID:** 13.10 (arming-prep hardening — anti-false-arming coverage for the go-live / serving-oracle trust gate).
- **Architecture sections it implements:** `ARCHITECTURE.md §12` (parity / serving GO conditions — the serving oracle is the trust axis; nothing stamped ⇒ untrusted ⇒ propose OFF) + `§13` (deployment / go-live gate — arming is owner-gated, never a flag-flip and never a truthy accident).
- **Related context (the three audit-named strict sites):**
  - `apps/worker/src/boot.ts:1171` — `provenanceStampingEnabled: config.copilotProvenanceStamping === true`.
  - `apps/worker/src/boot.ts:1173` — `goLiveArmed: config.copilotServingOracleGoLive === true // OFF-lock 1 (default unset ⇒ false)`.
  - `apps/worker/src/api/procedures/servingContextLoader.ts:239` — `if (sel.goLiveArmed === true && sel.loaderBacked !== undefined) return sel.loaderBacked;` (the pure `selectServingOracleFactory`).
  - **Subtlety worth pinning:** `selectServingOracleFactory` internally truthy-checks `provenanceStampingEnabled` (`:238` `if (!sel.provenanceStampingEnabled) return undefined;`) — the STRICTNESS for provenance-stamping lives ONLY at the boot mapping (`:1171`). So the boot.ts:1171 leg needs its OWN guard; the pure-fn test cannot cover it.
  - Worker **Lesson 23** (strict `=== true` guard-first — a truthy-coerce is a false-arming vector) + **Lesson 8** (independent OFF-locks; the shipped default is byte-equivalent, proven by adversarial pairwise-defeat). This slice is the adversarial-defeat coverage for the strict-equality property those lessons assert.

Pins `servingContextLoader.ts:239`.
- [ ] Positive control: `selectServingOracleFactory({ provenanceStampingEnabled: true, loaderBacked: <fake>, goLiveArmed: true })` returns the fake `loaderBacked` — so the guard test is NOT vacuous (the gate genuinely arms on a real `true`).
- [ ] A truthy-not-`true` `config.copilotServingOracleGoLive` does NOT arm the boot go-live path (the selected serving oracle stays interim/degraded ⇒ nothing stamped). Pins `boot.ts:1173`.
