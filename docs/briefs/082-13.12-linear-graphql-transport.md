# /tdd brief — linear_graphql_transport (connector round 6 slice 2 — Linear GraphQL read transport)

## Feature
Add **`createLinearHttpTransport(deps): ConnectorTransport`** — Linear issues read-only GraphQL transport — a thin per-vendor spec over the template's new POST+body path (slice 1, brief 081). Round 6 slice 2; the template's FIRST GraphQL-over-POST connector and the first consumer of the optional POST method + `buildBody`.

**GraphQL read, query-only.** Linear is a single `POST https://api.linear.app/graphql`; the query rides the JSON body. The connector sends ONLY a FIXED, whitelisted `query` operation (an issues list) — **NEVER a `mutation`** (a GraphQL POST endpoint accepts both on the same URL; read-only is the SPEC's contract + review, since the transport can't inspect an opaque body — brief 081's documented posture). Paging is Relay-style cursor via GraphQL `variables` (`first`/`after`), never string-interpolated into the query. Reuses the R5 widened `mapPage(json, request)` (the Option-A pre-pay) for the `pageInfo{hasNextPage,endCursor}` cursor.

**Dormant + no hard line.** Real `HttpTransport` + `SecretsAccessor` + the Linear token stay UNBOUND (byte-equivalent); binding is the owner's arming crossing.

## RED test outline (`connector-linear-transport.test.ts`)
1. **`linear_query_maps_to_page`** — a 200 GraphQL body `{data:{issues:{nodes:[{id,updatedAt,title}], pageInfo:{hasNextPage:true, endCursor:"c1"}}}}` ⇒ `TransportPage` (recordId=id, stable `payloadHash({id, updatedAt})`, raw=node, `nextCursor="c1"`, `done=false`); `hasNextPage:false` ⇒ `done=true`, no `nextCursor`.
2. **`linear_pagination_strict`** — `hasNextPage` non-`true` / `endCursor` null|empty|absent ⇒ `done=true`, no cursor advance (STRICT fail-closed — mirrors Granola).
3. **`linear_query_only_variables` (READ-ONLY + injection — lead reinforcements #1/#2):** the constructed body is `{query: LINEAR_ISSUES_QUERY, variables:{first, after}}`; the query string contains **no `mutation`** (query-only); the cursor rides `variables.after`, NOT the query text; first page ⇒ `after: null`.
4. **`linear_graphql_errors_fail_closed`** — a 200 body with a top-level `{errors:[...]}` (no data) ⇒ `TransportFailure`; also missing `data` / `data.issues.nodes` not-array / node missing `id` / non-object ⇒ `TransportFailure` (a GraphQL error is NOT a page).
5. **`linear_ssrf_and_rule7_on_post`** — admit `api.linear.app` over POST; an off-allowlist host refused (zero dispatch); the token appears ONLY in `Authorization`, NOWHERE in the body; 401 ⇒ `TransportFailure(auth_locked)`, safe status only.
6. **`linear_end_to_end_via_createLinearConnector`** — whole `ConnectorPort.fetch` chain over the POST transport (records/contentHash/payload; `done`/`nextCursor` threaded).
