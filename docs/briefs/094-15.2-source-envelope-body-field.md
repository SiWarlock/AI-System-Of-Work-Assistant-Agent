# /tdd brief — source_envelope_body_field (15.2)

## Feature
Thread source **content** through the spine: extend the frozen `SourceEnvelope` seam model with an **optional additive `body` field** — the candidate extracted text — so the note-body projection (15.3) can build a real note from the validated extraction instead of the `"source ingestion (C1)"` placeholder. `body` is **candidate data** (schema-gate-validated, never a bypass). **Contract-track** (`packages/contracts`): an ADDITIVE frozen-Appendix-A-seam change — the Zod model + generated JSON Schema + the co-frozen snapshot + the Appendix-A row move together (ADR-008 / contracts forbidden-pattern #3, Lesson 2).

## Use case + traceability
- **Task ID:** 15.2
- **Architecture sections it implements:** `ARCHITECTURE.md §19.2` (primary — source content threading), `§8` (source ingestion / SourceEnvelope); the model itself is an **Appendix A** frozen seam.
- **Related context:**
  - `SourceEnvelope` (`packages/contracts/src/models/source-envelope.ts`) today: `sourceId, workspaceId(req), origin, contentHash, type, sensitivity, routingHints`. Add `body?`.
  - **Lesson 15 (contracts) — additive-field discipline:** a NEW field ships OPTIONAL + standalone + fixture-tested, NEVER composed as a required gate until producers emit it (a required `body` would drop every existing source — nothing carries it yet). The 15.3 note-projection consumes `body` when present, degrades when absent.
  - **Lesson 2 (ADR-008 recipe):** each Appendix-A model = 4 files — `.ts` (Zod, `.strict()`) → generated `schemas/source-envelope.schema.json` → frozen `__snapshots__/source-envelope.snap` (top-level field-name set — `body` is a DELIBERATE add) → the schema-snapshot test. A field add REQUIRES editing the model + schema + `.snap` in the SAME round; `registry-all.test.ts` + the per-model snapshot test fail otherwise.
  - **Lesson 15 (embedder ripple):** an additive field on `SourceEnvelope` ripples through any embedder's generated JSON schema (`freezeGenerated` catches the nested add); use an explicit field-PICK (not a spread) at any `.strict()` boundary so `body` can't leak downstream until 15.3 intends it. Confirm embedders at Step 1.
