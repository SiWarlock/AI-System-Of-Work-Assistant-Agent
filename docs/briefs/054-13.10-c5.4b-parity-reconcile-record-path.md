# /tdd brief — parity_reconcile_record_path

## Feature
Build the **worker-side write path** that persists a reconciliation pass's `ParityReport` into the durable `ParityReportStore` (built B1). A narrow fail-closed **`ParityReportRecorder`** write-port + db-backed adapter (over the repo's `record`), plus a **`recordReconcileOutcome`** gate that records the report **only on a successful `reconcileParity` outcome** — an `err` is NEVER coerced into a stored "clean" report. Records the report **verbatim** (the reconciler's real `cleanForServing`/`coverageComplete` are never synthesized). Ships **DORMANT + reachability-waivered**: no reconcile trigger, no boot change, no caller — a later slice wires the Temporal reconcile trigger; B4 stays boot-binding + the GREEN-admission e2e.

**PRODUCER CORRECTION (lead Ruling A, 2026-07-13):** B3 records from **`reconcileParity`** (`packages/knowledge/src/gbrain/parity/reconciler.ts:110`) — the full task-4.16 ParityReconciler (classifies the full canonical∪db union incl. `present:"both"` → real `content_mismatch` detection + the rebuild-oracle `coverageComplete` determination). It does **NOT** record from `checkGbrainParity` (`parity.ts:139`), the narrow task-4.9 primitive that sees only `db_only` and would over-claim coverage at the trust gate — a false-clean defeat brief 052 forbids. The `checkGbrainParity` naming in briefs 052/053 + the tracker was an inherited context-note error; it is corrected in those docs this round.

## Acceptance criteria (what "done" means)
- [ ] A narrow **`ParityReportRecorder`** write-port (`record(report: ParityReport): Promise<void>`) + **`createParityReportRecorderAdapter(repo, now)`** — supplies `recordedAt` from the injected `now()` clock and delegates to `repo.record`; **REJECTS on a `DbError`** (fail-closed, mirroring the B1 read adapter — a store fault is visible, never a silent ok).
- [ ] **`recordReconcileOutcome(outcome, recorder): Promise<ParityRecordDisposition>`** records **only on `ok`** — `await recorder.record(outcome.report)` — and returns `{ kind: "recorded"; report }`. On `err` it records **nothing** and returns `{ kind: "skipped_reconcile_error"; error }` (guardrail #1 — a reconcile error is never coerced into a stored clean report; the future caller routes it to health/degrade).
- [ ] The report is recorded **VERBATIM** — `recordReconcileOutcome` passes `outcome.report` straight through; it never synthesizes/defaults `coverageComplete` or `cleanForServing` (guardrail #2 — a missing/partial rebuild-oracle's `coverageComplete=false` must not read as complete-and-clean).
- [ ] A **dirty** report (`cleanForServing=false`, a HARD divergence found) on a successful reconcile IS recorded verbatim (operational truth — the serve-time signal that makes serving degrade; never dropped).
- [ ] **Fault-vs-skip distinction (§12 fail-closed):** a `record` `DbError` REJECTS through `recordReconcileOutcome` (the future caller degrades + raises health); a reconcile `err` is a typed `skipped_reconcile_error` disposition (routed to health) — the two are distinguishable, neither is a silent success.
- [ ] Ships **DORMANT + reachability-waivered** — NO reconcile trigger, NO boot change, NO production caller; `record` idempotency (first-write-wins on `reportId`) means a replayed pass is a no-op.
- [ ] All unit tests pass; repo-wide `pnpm -w turbo run typecheck test` green; `/preflight` clean.
- [ ] **MANDATORY Step-8 dual review = security-reviewer + code-quality-reviewer** — this write path feeds the serve-time trust-gate coverage source; a wrongly-stored clean report is a trust-gate defeat.

## RED test outline (Step 2)
1. **`recorder_adapter_records_via_repo`** — `createParityReportRecorderAdapter(repo, clock).record(report)` calls `repo.record(report, clock())` + resolves. Asserts: the adapter supplies `recordedAt` from the injected clock + delegates. Why: §6 durable persistence.
2. **`recorder_adapter_rejects_on_db_fault`** — `repo.record` ⇒ `err(DbError)` ⇒ the adapter REJECTS. Asserts: a store fault is visible, never a silent ok. Why: §12 fail-closed (mirrors the B1 read adapter).
3. **`record_reconcile_outcome_ok_records_verbatim`** — `recordReconcileOutcome(ok(outcome), recorder)` ⇒ `recorder.record` called with `outcome.report` **byte-equal** (no field mutation — `coverageComplete`/`cleanForServing` untouched) ⇒ `{ kind: "recorded", report }`. Asserts: guardrail #2 (the reconciler's real determination recorded as-is). Why: §6/§12.
4. **`record_reconcile_outcome_workspace_mismatch_never_records`** — `recordReconcileOutcome(err({code:"workspace_mismatch",…}), recorder)` ⇒ `recorder.record` NOT called ⇒ `{ kind: "skipped_reconcile_error", error }`. Asserts: guardrail #1 (a reconcile err never coerced into a stored clean report). Why: §12 fail-closed.
5. **`record_reconcile_outcome_report_invalid_never_records`** — same for `err({code:"report_invalid",…})`. Asserts: the other err variant is also never recorded. Why: §12.
6. **`record_reconcile_outcome_propagates_record_fault`** — `ok(outcome)` but `recorder.record` REJECTS (DbError) ⇒ `recordReconcileOutcome` REJECTS (does NOT swallow to a "skipped" disposition). Asserts: a record fault ≠ a reconcile-err skip — the caller degrades on a fault (fault-vs-skip distinction). Why: §12 fail-closed both directions.
7. **`record_reconcile_outcome_dirty_report_still_recorded`** — `ok(outcome)` where `report.cleanForServing=false` ⇒ STILL recorded verbatim ⇒ `{ kind: "recorded", report }`. Asserts: a defect report is operational truth (the serve-time degrade signal), never dropped. Why: §6.

## Things to flag at Step 2.5
1. **Test + code file placement.** Default vote: recorder port+adapter + the `recordReconcileOutcome` gate all in `parityReportStore.ts` (all parity-store wiring in the one composition file, symmetric with the B1 read port); a NEW `parityReconcileRecording.test.ts` for the gate cases. Rationale: cohesion; the gate is the store's write side. Alternative: split the gate into its own `parityReconcileRecording.ts` if the file grows unwieldy.
2. **`ParityRecordDisposition` shape.** Default vote: a discriminated union `{ kind: "recorded"; report } | { kind: "skipped_reconcile_error"; error: ReconcileError }` — the future caller switches on `kind` to route health; a `record` DbError is NOT a disposition variant (it REJECTS). Rationale: keeps fault (reject) and skip (typed disposition) distinguishable.
3. **Does B3 surface `outcome.healthItems` / route degrade?** Default vote: **NO** — B3 is the RECORD gate only; the future trigger slice (which holds the full `ReconcilerOutcome`) routes `healthItems` + the `skipped_reconcile_error` to health/degrade. Keeps B3 minimal + dormant. (Do NOT pull health-routing into B3.)
4. **Record-fault handling in the gate — reject vs typed-degrade-disposition.** Default vote: **reject** (propagate the record `DbError` as a rejected promise, mirroring the B1 read adapter + the recorder port). Rationale: a store fault must be visible/fail-closed for the caller to degrade; folding it into a "skipped" disposition would blur fault-vs-skip.
