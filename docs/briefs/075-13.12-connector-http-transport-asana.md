# /tdd brief — connector_http_transport_asana (connector template, slice 2 — the REUSABLE template + the Asana instance)

## Feature
Build the reusable **`createConnectorHttpTransport`** — a real read-only HTTP `ConnectorTransport` builder — and its first instance **`createAsanaHttpTransport`**. This is the round-2 deliverable: the template every remote connector (Granola/Drive/Calendar/Todoist/Linear/GitHub, round 3+) specializes, established on Asana first (lowest risk: a read-only `tasks:read` base + a documented REST API). It mirrors the GbrainReadClient / knowledge **Lesson 1** pattern but produces a `ConnectorTransport` (the `@sow/integrations` seam) instead of a gbrain read.

**Dormant + no hard line.** The real `HttpTransport` + `SecretsAccessor` (the Asana PAT) stay **UNBOUND** — the shipped tree has no real vendor transport wired (only `createFileReadTransport`, for file sources), so `createAsanaHttpTransport` is constructible + fully unit-tested over fakes but never bound to a real network client in the default boot (byte-equivalent). Binding a real `HttpTransport` + provisioning the Asana token is the **owner's arming crossing** (real external network I/O = HARD LINE) — NOT this slice.

## Acceptance criteria (what "done" means)
- [ ] `createConnectorHttpTransport(spec, deps): ConnectorTransport` exists in `@sow/integrations` (e.g. `adapters/http-transport.ts`). `spec` carries the connector-specific config (base URL, allowlisted hosts, resource path, and a candidate `mapPage` wire-mapper); `deps` carries the INJECTED `{ transport: HttpTransport, secrets: SecretsAccessor, tokenRef }` (UNBOUND in prod).
- [ ] **The flow (mirror GbrainReadClient), fail-closed + RETURNING a typed `TransportFailure` (never throwing across the `ConnectorTransport` seam, §16):**
  1. **SSRF guard FIRST** — `isAllowedRemoteEndpoint(fullUrl, spec.allowedHosts)`; reject ⇒ a redacted `TransportFailure` (safe host ref via `endpointHostRef`), **zero token read, zero dispatch**.
  2. **Token** — `secrets.getSecret(tokenRef)`; a typed unavailable OR a THROWING accessor ⇒ a redacted `TransportFailure` (reason token only, never the token). Rides ONLY the `Authorization` header; never logged/echoed.
  3. **Build the GET** — read-only (ING-7): method GET, the cursor→vendor paging param, `Authorization: Bearer <token>`. `readScope` is the adapter's declared least-privilege read scope (informational).
  4. **Dispatch** — `transport.send`; a reject ⇒ a redacted `TransportFailure` (raw cause discarded).
  5. **Positive 2xx gate** — non-integer / <200 / ≥300 ⇒ a redacted `TransportFailure` carrying ONLY the safe status number (never the body).
  6. **Parse** the 2xx body; non-JSON ⇒ a redacted `TransportFailure`.
  7. **Map** via `spec.mapPage(json)` → `TransportPage {ok:true, items, nextCursor?, done}`.
- [ ] `createAsanaHttpTransport(deps): ConnectorTransport` = `createConnectorHttpTransport(ASANA_SPEC, deps)` with `ASANA_SPEC = { baseUrl: "https://app.asana.com/api/1.0", allowedHosts: ["app.asana.com"], resourcePath: "/tasks", readScope: "tasks:read", mapPage: asanaMapPage }`.
- [ ] **Candidate wire shape (arch_gap, Lesson 21):** `asanaMapPage` maps the Asana list envelope (candidate: `{ data: Task[], next_page?: { offset } }` → `items = data.map(t => ({ id: t.gid, hash: <stable>, raw: t }))`, `nextCursor = next_page?.offset`, `done = next_page === undefined`) — parsed FAIL-CLOSED (a missing/renamed field ⇒ a `TransportFailure`/empty-done, never a false page). Documented in-code as NOT confirmed until run at arming.
- [ ] **Every fault is redaction-safe** (safety rule 7): only a status number / a `SecretUnavailable.reason` / a host-only ref — never the token, raw body, or raw cause. Round-trips through a fake transport in tests.
- [ ] **Dormant + reachability-waivered:** no boot wiring (the real `HttpTransport`/`SecretsAccessor` stay unbound; `createAsanaConnector` is not bound to this transport in the shipped default). Byte-equivalent. `/preflight` clean; repo-wide green.

## RED test outline (Step 2)
1. **`ssrf_guard_rejects_before_token_and_dispatch`** — an off-allowlist / non-https / loopback base URL ⇒ `TransportFailure`; the fake `secrets` + `transport` spies assert ZERO invocations (guard is first).
2. **`token_unavailable_and_throwing_accessor_fail_closed`** — a typed-unavailable secrets result AND a THROWING accessor both ⇒ `TransportFailure` (redacted reason); no dispatch; the token never appears in the failure.
3. **`non_2xx_and_transport_throw_redacted`** — a 500/401/429 status AND a rejecting transport ⇒ `TransportFailure` carrying only the safe status / host ref (never the body/cause).
4. **`malformed_body_fails_closed`** — a 2xx non-JSON body ⇒ `TransportFailure` (body never echoed).
5. **`asana_2xx_maps_to_page`** — a candidate Asana envelope (`{data:[…], next_page:{offset}}`) ⇒ `TransportPage` with `items` (gid→id, stable hash, raw preserved), `nextCursor = offset`, `done=false`; and a no-`next_page` body ⇒ `done=true`. Drive it end-to-end through `makeConnector`/`createAsanaConnector` to prove the whole `ConnectorPort.fetch` chain (raw→records).
6. **`token_rides_only_authorization_header`** — assert the dispatched request carries the token ONLY in `Authorization` and it never enters any `TransportFailure`.

## Things to flag at Step 2.5
