## Things to flag at Step 2.5
1. **The single arming input — env var vs `BootConfig` field (LOAD-BEARING).** The owner's step-6 flip must arm the WHOLE bundle at once (transport gate + armed route + auth-guard fire + health source + ContextRef), AND-locked (one flip, no split-brain — L52). Options: (a) a `BootConfig.subscriptionArming?` opt-in object (mirrors `proofSpineParams`, carries the owner-set model id + cap so they're config-not-hardcoded — L2/L29; presence + `enabled === true`); (b) a default-OFF `SOW_*` env flag (mirrors `SOW_INGEST_WATCH`/L37) with model+cap still on config. My default vote: **(a) a `BootConfig.subscriptionArming` opt-in object** — it single-sources the owner-set values through config (not env), keeps `assertSubscriptionAuthEnv` as the ENV guard (separate concern), and matches the `proofSpineParams` owner-opt-in precedent. Confirm the exact shape + that boot derives `armed` from the SAME predicate the route knob / `selectProviderRunner` read (`providerTransport.enabled === true`) so they can't split.
2. **Auth-guard failure mode on the armed boot path.** A set shadowing var on the armed path ⇒ what? Options: a typed boot error that HALTS boot; a degraded HealthItem + fail-closed (no arming); a hard throw. My default vote: **a fail-closed refusal that surfaces (boot-visible typed error / degraded HealthItem) and does NOT construct the transport gate** — loud, never a silent raw-API fallback (§16/L52/L39). A mis-provisioned armed path SHOULD be loud; prefer degrade-and-surface over crash where the boot contract allows.
3. **ContextRef population — co-gate with arming, or populate-always + pin-no-effect?** Populating `{refKind:"source"}` on EVERY source-process job (even unarmed) changes the shipped `AgentJob` shape (the resolver only reads it on the armed cloud route; the local route ignores it). Options: (a) co-gate the population with the arming input ⇒ flag-OFF byte-equivalent, but the assembly reads the arming signal; (b) populate always + PIN that admission/veto/route-resolution on the local path are unaffected. My default vote: **(a) co-gate** — preserves the flag-OFF byte-equivalence invariant every prior slice protected; the assembly already sits where the arming signal is reachable. If tracing shows the assembly can't see the arming signal cleanly, fall back to (b) with an explicit local-path-unaffected pin.
4. **Locate the exact ContextRef-assembly site.** Trace where the `source.process` `AgentJob` is built (the source-extraction activity / the `gateAutoIngest` → `AutoIngestWiring.sourceIngestion.extraction` path / `buildActivities` source leg). 5. **Slice split (safety).** This bundles the composition-root wiring with the safety helpers. My default vote: **split into two commits** — **(A)** the pure helpers (full 8-var enumeration + `gateSubscriptionExtraction` gate + single-source route export + memoized health binding), byte-equivalent BY CONSTRUCTION, no boot edit (L23); **(B)** the boot composition-root edit (call the gate, augment `providerTransport`/`capabilityDefaults`, fire the auth guard) + the ContextRef population, the focused-review commit. security-reviewer on BOTH.
