# /tdd brief — autoingest_output_subtree_feedback_guard (15.6)

## Feature
Stop the auto-ingest vault watcher from re-firing on the KnowledgeWriter's OWN ingestion-note output (closes **G6**). The watcher already scopes to `.md` under the vault root (`isMarkdown`); 15.6 adds an **output-subtree exclusion** so a written ingestion note (`sources/<ws>/<digest>.md`, per `deriveSourceNotePath`) is NOT captured/dispatched — today every written note lands inside the watched root and re-triggers, an infinite feedback loop (source → KW writes note.md → watcher re-fires → re-ingest → …). Worker-track; **ISOLATED, security-reviewer=invariant** (a resource-safety loop-guard whose robustness + no-over-exclusion matter). Pure-build — the watcher is default-OFF (`SOW_INGEST_WATCH`); NO hard line.

## Acceptance criteria (what "done" means)
- [ ] The watcher EXCLUDES a captured `.md` whose relPath is under the ingestion-output subtree (`sources/…`, root-anchored) — such a path produces NO dispatch (the feedback loop is broken). Asserted by a watcher test that COUNTS dispatches (0 for an output-subtree note).
- [ ] **No over-exclusion:** a `.md` OUTSIDE `sources/` (a real user/source note) STILL dispatches — the guard is narrow, doesn't break legitimate ingestion.
- [ ] **Root-anchored, separator-safe:** the exclusion matches `sources/` at the vault-root (a `sources/` path SEGMENT boundary) — `sources/ws/x.md` excluded, but a nested `mynotes/sources/x.md` (a user note, NOT the output) still dispatches, and a `sourcesX.md` (no separator) is NOT excluded.
- [ ] **No drift from the real output path:** the excluded subtree is exactly what `deriveSourceNotePath` produces — a test writes at the REAL `deriveSourceNotePath(ws, identity)` output and asserts it's excluded (ideally via a SHARED subtree constant so the watcher-exclusion and the note-path producer can't drift — see Q2).
- [ ] **`.md`-only scope unchanged:** a non-`.md` path still never dispatches (the existing scope is preserved).
- [ ] `/preflight` clean (worker + repo-wide).

## Things to flag at Step 2.5
1. **Exclusion predicate (LOAD-BEARING).** Root-anchored `sources/` segment match on the relPath (`sources/…` excluded; `mynotes/sources/…` + `sourcesX…` NOT). My default vote: a `relPath === "sources" || relPath.startsWith("sources/")` (segment-boundary) check on the vault-relative path. Confirm the exact match + that `sources/` is a RESERVED system subtree (the user doesn't put their own top-level `sources/` notes there — or if they might, is excluding them acceptable? 2. **Shared subtree constant (no-drift, recommended).** `deriveSourceNotePath` hardcodes `sources/…`; the watcher exclusion must use the SAME prefix. My default vote: extract a shared `SOURCE_NOTE_SUBTREE = "sources"` constant referenced by BOTH `sourceNotePath.ts` and the watcher, so a future change to the output subtree can't silently break the exclusion (re-opening the loop). Confirm — shared constant vs a duplicated literal + a drift-pinning test.
3. **Path form.** The watcher captures relPaths (vault-relative); confirm the exclusion operates on the relPath (not absolute), and the `deriveSourceNotePath` output is the same relative form (`sources/<ws>/<digest>.md`) so they compare cleanly.
