# /tdd brief — triage_reroute_target (15.8 worker leg)

## Feature
Extend the ingestion-triage disposition command so a **`reroute` disposition carries an explicit target `workspaceId` (and optional `projectId`)** that is **registry-validated** (the 14.6 typed-Project / WS-8 workspace registry) before the pipeline re-entry — closing the exact **REQ-F-017 no-inference** ambiguity the low-confidence park exists for: a parked "which workspace/project?" item is **human-resolved by an explicit target**, never invented. Today `disposeTriageCommand`/`TriagePort.reenterIngestion` (`apps/worker/src/api/procedures/triageCommands.ts:45-49,64-77`) carry only `{ sourceId, idempotencyKey, disposition }` — there is no way to say *where* to reroute. Worker-track (the **worker leg** of 15.8). **Pure-build, NO hard line** — re-entry stays replay-safe (idempotencyKey reused verbatim) and one-writer (the pipeline is the only writer); no external transport, no propose-flip. Closes the worker half of **G60**.

## Shared command contract (pinned — the desktop leg, brief 106, consumes this verbatim)
Extend the command + port input to (additive, optional target so accept/reject stay valid):
```ts
{
  sourceId: string;
  idempotencyKey: string;         // reused verbatim on re-entry (ING-4 replay-safety)
  disposition: TriageDisposition; // e.g. "accept" | "reject" | "reroute"
  target?: {                      // REQUIRED when disposition === "reroute"; forbidden otherwise
    workspaceId: string;          // registry-validated (must exist + be membership-permitted)
    projectId?: string;           // optional; if present, must resolve in the 14.6 Project registry under that workspace
  };
}
```
The result surface stays `{ idempotencyKey }` (the replay-safety proof).

## Acceptance criteria (what "done" means)
- [ ] **Reroute requires an explicit target (REQ-F-017):** `disposition === "reroute"` with NO `target` (or a target missing `workspaceId`) ⇒ a typed rejection (e.g. `reroute_target_required`), NEVER a guessed/defaulted workspace. Pin it.
- [ ] **Target is registry-validated (WS-8, L36/L30/L32):** a `target.workspaceId` not in the workspace registry ⇒ typed rejection (e.g. `reroute_target_unknown`); a `target.projectId` present but not in the 14.6 Project registry under that workspace ⇒ typed rejection. Never a raw bind of an unvalidated target. Pin both.
- [ ] **Happy reroute re-drives with the resolved target, reusing the idempotencyKey:** a valid reroute re-enters the ingestion pipeline through the injected `TriagePort.reenterIngestion` carrying the validated target, REUSING `idempotencyKey` verbatim (replay-safe). Pin the reused key on the result + that the port received the target.
- [ ] **accept/reject unchanged (byte-equivalent):** a non-reroute disposition with no target behaves exactly as today (additive-optional — existing callers/tests valid, worker L15/L35). Pin.
- [ ] **Idempotent replay (rule 3 / ING-4):** a re-applied reroute (same idempotencyKey) does not double-drive (mirror the 15.5 re-enter replay). Pin.
- [ ] **One-writer / never-throws (§16, safety 1/3):** the command performs NO direct Markdown/external write — dispatch only via the injected port; every path returns a typed `Result` (a validator/registry fault fails closed to a typed err, never throws). Confirm at Step 8.
- [ ] `/preflight` clean (worker + repo-wide).
