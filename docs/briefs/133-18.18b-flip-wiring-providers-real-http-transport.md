The real, dependency-injected `HttpTransport` implementation (`createRealModelHttpTransport`) over Node `fetch` for the model-provider adapters — the concrete **`<real fetch client>`** the owner injects into the crossing gate (`config.providerTransport.make → createRealProviderRunner({ transport, ... })`). **Mock-fetch tested (deterministic, no real network); NOT enabled** (zero production call-site until the worker gate-assembly helper injects it at owner arming). ## Acceptance criteria (what "done" means)
- [ ] `createRealModelHttpTransport(deps): HttpTransport` implements `send(req: HttpTransportRequest, signal?: AbortSignal): Promise<HttpTransportResponse>` by calling injected `fetch(req.url, { method: req.method, headers, body: req.body, signal })` and returning `{ status: res.status, body: await res.text() }`.
- [ ] `fetch` is **injected** (default = the Node 22 global `fetch`) so tests substitute a deterministic fake — **no real network call in any test.**
- [ ] Non-2xx responses pass through as `{ status, body }` with **no throw and no classification** — the shared `executeCompletion` maps status → `ProviderError` (`providerErrorFromStatus`). The transport never returns a `ProviderError`.
- [ ] A network reject (fetch throws) **propagates** unchanged — `executeCompletion`'s `classifyTransportThrow` maps abort→`cancelled`, timeout→`timeout`, else `transport_error`. The transport does not swallow, wrap, or re-map throws.
- [ ] The `AbortSignal` is forwarded to `fetch` (cooperative cancel on a budget breach).
- [ ] **Rule-7:** the transport carries the resolved secret in `req.headers` (`x-api-key`) — it takes **no log sink**, and never logs/echoes the request, headers, or body. Nothing observable leaks the secret.
- [ ] All new unit tests pass; `/preflight` clean.

## RED test outline (Step 2)
`packages/providers/test/model/real-http-transport.test.ts` (inject a fake `fetch` capturing calls):
1. **`request_maps_verbatim_to_fetch`** — Asserts: fake `fetch` receives `req.url`, `{ method: "POST", headers, body }` unchanged. Why: the transport is a faithful pass-through (the wire adapter already shaped the request).
2. **`ok_response_maps_status_and_body`** — Asserts: fetch resolves `{ status:200, text:()=>"…" }` ⇒ `send` returns `{ status:200, body:"…" }`. Why: normalized `HttpTransportResponse`.
3. **`non_2xx_passes_through_no_throw`** — Asserts: fetch resolves `status:500` ⇒ `send` returns `{ status:500, body }` (no throw). Why: status classification is the executor's job (`providerErrorFromStatus`), not the transport's.
4. **`network_throw_propagates_unchanged`** — Asserts: fake fetch throws `E` ⇒ `send` rejects with **the same** `E` (use `rejects.toBe`/reason-capture, **not** assertions only inside `.catch` — L15 vacuous-green guard). Why: `classifyTransportThrow` owns mapping; the transport must not swallow.
5. **`abort_signal_forwarded_to_fetch`** — Asserts: the `signal` arg reaches `fetch`'s options; an already-aborted signal ⇒ fetch's `AbortError` propagates. Why: cooperative cancel on budget breach (5.4).
6. **`no_secret_in_observable_output`** — Asserts: `createRealModelHttpTransport` accepts **no** log-sink dep and makes no `console.*` call; a header carrying `x-api-key` is never emitted anywhere. Why: rule-7 / §16 — the secret rides headers and must never reach a sink.

## Things to flag at Step 2.5
1. **Built-in request timeout?** Default vote: **NO** — cancellation rides the injected `AbortSignal` (the broker's budget enforcer aborts; 5.4). A built-in `AbortSignal.timeout` here would double-govern runtime. Ground `fetch`/`AbortSignal` semantics on **Context7** (not the Anthropic wire — that's already in `claudeWireAdapter`).
2. **`fetch` injection default:** default to the Node 22 global `fetch` when the dep is omitted (prod path), require the fake in tests. Default vote: **inject with a global default.**
3. **Response body read:** always `await res.text()` (the wire adapters parse JSON from the string). Default vote: **text() always** — the transport does no interpretation.
