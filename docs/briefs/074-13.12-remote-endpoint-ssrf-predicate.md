# /tdd brief — remote_endpoint_ssrf_predicate (connector template, slice 1 — the vetted SSRF primitive)

## Feature
Add the single vetted **remote-endpoint SSRF/authority-isolation predicate** to `@sow/policy` — `isAllowedRemoteEndpoint(endpoint, allowedHosts): boolean` — the OUTBOUND counterpart to `isLoopbackEndpoint`. It is the safety core every connector HTTP transport (Asana this round; Granola/Drive/Calendar/… later) reuses to decide whether a bearer-token-carrying request may egress to a given URL. This is slice 1 of the round-2 connector arc (round 2 = **(1) this predicate → (2) the `createConnectorHttpTransport` template + the Asana instance**); Granola leads round 3 (it depends on the template, so it is not a *parallel* slice this round).

**Live once (Lesson 1/17):** the predicate COMPOSES the existing vetted primitives in `@sow/policy/processors.ts` (`extractHost`/`extractAuthority`/`isLoopbackHost`) — it MUST NOT re-parse URLs (re-parsing was the loopback-spoof hole `evil.com/@127.0.0.1` those primitives already close). It is the inverse safety posture from gbrain: gbrain's transport requires LOOPBACK; a connector requires an ALLOWLISTED REMOTE host and must REJECT loopback (SSRF-to-local defense).

**No hard line.** A pure predicate — no I/O, no network, no secrets. It only decides admissibility; nothing dispatches here.

## Acceptance criteria (what "done" means)
- [ ] `isAllowedRemoteEndpoint(endpoint: string, allowedHosts: readonly string[]): boolean` exists in `@sow/policy/processors.ts`, exported. Returns `true` IFF ALL hold: the endpoint parses to a host (`extractHost !== null`); the scheme is **https** (a bearer-token egress requires TLS — plaintext/`http:`/scheme-relative ⇒ false); the host is **NOT loopback** (`isLoopbackHost` false — reject SSRF-to-local); and the host is on `allowedHosts` (exact, lowercased match). 
- [ ] **Fail-closed on every reject axis:** unparseable / empty / non-string / `http:` / protocol-relative `//host` / `user@host` spoof / loopback (127.0.0.0/8, `localhost`, `::1`, the prefix/suffix tricks `127.0.0.1.evil.com`/`app.asana.com.evil.com`) / off-allowlist host ⇒ `false`. Never throws (pure).
- [ ] **Allowlist is exact-host, not substring:** `app.asana.com.evil.com` and `evilapp.asana.com` do NOT match an allowlist of `["app.asana.com"]` (the host is compared whole, lowercased — reuse `extractHost` so the parse is the vetted one).
- [ ] **Does NOT re-parse URLs** — composes `extractHost`/`extractAuthority`/`isLoopbackHost` (a targeted review check; re-mirroring the parse is the forbidden pattern).
- [ ] `/preflight` clean; repo-wide green. Pure addition — no behavior change to existing predicates.

6. **`fail_closed_unparseable`** — `""`, a non-string (cast), and garbage ⇒ false; never throws.

## Things to flag at Step 2.5
1. **Name.** `isAllowedRemoteEndpoint` vs `isAllowedEgressEndpoint` vs `isAllowedConnectorEndpoint`. Default vote: `isAllowedRemoteEndpoint` (mirrors `isLoopbackEndpoint`; "remote" is the inverse posture). Flag if a better name fits the egress vocabulary.
2. **Private-range block scope.** The exact-host allowlist is the PRIMARY control (a private IP isn't `app.asana.com`). Beyond loopback, do we ALSO block RFC-1918 / link-local / ULA ranges now (defense-in-depth), or defer? Default vote: **loopback-reject now** (reuse `isLoopbackHost`) + **document the broader private-range + DNS-rebinding IP-pinning as an arming-era residual** (arch_gap — the injected real `HttpTransport` at arming pins/validates the resolved IP; a hostname allowlist alone can't catch DNS-rebind). Flag if you'd rather add a `isPrivateHost` predicate this slice (it would also live once in `@sow/policy`).
3. **Allowlist entry shape — host vs host:port.** Default vote: exact HOST (via `extractHost`, port-stripped) — connectors hit vendor APIs on 443; a port-aware variant can use `extractAuthority` later if needed. Confirm.
4. **https-only.** Default vote: require `https:` strictly (a bearer token must never ride plaintext). Confirm no connector needs `http:` (none should).
