# /tdd brief — broker_real_transport_bind_dormant

## Feature
Bind the real `ModelProviderPort`/`AgentRuntimePort` transport into the broker's **run leg** — replacing `createStubProviderRunner` — behind a **default-OFF dormancy gate**, so with the transport unbound the behavior is **byte-identical** to today's stub. The provider key resolves through the `getSecret` facade; local routes use a loopback zero-egress transport. **This is a SAFE-BUILD slice: the real runner is built and wired but never flipped — no real endpoint, no real key, no real API spend. The flip is the owner crossing (out of scope here).**

## Acceptance criteria (what "done" means)
- [ ] A real `ProviderRunner` — signature `(route, job, budget, signal?) => Promise<GateResult<AgentResult>>` — that: selects the `ModelProviderPort` adapter for the resolved `route`'s `providerId`, builds a `ProviderRequest` from `(route, job, budget)`, calls `.complete()`, maps `ProviderOutput → AgentResult` (via `makeAgentResult`) and `ProviderError → GateResult err`, and **never throws** across the boundary (§16).
- [ ] The runner is selected in `assembleBackends` through a **default-OFF presence gate** — `config.providerTransport` **unset ⇒ `createStubProviderRunner(extraction)`** (byte-identical shipped default); **set ⇒ the real runner**. Mirrors `selectAdapterTransport(config.writeTransport)`.
- [ ] **Gate OFF ⇒ byte-identical stub behavior.** A factory-spy test pins that the **real runner factory is NEVER invoked** when `config.providerTransport` is unset (L16/L27 zero-invocation).
- [ ] The provider key resolves through the injected SecretsPort `getSecret(secretRef)` accessor (17.3 `createLockRoutingSecretsAccessor`, secret-ref per 17.4). A **missing/locked** key ⇒ the adapter's `auth_unavailable`/`retryable` degrade ⇒ the **never-reject `KeychainLockController`** holds the job **retryable** + mints a **keychain-locked HealthItem** — **no plaintext fallback, no terminal reject** (safety rule 7).
- [ ] A **local (loopback) route** uses the zero-egress transport and is validated by `assertLocalEndpointAllowed` (allowlist default `http://127.0.0.1:11434`); an off-allowlist local endpoint is rejected `invalid_request` (safety rule 5 — local zero-egress).
- [ ] The broker's fixed-order pipeline is **UNCHANGED** — the **egress veto still runs BEFORE the run leg** (pin the ordering; 18.9 builds on it).
- [ ] **No real network call in any test** — the real runner is exercised against a **fake `HttpTransport` / fake `ModelProviderPort`** returning a canned OpenAI-compatible body. No real endpoint, no real key, no real spend.
- [ ] All unit tests in `apps/worker/test/composition/` (new file) pass; `/preflight` clean.

## Wiring / entry point (Step 7.5)
The broker run leg is reached in production via `assembleBackends` → `createBroker({ run })` → `buildProofSpineActivities` (binds the activity factories over the backends bundle) → the meeting/source workflow activities (`broker.runJob` call path). The real runner lands **behind `config.providerTransport` in `assembleBackends`**; the default (unset) keeps the stub, so this slice is **reachable-but-inert** until the owner crossing flips the gate. The FLIP itself (binding a real endpoint + provisioned key) is NOT in this slice.

## RED test outline (Step 2) — `apps/worker/test/composition/provider-runner.test.ts`
1. **`real_runner_maps_provider_output_to_agent_result`** — fake port returns a `ProviderOutput`; runner returns `GateResult<AgentResult>` with the mapped `candidateOutput`/`usage`. Why: §19.5 run leg.
2. **`real_runner_maps_provider_error_to_gate_err_never_throws`** — fake port returns `Err(transport_error)`; runner returns a `GateResult` err and **never throws**. Why: §16 never-throw.
3. **`gate_off_uses_stub_byte_identical`** — `config.providerTransport` unset ⇒ the `createStubProviderRunner` path; the assembled `broker` produces byte-identical output to today. Why: L16/L27 byte-equivalence.
4. **`gate_off_never_invokes_real_factory`** — factory-spy asserts **zero invocation** of the real-runner factory when the knob is unset. Why: L16/L27 zero-invocation pin.
5. **`missing_secret_holds_retryable_no_plaintext`** — `getSecret` yields locked/missing ⇒ `auth_unavailable`/`retryable` ⇒ lock controller **holds retryable** + surfaces a keychain-locked HealthItem; assert **no plaintext** anywhere + **no terminal reject**. Why: safety rule 7 + L21/L29.
6. **`local_route_uses_loopback_allowlist`** — a local `providerId` route is validated by `assertLocalEndpointAllowed`; an off-allowlist local endpoint ⇒ `invalid_request`. Why: safety rule 5 (local zero-egress).
7. **`egress_veto_precedes_run_leg`** — with the real runner bound (fake transport), an egress-vetoed job is denied **before** the run leg fires (spy the runner is not called). Why: §7 pipeline order — 18.9 depends on it.
