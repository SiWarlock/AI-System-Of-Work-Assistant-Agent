## Feature

## Things to flag at Step 2.5
1. **Gate shape.** `keychainSecrets?: { execFile?: KeychainExec }` (inject the exec — testable, and the real `execFile` wrapper is the default) vs a bare boolean + a hard-wired real `execFile`. My default vote: **inject an optional `execFile` (defaulting to the real bounded `execFile` wrapper when the gate is present but no exec supplied)** — so tests provision a fake exec, and production provisions the real one, through ONE gate. The gate's PRESENCE (owner config) is the provisioning switch.
2. **`getSecret` facade error mapping.** The adapter's `SecretUnresolved` reasons are `missing|locked|denied|backend_error|invalid_ref`; the providers' `SecretUnavailable` is `missing|locked|denied`. My default vote: **map `missing→missing`, `locked→locked`, `denied→denied`, and `invalid_ref`/`backend_error`→`missing`** (fail-closed: an unresolvable ref reads as "no key" to the provider, which degrades it — never a throw or a false success). Flag if you'd rather widen `SecretUnavailable`.
3. **Degraded-controller integration scope — fold in now or Slice 4?** Routing a `locked` adapter result to the existing `KeychainLockController.onKeychainLocked` needs a resolution CALL-SITE (the serving-oracle signing-key resolve and/or the provider API-key resolve — both currently DORMANT). My default vote: **if the call-site is clean + dormant-safe, fold in a minimal `locked→onKeychainLocked` routing at the provider/secrets-resolution boundary; if it ripples into the dormant provider dispatch, DEFER to a Slice-4 follow-up with the wiring point named** (the controller exists; the routing is the only gap). Decide at Step 2.5 based on the real call-site — don't force it into the dormant provider path.
4. **Real `execFile` wrapper hardening.** Bounded `timeout` + `maxBuffer` + `windowsHide`-n/a (macOS) + no shell. My default vote: **a thin `node:child_process execFile` wrapper with a short timeout (e.g. 5s) + a small maxBuffer (the key is tiny), returning `{code, stdout: Buffer, stderr}`** — built only on the provisioned path. Reviewer confirms no shell + the Slice-2 args-array contract.
