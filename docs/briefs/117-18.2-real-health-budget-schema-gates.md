# /tdd brief — broker_real_health_budget_schema_gates

## Feature
Replace the broker's three stub gates in `assembleBackends` with the **real** HEALTH / BUDGET / SCHEMA gates (which already exist in `packages/providers/src/broker/`), and introduce the **NEW pluggable `BudgetLedgerPort` seam** for single-run/in-boot budget accounting (the durable cross-run ledger plugs in *backward* at 19.11 — no forward dependency). The gates are **pure policing** — they only DENY (schema-invalid → reject-before-side-effect; over-budget → cancelled; unhealthy → fail-closed); **no spend, no egress, no external write ⇒ NO hard line crossed.**

## RED test outline (Step 2) — `apps/worker/test/composition/broker-gates.test.ts`
1. **`schema_gate_rejects_invalid_candidate_no_side_effect`** — fake run yields a candidate that fails the ajv/output-schema; SCHEMA gate → `schema_rejected` (branch `rejected`, not retryable); no emit, no downstream side effect. Why: safety rule 2 / REQ-S-006.
2. **`schema_gate_accepts_valid_candidate`** — a schema-valid candidate passes → `BrokerCandidate` emitted. Why: §7 candidate-vs-schema accept path.
3. **`budget_pre_failcloses_when_no_bounded_cap`** — a job with no runtime cap + config that can't derive one → `pre` denies (fail-closed, COST-2). Why: budget-enforcer COST-2 hard floor.
4. **`budget_post_denies_over_budget_run`** — usage exceeds the enforced cap → `post` → `cancelled_budget`, no partial side effect. Why: COST-1 / REQ-S-007.
5. **`budget_ledger_seam_single_run_accounts`** — the `BudgetLedgerPort` records/consults this run's spend; default/unset impl = single-run behavior (no cross-run enforcement); assert the seam is invoked. Why: 18.2 seam for 19.11 backward-plug.
6. **`health_gate_denies_unhealthy_provider`** — fake health source returns `unreachable`/`keychain_locked` → HEALTH gate fail-closed deny (health-classed, retryable), zero run. Why: §7 provider health.
7. **`health_gate_proceeds_healthy`** — fake healthy source → proceed. Why: accept path.
8. **`assembleBackends_default_meeting_close_stays_green`** — `assembleBackends({})` drives a local employer_work meeting.close job through `broker.runJob` with the REAL gates; the valid stub extraction passes health+budget+schema and yields the accepted candidate (proof-spine unbroken). Why: accept-path byte-equivalence for a valid job.
