# /tdd brief — subscription_spawn_env_minimal_allowlist

## Feature
Make the subscription-extraction rule-5 (egress-veto / no-surprise-spend) invariant **complete-by-construction** by spawning the Agent SDK `query()` on the armed path with a **minimal allowlisted child env** instead of relying on a denylist of shadow env vars. Grounded fact (sdk.d.ts:1391-1409, SDK 0.3.201): `query()`'s `env` option **REPLACES the subprocess environment ENTIRELY** (not merged with process.env; omitted ⇒ inherits). Today the call site passes no `env` → the child inherits the worker's full process.env = every known/unknown shadow var, INCLUDING the `CLAUDE_ENV_FILE` dotenv-bypass that no process.env scan can catch. Passing an explicit minimal allowlist (`PATH`/`HOME` + the OS operational set; **no credential vars** — the subscription login is ambient via `~/.claude` + macOS Keychain, not env) means the child claude sees ONLY the allowlist → NO shadow var can reach it, regardless of what's set in the parent or injected via a dotenv. This is the **completeness guarantee** rule-5 requires (a denylist is structurally unwinnable — 4 re-grounds each found more, and `CLAUDE_ENV_FILE` defeats the scan model outright). It also **retires the "re-verify the shadow set vs the bundled claude-code version at every bump" HARD precondition** (drift-immune by construction) and makes the Option-B premise ("worker runs `ANTHROPIC_API_KEY` UNSET") TRUE-by-construction for the child. The 18.38 69-key denylist guard STAYS as defense-in-depth (a loud early degrade+health if the operator's process.env looks misconfigured); this env-scrub is the suspenders. Armed-path only; byte-equivalent default (omit `env` ⇒ inherit, unchanged). Owner-authorized §ARM-18 CHECKPOINT-1 GATE-1 completeness fix.

- [ ] **Worker computes a minimal child-env allowlist** from `process.env` on the armed path — a pure builder that copies ONLY the allowlisted keys (`PATH`, `HOME`, + the OS operational set finalized at Q1) and their values, dropping everything else. STRUCTURALLY excludes every `ANTHROPIC_*`/`CLAUDE_*`/`AWS_*`/`GOOGLE_*`/proxy/cert var + `CLAUDE_ENV_FILE` + `CLAUDE_CONFIG_DIR` (a pattern/prefix denial in addition to the positive allowlist, so a new shadow var can never slip into the child).
- [ ] **Providers forwards it:** the `query()` call site accepts an optional `childEnv?: Record<string,string>` and passes it as `options.env`; **byte-equivalent when omitted** (no `env` ⇒ inherit, unchanged shipped default). Layer rule respected (no providers→worker import — the worker computes, providers forwards).
- [ ] **Complete-by-construction pin:** a test sets shadow vars (`ANTHROPIC_API_KEY`, `ANTHROPIC_BASE_URL`, `CLAUDE_ENV_FILE`, a `CLAUDE_CODE_USE_*` switch) in the source `process.env`, builds the allowlist, and asserts the result contains ONLY the allowlisted keys and NONE of the shadows — the child can't see them even though the parent has them.
- [ ] **Armed-path only / byte-equivalent default:** the allowlist is computed + passed ONLY on the armed path (`isProviderTransportArmed`/the arm signal); the unarmed default omits `childEnv` ⇒ `query()` inherits ⇒ byte-identical to today. Factory-spy/zero-invocation OFF pin (L23).
- [ ] **Under-allowlisting fails CLOSED:** if the allowlist omits something `query()` needs, the run fails with an auth/spawn error (never a silent wrong-credential run). Documented + validated by the dry-run/real run.
- [ ] **⛔ SDK control-var validation gate:** since `env` REPLACES entirely, confirm empirically (the `$0 SOW_L64_DRYRUN` + the gate-6 real run) that the SDK still injects its OWN stdio/IPC control vars (`CLAUDE_CODE_SDK_*` handshake) on top of the provided env. If it does NOT, the allowlist MUST pass through the SDK control vars (the worker must not strip them). This is the load-bearing runtime validation — the brief is not "done" until a real armed run spawns + completes.
- [ ] **18.38 denylist retained as defense-in-depth:** the armed-path shadow-env guard (69) + the settings guard (18.36/18.39) STAY active — a misconfigured operator process.env still degrades loudly with a health signal BEFORE the run (belt); this slice is the completeness suspenders. Do NOT remove the denylist.
- [ ] `/preflight` clean. No egress/spend/key read in the builder (it copies env VALUES for allowlisted operational keys only — never logs them, rule 7-adjacent: PATH/HOME are not secrets, but keep the builder value-quiet).
