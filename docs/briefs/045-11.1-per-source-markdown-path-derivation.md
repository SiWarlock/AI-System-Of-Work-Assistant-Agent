# /tdd brief — per_source_markdown_path_derivation (Option A — port thread + per-source path)

> **Revised 2026-07-13** after impl14's Step-2.5 broken-premise finding: a derivation-ONLY change can't deliver multi-file persistence, because the per-file identity never reaches the derivation. **Orchestrator ruling (lead-endorsed, build-time design — no owner gate): Option A.** The old brief's driver/port scope-guard fenced off the exact code that must change — it was wrong for this slice; scope is **expanded** to the port + driver + sandbox.

## Feature
Make MANY dropped files persist per workspace (not one) by threading the **per-file source identity** (`context.source`) into the ingestion output derivation, and deriving BOTH a **traversal-safe per-source Markdown path** AND the **planId** from it — so distinct files become distinct durable notes, while a same-file re-drop stays idempotent (replays via the durable 2a store). The build stage *should* know which source it's building; passing only `(validated, ws)` is the actual bug. All-local; the write stays the KnowledgeWriter sole-writer path — NO hard line.

## Acceptance criteria (what "done" means)
- [ ] A DEDICATED `SourceBuildOutputsPort.build(validated, ws, source: SourceNoteIdentity)` carries the per-file source identity; the `runSourceIngestion` driver passes the in-scope `context.source` (projected); the SOURCE sandbox proxy passes it through. The shared `BuildOutputsPort` is byte-unchanged (see Step-2.5 #1).
- [ ] **Distinct files → distinct notes:** two different dropped files into the same workspace derive DIFFERENT `creates[].path` AND DIFFERENT planId → two durable notes, no collision/replay-away.
- [ ] **Traversal-safe path by construction (load-bearing safety):** the per-source path segment is derived so it can NEVER contain `..`, an absolute prefix, an escaping separator, a leading `/`/`~`, a NUL, or a control char — regardless of what the raw source identity contains. The path stays under the routing-bound `sources/<ws>/`.
- [ ] **WS-segment guard (load-bearing — impl14 Step-2.5 catch):** the helper guards the `ws` path segment too, NOT just the hash. `WorkspaceId` is a bare branded string (`ids.ts:35` rejects only empty/whitespace — NO charset validation), and it's interpolated into `sources/${ws}/`; an unsafe `ws` (`.`, `..`, `a/b`, a separator, NUL, control char) would break cross-workspace `sources/<ws>/` confinement (the vault-root guard only backstops WHOLE-vault escape, not cross-workspace-within-vault). The helper REJECTS an unsafe `ws` fail-closed (→ `build_failed`), never emits an escaping path. (`ws` is routing-bound/trusted-config today, but the no-schema-validation + injection-surface mandate means harden it here.)
- [ ] **Same-file re-drop is idempotent (no duplicate):** re-dropping the SAME file with the SAME content yields the SAME planId → the durable 2a store replays → no duplicate note, no second write. (Idempotency semantics per Step-2.5 #2.)
- [ ] **WS-8 preserved:** path under `sources/<the routing-bound ws>/`, stamped from `ws` (never a source-content value); a source can't write into another workspace's tree. The vault-root guard remains the whole-vault backstop.
- [ ] **Invariants preserved:** default-OFF/owner-opt-in (behind the same `gateAutoIngest`); KnowledgeWriter sole-writer (unchanged); ING-7 read-only ingestion; real `enforceHumanOwnership`/`scanForSecrets` defaults; the durable-store fail-closed (unchanged, honored at `createCommitActivity`).
- [ ] **Shared port + other consumers unaffected (zero regression by construction):** the shared `BuildOutputsPort` has NO diff → meeting-closeout, hermes, crossCalendar, projectSync are untouched (their tests stay green; verified by the no-shared-port-diff, not just behavior).
- [ ] All new unit tests pass; repo-wide `pnpm -w turbo run typecheck test` green; the `SOW_TEMPORAL=1` ingestion suite passes with a NEW two-distinct-files → two-distinct-notes case. **MANDATORY Step-8 review = security-reviewer + code-quality-reviewer** — path-construction is an injection surface; review it HARD for traversal/escape.
- [ ] No hard line crossed — local KnowledgeWriter Markdown write only; propose + GBrain write-through untouched/OFF; no external I/O.
