# /tdd brief — real_extraction_content_resolver

## Feature
Implement the real `ExtractionContentResolver` (the seam 18.20 left unbound): deref an extraction `AgentJob`'s input to the parked `sourceId`, read the body via the existing `ParkedSourceReader`, and return it as the inline text the subscription runner sends as `userPrompt` — DORMANT (no production caller until the owner ENABLE flip binds it).

## Acceptance criteria (what "done" means)
- [ ] `createRealExtractionContentResolver(deps): ExtractionContentResolver` where `deps` injects the `ParkedSourceReader` (a fake in tests). `resolve(job)`: derives the `sourceId` from the job's input (`contextRefs` deref — see Step-2.5 #1), reads via `ParkedSourceReader.read(sourceId)`, and returns `ok(envelope.body)` when a non-empty body is present.
- [ ] Fail-closed typed faults (code-only, rule 7 — never the content): unresolvable/absent `sourceId` from the job ⇒ `err({code:"source_ref_unresolved"})`; `ParkedSourceReader.read` err (`source_unavailable`) ⇒ `err({code:"source_unavailable"})`; a present-but-absent/empty `body` ⇒ `err({code:"no_body"})`. NEVER returns an empty string as ok (the runner would send an empty prompt).
- [ ] Meeting-vs-source discrimination on `job.capability` if the deref differs between the two (Step-2.5 #2) — or a single uniform deref if both park the same way; pin whichever holds.
- [ ] Never throws (§16 totality) — a reader throw / a malformed job folds to a typed `err` (the runner's outer catch is the backstop, but the resolver is total on its own).
- [ ] The resolver reads ONLY the parked body — it does not re-inline `contextRefs` markers, does not read the vault directly, does not touch the network.
- [ ] All unit tests in `apps/worker/test/composition/real-extraction-content-resolver.test.ts` pass (fake `ParkedSourceReader`).
- [ ] `/preflight` clean. No cross-doc invariant change (internal worker port; `SourceEnvelope`/`AgentJob` unchanged).
