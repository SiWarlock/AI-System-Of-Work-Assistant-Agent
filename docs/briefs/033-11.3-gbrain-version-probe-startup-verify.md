# /tdd brief — gbrain_version_probe_startup_verify (Phase 11, slice 11.3-a)

## Feature
Make the GBrain version-pin startup check REAL over a LOCAL probe: a pure `verifyGbrainStartup` composition that probes the running gbrain version and runs the already-built pure `checkVersionPin` (serving-vs-degrade + a System Health item), an injected `GbrainVersionProbe` port, a pure `parseGbrainDoctorJson` parser (untrusted subprocess output → `RunningGbrainVersion | undefined`, fail-closed), and a thin real `gbrain doctor --json` / `gbrain --version` adapter. Same shape as the install-doctor collectors (Lesson 19): pure core + injected port + a thin LOCAL never-throwing adapter. LOCAL-ONLY. The pin RE-capture (writing `config/gbrain.pin`), the boot-degrade WIRING, and the `writeThroughEnabled` flip stay HITL/deferred.

- **Task ID:** 11.3 (GBrain version-pin startup verify — the make-REAL probe; the deterministic decision core `checkVersionPin` is already built)
- **Architecture sections it implements:** `ARCHITECTURE.md §13` (Deployment / Install / GBrain startup verify), with `§12` (write-through/serving eligibility layered on a matched pin) supporting. (§13, §12 ∈ Phase 11 Spec anchors — no widen.)
- **Related context:** the pure decision core `checkVersionPin` (`packages/knowledge/src/gbrain/version-pin.ts`) is ALREADY BUILT — it takes `(pin: GbrainPin, running: RunningGbrainVersion | undefined, ctx)` and returns `Result<VersionPinServing, VersionPinDegraded>` (fail-closed on sha-mismatch / index-schema-mismatch / PENDING-sentinel / gbrain-unavailable; opens a distinct `HealthItem`; never throws). NOTHING PRODUCES the `RunningGbrainVersion` yet — this slice adds the real LOCAL probe + the composition. `decideWriteThroughEnablement` (the separate 6-leg FLIP-precondition gate) is also built + distinct — do NOT touch it. **HARD LINE:** LOCAL-ONLY — `gbrain doctor --json` / `gbrain --version` is a LOCAL CLI read (no network); the probe reads, never writes. The pin RE-capture (which WRITES `config/gbrain.pin`) is the HITL step and is OUT of scope.

## Acceptance criteria (what "done" means)
- [ ] `parseGbrainDoctorJson(stdout: string): RunningGbrainVersion | undefined` — a PURE parser: extracts the running commit `sha` (+ optional index `schema_version` → `indexSchemaVersion`) from `gbrain doctor --json` output; returns `undefined` (never throws) on absent/malformed/non-JSON/missing-sha — a fail-closed "unavailable" that `checkVersionPin` degrades.
- [ ] `verifyGbrainStartup(deps): Promise<Result<VersionPinServing, VersionPinDegraded>>` — a composition that awaits the injected `GbrainVersionProbe`, then returns `checkVersionPin(pin, probed, ctx)`. NEVER throws (a thrown probe folds to `undefined` → `gbrain_unavailable` degrade). Does NOT re-implement the decision — it delegates to the built `checkVersionPin`.
- [ ] `GbrainVersionProbe = () => Promise<RunningGbrainVersion | undefined>` injected port; the pure composition + parser are unit-testable with fakes.
- [ ] A real `createGbrainVersionProbe` adapter execs `gbrain doctor --json` (fixed argv ARRAY, `shell:false`, timeout+cap, errno-redacted) → `parseGbrainDoctorJson`; a nonzero exit / ENOENT / timeout / malformed → `undefined` (fail-closed, never-throw). LOCAL-ONLY.
- [ ] Fail-closed end-to-end: an unavailable/malformed/mismatched gbrain → the `read_only_index_only` degrade branch + the distinct `HealthItem`, never a fabricated `serving`.
- [ ] Fast-unit (injected-fake probe: serving / each degrade reason; parser: valid/malformed/missing-sha/missing-schema) + an OPTIONAL `SOW_GBRAIN_REAL=1`-gated real-adapter test (a dev machine's gbrain — tolerating a PENDING-sentinel degrade as EXPECTED, asserted well-formed not serving). Default suite never shells out. Repo-wide `turbo typecheck test` green; `/preflight` clean.
