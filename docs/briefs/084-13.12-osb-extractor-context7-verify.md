### Part 1 — Context7 back-verify the 4 candidate shapes (per-extractor verdict)
- [ ] **web** (`web-source.ts`, `WebPage`: url/title/byline?/publishedAt?/text) — verify vs a Readability/readability extraction output shape. Verdict CONFORMANT or CORRECTED.
- [ ] **podcast** (`podcast-source.ts`, `PodcastEpisode`: episodeId=guid/title/showTitle?/publishedAt?/audioUrl?/transcript) — verify vs **RSS 2.0** `<item>`/`<enclosure>`/`<guid>` (+ the transcript is a downstream transcription concern, not an RSS field — keep that boundary). Verdict.
- [ ] **youtube** (`youtube-source.ts`, `YouTubeExtract`: videoId/watchUrl/title/channel/transcript/publishedAt?) — verify vs `youtube-transcript-api` (transcript) + YouTube video metadata. Verdict.
- [ ] **file** (`file-source.ts`, `ExtractedFile`: path/filename/mime/text) — verify vs the pinned OSB extractor's `--emit-json` contract. Verdict.
- [ ] **Each shape gets a Context7 (or authoritative-spec, if no clean Context7 lib exists — e.g. RSS 2.0 spec / Mozilla Readability) citation comment + an `arch_gap` marker** (these are documented candidates). Any **drift is fixed as a TESTED change** (a RED test pinning the corrected field → fix), Asana-style — NEVER a silent shape edit. A CONFORMANT shape needs only the citation comment (note "verified CONFORMANT, no behavior change").

### Part 2 — youtube ING-7 header fix (doc-only)
- [ ] Add the **ING-7 read-only bullet** to `youtube-source.ts`'s header comment, matching the wording its siblings carry (the `file-source.ts` header is the reference: "consumes UNTRUSTED … content read-only/emit-only; no mutating path"). Doc-only, no behavior change.

### Part 3 — ING-7 admission verification (LOAD-BEARING — a gap is a Finding)
- [ ] **VERIFY (trace, don't assume)** that ING-7 read-only is genuinely enforced at **JOB ADMISSION** (`packages/policy/admission.ts` `untrustedJob`/`MUTATING_POLICY`) for **ALL FOUR** untrusted source types (web/podcast/youtube/file content): a consuming agent job over that untrusted content is admitted **READ-ONLY**, and a job declaring a **mutating tool is REJECTED**.
- [ ] **PIN it with a test** — for each of the 4 untrusted source types (or the untrusted-content job class that covers them), assert admission **REJECTS a mutating-tool job** AND **ADMITS a read-only job**. Make the coverage explicit (name the 4 source types in the test).
- [ ] **If ANY of the 4 slips the admission gate** (untrusted source content can be consumed by a job that was NOT admitted read-only) → **STOP + escalate a Finding** to the orchestrator (→ lead) BEFORE proceeding. Do NOT paper over a gap with a local fix.

- [ ] `/preflight` repo-wide green. NO hard line; NO new file; NO count-pin bump (confirm anti-corruption stays 19/19).
