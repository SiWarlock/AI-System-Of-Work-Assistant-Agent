# /tdd brief — reconcile_scheduler

## Feature
Build the **pure worker-side reconcile SCHEDULER** (piece **E** of the reconcile-TRIGGER arc, task 13.10) — the trigger-origin machinery that accumulates pending reconcile triggers, **burst-collapses** them via the pure `collapseToMaxRevision` (LIFE-2 — a burst of vault changes at rising revisions fires **ONE** reconcile at the max revision, never one-per-change), and dispatches exactly one `runReconcileForWorkspace` (piece D) per flush, then routes the driver's outcome through a **redacted** log (safety rule 7). **The pre-delegated workflow-weight decision RESOLVES here → the lighter worker-side scheduled pass** (not a Temporal workflow): the reconcile is idempotent read+record (not an external side effect), so Temporal's exactly-once/durability is over-engineering; `collapseToMaxRevision` IS the LIFE-2 catch-up-collapse; a crash → degrade → next-trigger-recovers is fail-safe. Ships **DORMANT + reachability-waivered**: it is **trigger-SOURCE-agnostic** (the real fs-watch / schedule / post-`sourceIngestion`-commit source, the flush timing, the health surface, and the arming all bind at **piece F**), and the driver + its never-reject collaborators are injected (fakes in tests).

## Acceptance criteria (what "done" means)
- [ ] An exported factory (default name **`createReconcileScheduler`**) returning a scheduler with `enqueue(workspaceId: string, trigger: PendingTrigger): void` and `flush(workspaceId: string): Promise<void>`. Deps injected: `runReconcile: (ws, origin) => Promise<ReconcileDriverOutcome>` (piece D bound), `log: (safe: <redacted summary>) => void`. Every dep a fake in tests.
- [ ] **LIFE-2 burst-collapse (THE core):** N `enqueue`s for one workspace at rising `seq` then ONE `flush` ⇒ `runReconcile` called **exactly once** with the **max-`seq`** trigger's `origin` (via `collapseToMaxRevision`) — never once-per-enqueue. Asserts: `runReconcile` call-count === 1; the origin is the winner's.
- [ ] **Empty flush is a no-op:** `flush` with no accumulated triggers (or after a prior flush cleared them) ⇒ `collapseToMaxRevision([]) → undefined` ⇒ `runReconcile` NOT called. Asserts: 0 dispatches.
- [ ] **Queue clears on flush:** after a `flush` dispatches, a second `flush` (no new enqueues) is a no-op (the burst was consumed — not re-run). Asserts: exactly one dispatch across two flushes.
- [ ] **Per-workspace isolation:** triggers for workspace A and workspace B accumulate + flush independently — a `flush(A)` dispatches only A's collapsed trigger, leaves B's queue intact. Asserts: `runReconcile` called with A's ws only; B still pending.
- [ ] **Redacted outcome routing (constraint b, safety rule 7):** every `ReconcileDriverOutcome` kind is logged through `log` as a **redacted** summary — `pass_faulted.cause` + `skipped_derive_error.error` are stripped to a code/tag, NEVER the raw object. Asserts: the logged payload contains no raw cause/error object (a marker secret in a fake cause does not appear).
- [ ] **Never throws:** `flush` never throws — the driver never throws (piece D), and a burst-collapse over a fake queue is total. Asserts: `flush` resolves for every path.
- [ ] Ships **DORMANT + reachability-waivered** — no production caller (piece F binds the real trigger source + flush timing + health surface + arming), no boot, no Temporal, no real fs/timer; the driver + collaborators injected. `/wired` reports test-only reachability; the waiver names piece F.
- [ ] All unit tests pass; repo-wide `pnpm -w turbo run typecheck test` green; `/preflight` clean.
- [ ] **MANDATORY Step-8 dual review = security-reviewer + code-quality-reviewer** — the scheduler drives the trust-gate-feeding reconcile + owns the outcome redaction; a burst that multiplies passes (or a leaked raw cause) is a defect. Safety-critical; own commit.

## RED test outline (Step 2)
Tests in `apps/worker/test/composition/reconcileScheduler.test.ts`:
1. **`burst_collapses_to_one_dispatch`** — 3 enqueues (seq 1,2,3) for ws → 1 flush ⇒ `runReconcile` called ONCE with seq-3's origin. Why: §12 LIFE-2 burst-collapse.
2. **`empty_flush_no_dispatch`** — flush with no triggers ⇒ 0 dispatches (`collapseToMaxRevision([]) → undefined`). Why: §12 nothing to reconcile.
3. **`flush_clears_queue`** — enqueue → flush (1 dispatch) → flush again ⇒ still 1 dispatch total. Why: the burst is consumed, not re-run.
4. **`per_workspace_isolation`** — enqueue for A + B; `flush(A)` ⇒ dispatch A only, B's queue intact (a `flush(B)` then dispatches B). Why: WS isolation; one ws's burst doesn't fire another's.
5. **`outcome_redacted_before_log`** — a `pass_faulted` outcome whose `cause` embeds a marker secret ⇒ the `log` payload carries a code/tag, NOT the raw cause (marker absent). Also a `skipped_derive_error` ⇒ error stripped to its code. Why: safety rule 7 (constraint b).
6. **`flush_never_throws`** — every outcome kind (reconciled / skipped_absent / skipped_derive_error / pass_faulted) ⇒ `flush` resolves, never throws. Why: the scheduler is fail-safe (the driver never throws).
