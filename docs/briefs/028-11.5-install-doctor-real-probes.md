# /tdd brief — install_doctor_real_prerequisite_probes (Phase 11, slice 11.5-a)

## Feature
Make the install-doctor REAL: build the local probe COLLECTOR adapter that fills the `ProbeSnapshot` the already-built pure `runDoctor(snapshot)` consumes — starting with the prerequisite-tooling probes (node/pnpm, Temporal CLI, gbrain CLI, git remotes, loopback ports). Each collector runs a REAL LOCAL check (a local subprocess / a loopback bind) over an INJECTED exec/net port, never throws, and fail-closes to an absent/unknown probe (which `runDoctor` already maps to a `finding`). LOCAL-ONLY — no network/external/cloud.

## Use case + traceability
- **Task ID:** 11.5 (install doctor — the REAL prerequisite-probe collectors; the deterministic `runDoctor` check-engine core is built, `fc1960d`)
- **Architecture sections it implements:** `ARCHITECTURE.md §13` (Deployment / Install / doctor & repair — typed prerequisite checks), with `§16` (System Health / doctor report) supporting. (§13 is in Phase 11's Spec anchors — no scope widen.)
- **Related context:** `apps/worker/src/install/doctor.ts` `runDoctor(snapshot: ProbeSnapshot): DoctorReport` is PURE + no-I/O (Lesson 8 — "real collectors = a separate deferred adapter"); each `diagnose*` fail-closes to `finding` on an absent/unknown/malformed probe. `ProbeSnapshot` (`apps/worker/src/install/probe-snapshot.ts`) has 10 optional probe fields. This slice fills FIVE (`nodePnpm`, `temporalStartable`, `gbrainStartable`, `gitRemotes`, `loopbackPorts`) with real collectors; the macOS-security probes (`filevault`/`keychain`) + the one-writer POSTURE probes (`vaultAcl`/`gbrainMount`/`strayGbrainProcess`, REQ-S-NEW-008 / safety rule 1) are SEPARATE follow-up slices (11.5-b/c — isolated for focused safety review). **HARD LINE:** these probes are LOCAL-ONLY (local `execFile` of `--version`/`which`, a loopback `net` bind, `git remote -v` reading local config — NO fetch). Any probe that would make a real EXTERNAL/network/cloud call is the hard line → ESCALATE before wiring; none here does.

## Acceptance criteria (what "done" means)
- [ ] A `collectPrerequisiteProbes(deps)` (pure over an injected `RunCommand` + `ProbeLoopbackBind` port) returns a partial `ProbeSnapshot` populated with `nodePnpm`, `temporalStartable`, `gbrainStartable`, `gitRemotes`, `loopbackPorts` — each mapped from the real check result.
- [ ] **Never-throws (§16):** every collector runs its injected check under a total guard — a missing binary (ENOENT), a non-zero exit, a timeout, or a malformed output becomes the probe's "absent/unknown" shape (which `runDoctor` fail-closes to `finding`), NEVER an uncaught throw.
- [ ] A real `createLocalCommandRunner()` adapter wraps `node:child_process.execFile` with: an argv array (NO shell string — no injection surface), a bounded timeout, a bounded output cap, and a fixed argv per probe (never caller-interpolated). A real `createLoopbackBindProbe()` adapter attempts a `net` bind on `127.0.0.1:<port>` (loopback only) and releases it.
- [ ] The doctor entry composes `collectPrerequisiteProbes(realAdapters)` → `runDoctor(snapshot)` → `DoctorReport` — the real collectors feed the real (built) engine.
- [ ] Fast-unit tests inject FAKE exec/bind results (present/absent/nonzero/timeout/malformed) and assert each probe shape + the never-throw guarantee — no real subprocess in the default suite. An OPTIONAL gated test (`SOW_DOCTOR_REAL=1` or similar) exercises the REAL adapter against the actual local environment.
- [ ] Repo-wide `turbo typecheck test` green (no real subprocess in the default suite); `/preflight` clean.

## Things to flag at Step 2.5
1. **Exec adapter safety.** Default vote: **`execFile` with a fixed argv ARRAY (never a shell string / never caller-interpolated), a bounded timeout + output cap, and an allowlisted binary name per probe.** No command-injection surface — the argv is a per-probe constant. Confirm the timeout/cap values.
2. **Probe fail-closed shape.** Default vote: **on any fault (ENOENT / nonzero / timeout / malformed), emit the probe's "unknown/absent" variant (NOT a fabricated "ok")** — the pure `diagnose*` already fail-closes that to `finding` (assume-worst, Lesson 8). Never guess a version/presence.
3. **Slice size / split.** Default vote: **all 5 prerequisite probes in this slice** (they share the injected-exec pattern). If it's too big, split loopback_ports into a follow-up — flag at Step 2.5.
4. **Gated real-adapter test.** Default vote: **an opt-in gated test** (the default suite must not shell out — CI may lack `temporal`/`gbrain`); the real adapter is otherwise exercised via the gated path.
