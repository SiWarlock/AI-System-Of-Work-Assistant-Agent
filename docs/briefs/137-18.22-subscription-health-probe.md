## Things to flag at Step 2.5
1. **What is a meaningful, cheap subscription reachability check?** Options: (a) local `claude` login presence (a file/config check — cheap, no SDK); (b) a lightweight SDK reachability ping (heavier, real I/O — arming-era). Default vote: **(a) login-presence via an injected check for THIS dormant build** (the real deep reachability check can bind at the flip); keep the probe's SEAM able to accept a richer check later. Sync vs async: match whatever the injected check needs (a file check is sync-ish; keep the verdict shape stable).
2. **Verdict shape — align to `HealthGateSources`.** The worker wrap (18.23) needs to produce `health: ()=>{state:"healthy"|"unreachable"|...}` + `availability: ()=>{modelPresent, conformanceStatus}`. Default vote: **return a minimal `{reachable, reason?}` verdict** and let the worker wrap map it to both `health` + `availability` fields; don't couple the providers probe to the worker's `HealthGateSources` shape (layer cleanliness). Confirm the exact `health.state` / `availability` enum members the wrap needs (from `provider-runner.ts` `UNAVAILABLE_HEALTH_SOURCES` / `DEFAULT_HEALTH_SOURCES`).
