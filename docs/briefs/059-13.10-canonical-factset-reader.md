# /tdd brief — canonical_factset_reader

## Feature
Build the **canonical-snapshot → fact-set composition helper** (piece **C** of the reconcile-TRIGGER arc, task 13.10) — a thin worker-side composition that reads the committed vault at head via the existing (injected) **`CommittedVaultReader`**, runs the existing pure **`deriveCanonicalFacts`** over the resulting `CanonicalVaultSnapshot`, and returns the **`CanonicalFactSet`** piece A's `runReconcilePass` consumes as `req.canonicalSet`. Fail-closed: it **distinguishes** a benign **absence** (no/empty vault) from a real **derive error** (a structurally-broken vault) so piece D can route the derive error to health while treating an absence as a benign skip. Ships **DORMANT + reachability-waivered**: no production caller (piece D), LOCAL-fs only (NOT a gbrain line), the reader is injected (fake/fixture map in tests).

## Acceptance criteria (what "done" means)
- [ ] An exported helper (default name **`buildCanonicalFactSet`**) `buildCanonicalFactSet(reader: CommittedVaultReader, workspaceId: string): Promise<CanonicalSnapshotOutcome>` where `CanonicalSnapshotOutcome` is a discriminated union (Step-2.5 #1): `{ kind: "derived"; set: CanonicalFactSet } | { kind: "absent" } | { kind: "derive_error"; error: DeriveError }`. The reader is injected (fake in tests).
- [ ] **Happy path:** a readable vault snapshot → `deriveCanonicalFacts` `ok` → `{ kind: "derived", set }`. Asserts: the returned `set` is the deriver's output verbatim (workspaceId/revisionId/facts).
- [ ] **Absent ⇒ benign skip:** the reader returns `undefined` (unmapped/empty vault/fault) → `{ kind: "absent" }` (piece D does not reconcile — no canonical reference). Asserts: `absent` on reader-`undefined`.
- [ ] **Derive error ⇒ typed, distinguished:** the deriver returns `err(DeriveError)` (e.g. `invalid_page_path` / `duplicate_fact_identity`) → `{ kind: "derive_error", error }` — NOT collapsed into `absent` (a broken vault is a real defect piece D routes to health, distinct from a benign empty vault). Asserts: `derive_error` carrying the typed error.
- [ ] **Never throws (fail-closed):** a reader that (defensively) rejects is treated as `absent` — never a throw across the boundary (Step-2.5 #4). Asserts: a rejecting reader ⇒ `absent`, captured + asserted unconditionally (Lesson 15).

- [ ] **MANDATORY Step-8 dual review = security-reviewer + code-quality-reviewer** — this produces the canonical reference the reconcile pass compares the DB against; a mis-built or wrongly-absent canonical set corrupts the parity verdict. Safety-critical; own commit.

## Things to flag at Step 2.5
1. **Outcome shape — a 3-way discriminated union vs `CanonicalFactSet | undefined`.** **Default vote: the 3-way union** (`derived` / `absent` / `derive_error`) so piece D can distinguish a benign empty/absent vault (skip) from a structurally-broken vault (route the `DeriveError` to health) — collapsing both to `undefined` would drop the defect signal. Flag if you'd rather collapse (simpler, but loses the derive-error distinction).
2. **[NAME-THE-GAP] The empty-vault `db_only` detection gap.** `createCommittedVaultReader` collapses an EMPTY vault (zero committed `.md`) to `undefined` — indistinguishable from "no vault mapped" — and surfaces no head revision. So piece C's `absent` can't build an empty `CanonicalFactSet`, which means an **empty vault + a populated DB projection** (whose facts would all be `db_only` HARD parity defects, safety rule 1) would be **skipped, not caught**, by piece D. **Default vote:** piece C returns `absent` on reader-`undefined` (the smallest honest slice); the empty-vault-`db_only`-detection gap is a **NAMED follow-up** (it needs the reader to surface an empty snapshot WITH a head revision — a `CommittedVaultReader` change out of piece C's scope, and only reachable once real corpora exist at the owner-gated arming). Flag if you think piece C must instead synthesize an empty fact set now (it can't without a revision from the reader).
3. **File placement + injected reader.** **Default vote: worker-composition** (`apps/worker/src/composition/canonicalFactSet.ts`), reader injected (fake in tests) — consistent with A/B, no direct fs. Flag if you'd rather site it elsewhere.
4. **Defensive try/catch around the reader call.** The `CommittedVaultReader` contract is never-throw (it catches internally → `undefined`). **Default vote:** add a belt-and-suspenders try/catch that maps a (contract-violating) rejection to `absent` — cheap, fail-safe, consistent with piece B's non-Result/rejection guard. Flag if you'd rather rely on the contract (no guard).
