# /tdd brief — serving_coverage_reader_parity_wiring

## Feature
Wire the durable **`ParityReportStore`** read-port (built in Brick-B slice 1 / brief 052, `ca10090`) into **`createServingCoverageReader`** so the serving-coverage **parity leg** reads the LATEST persisted `ParityReport` for a workspace @ its head revision — replacing today's unconditional `parity: undefined`. Widen the `ServingCoverageReader` seam to **sync-or-async** (mirroring the async `CommittedVaultReader` sibling), make the reader `async`, and make `createServingContextLoader` **`await`** it. Fail-closed (§6 nothing-fails-silently): a store fault ⇒ degrade, never a false green. Ships **DORMANT** — boot does not bind a real store until B4 (the `store` dep is OPTIONAL; unbound ⇒ `parity: undefined` ⇒ degrade, byte-equivalent to today).

**HONEST-INTERIM continues (the C5.4a pattern):** B2 wires the PARITY leg only. `oracleBuildOk` (the rebuild-oracle build-status leg) stays `false` — a distinct later slice — so coverage remains AND-degraded even with a green report + valid pin. The reader is REAL wiring that returns real data when a bound store has a report and degrades honestly otherwise. **Propose stays structurally OFF**; nothing is armed here.

## Things to flag at Step 2.5
1. **Store dep OPTIONAL (unbound⇒degrade, boot untouched, B4 binds) vs REQUIRED (B2 touches boot now).** My default vote: **optional `store?`** — keeps B2 atomic + boot byte-equivalent + dormant (the waiver-holder pattern, worker Lesson 20); B4 is the single composition-root entry that binds the real adapter + the e2e. Avoids splitting boot changes across two slices; preserves the "unbound seam degrades" precedent (`resolveVault`).
2. **Reader ALWAYS async vs async-only-when-store-bound.** My default vote: **always async** (returns a `Promise` even store-unbound) — mirrors `CommittedVaultReader` (always async), simplest union, `await` no-ops downstream; the seam already documents sync-or-async so existing sync fakes still typecheck.
3. **`oracleBuildOk` — wire in B2 or leave `false`?** My default vote: **leave `false`** — the rebuild-oracle build-status leg is a distinct safety-relevant concern (its own slice). B2 wires the PARITY leg only; coverage stays AND-degraded (correct + inert). Do NOT bundle it.
4. **Loader revision-scope re-check (`:173-177`) — keep or drop as now-redundant?** My default vote: **KEEP unchanged** — the store query scopes by revision AND the B1 read-back re-gate asserts payload rev==query, but the loader's re-check is the staleness kill-switch's last line (defense-in-depth). Removing it would rely solely on the store.
