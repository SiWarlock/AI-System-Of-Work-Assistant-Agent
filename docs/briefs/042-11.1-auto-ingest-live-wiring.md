# /tdd brief — auto_ingest_live_wiring

## Feature
Make the shipped Electron desktop app run the vault→ingestion loop live, behind an **owner-configurable, default-OFF** opt-in: when the owner sets the env opt-in AND a real `vaultRoot`, the worker-host augments the `bootWorker` call with `vaultWatch` + a minimal `proofSpineParams` (registering the Temporal worker) + a `temporalAddress`, so a `.md` dropped in the owner's vault is captured → dispatched → run by `sourceIngestion` on a local Temporal dev-server. Default (env unset) = TODAY'S exact safe degraded boot (no watcher, no Temporal worker). All-local; crosses NO hard line (the built 11.8 machinery is reused; the write is the local KnowledgeWriter sole-writer path — NOT the propose bridge, NOT GBrain write-through).

## Acceptance criteria (what "done" means)
- [ ] A pure `gateAutoIngest(opts)` helper returns `{ vaultWatch, proofSpineParams, temporalAddress } | undefined`: **undefined** (⇒ today's degraded boot) when the opt-in is OFF **OR** `vaultRoot` is absent; the wiring object only when opt-in ON **AND** `vaultRoot` present (fail-safe, mirrors `gateCopilotVaultReadDeps`).
- [ ] **Default OFF** — with the env opt-in unset, the worker-host passes NO `vaultWatch` and NO `proofSpineParams` to `bootWorker` (byte-equivalent to today's degraded boot); no watcher, no Temporal worker, no behavior change to the shipped default.
- [ ] **Owner-configurable, no hardcoded path** — `vaultRoot` (the owner's real Obsidian vault) + the opt-in + the ingest `workspaceId` + optional `temporalAddress` come from env/config (NOT hardcoded); resolved in `main/index.ts` (where `process.env` is already used) and threaded to the worker-host over the existing IPC config path (the same path `vaultRoot` already takes).
- [ ] When opted-in + `vaultRoot` present: the built `vaultWatch` carries the configured `{ workspaceId, sensitivity }`; the built `proofSpineParams` carries a real `.sourceIngestion` binding (`boundWorkspaceId` = the configured workspace) with inert/deterministic meeting leaves; `temporalAddress` defaults `127.0.0.1:7233`.
- [ ] **WS-8** — ingestion is bound to the configured workspace by POLICY (`vaultWatch.workspaceId` / `sourceIngestion.boundWorkspaceId`), never content-inferred (reuse the existing binding-vs-decoy guarantee).
- [ ] **Degraded-safe / fail-closed (§16)** — opt-in ON but Temporal absent ⇒ `dispatchSourceIngestion` fails closed (`temporal_unavailable`, health item, never throws); the shipped default (opt-in OFF) is unchanged. NO crash on any path.
- [ ] All new unit tests pass; repo-wide `pnpm -w turbo run typecheck test` green; `SOW_TEMPORAL=1` integration suites stay green; `/preflight` clean. **MANDATORY Step-8 review = security-reviewer + code-quality-reviewer** (real-I/O boot activation + untrusted-content ingestion).
- [ ] No hard line crossed — no propose flip, no GBrain write-through, no external fetch/write. The ingestion write is the local KnowledgeWriter sole-writer path (existing, governed).
