# /tdd brief ‚Äî external_write_transport_owner_gate

## Feature
Close the audit-found gap that the outbound external-write transport can be swapped from the deterministic stub to a real vendor HTTP client by a **one-line SOURCE EDIT with no owner gate**. Add a default-OFF, strict-`=== true` config seam (MIRRORING the existing `gateAutoIngest` / `keychainSecrets` / reconcile-gate idiom) so a non-stub `AdapterTransport` can only be selected by a **deliberate owner config**, never by editing a hardcoded call. The shipped default stays the stub ‚Äî **byte-equivalent, fully dormant**. This change makes real external writes HARDER to enable; it does NOT enable them (no real transport is built or bound ‚Äî the real-factory seam stays unbound by default).
