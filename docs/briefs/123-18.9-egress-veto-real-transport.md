# /tdd brief — egress_veto_honored_under_real_transport

## Feature
**Assert** the Employer-Work egress veto is honored under the real transport: an employer-work job with egress-ack OFF + only a cloud route **fails CLOSED** (never a cloud fallback), the same job with a local provider runs on **loopback** (zero-egress), the veto is ordered **before the run leg**, and OpenRouter is treated as **its own cloud processor** (not an OpenAI/local alias). The veto CODE is already built + robust (`vetoJobEgress` fail-closes on DENY + refuses a widened/substituted route); this slice PINS its behavior at the assembled-broker level against the real (dormant) transport config. **SAFE-BUILD: deterministic assertions against the dormant transport — no real cloud call; the leakage-eval (zero raw employer content on a real cloud egress) is the crossing.**

## Acceptance criteria (what "done" means)
- [ ] **Employer-work + ack-OFF + cloud-only route ⇒ fail CLOSED:** the broker's egress veto DENIES (`EMPLOYER_RAW_EGRESS_UNACKNOWLEDGED`); no run, no cloud fallback. Asserted against the assembled broker with the real (dormant) transport config.
- [ ] **Employer-work + ack-OFF + local provider present ⇒ loopback:** the veto ALLOWS the local (loopback) route; the job runs zero-egress.
- [ ] **Veto ordered before the run leg:** on a denied job the run leg is NEVER reached (spy-confirmed); a widened/substituted route from the veto is refused (`MALFORMED_POLICY_INPUT`, fail-closed).
- [ ] **OpenRouter is its own cloud processor:** an OpenRouter route (egressClass cloud) is DENIED for employer-work + ack-OFF, exactly like OpenAI — never miscategorized as local or an OpenAI alias.
- [ ] **SAFE-BUILD:** no real cloud call — the assertions run against the dormant transport; the leakage-eval is the crossing (not this slice).
- [ ] All unit tests pass; `/preflight` clean.
