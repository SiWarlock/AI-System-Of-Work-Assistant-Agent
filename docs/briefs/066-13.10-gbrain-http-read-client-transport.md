## RED test outline (Step 2)
Tests in `packages/knowledge/test/gbrain/gbrain-http-read-client.test.ts` (fake `HttpTransport` + fake `SecretsAccessor`):

1. **`invoke_dispatches_through_injected_transport`** — Asserts: `invoke("graph", payload)` calls the fake transport once with the expected URL/method/path/body (payload serialized) + a bearer header. Why: §7 injected-transport dispatch.
2. **`two_xx_body_resolves_parsed_json`** — Asserts: a 2xx JSON body resolves as the parsed object. Why: §6 read surface.
3. **`token_resolved_from_tokenRef_never_logged`** — Asserts: the fake `SecretsAccessor.getSecret` is called with `grant.tokenRef`; the resolved token rides only the request header; `JSON.stringify` of any emitted log/fault never contains the token. Why: REQ-S-003 / safety rule 7.
4. **`locked_or_missing_token_fails_closed_no_dispatch`** — Asserts: a `SecretUnavailable` result ⇒ `invoke` rejects (redacted) and the transport is NEVER called (capture-reason + assert unconditionally, Lesson 15). Why: fail-closed token (adapter → `transport_fault` → degrade).
5. **`off_allowlist_or_non_loopback_endpoint_refused`** — Asserts: an endpoint not loopback / not on the allowlist ⇒ refused, transport NOT called; a loopback + allowlisted endpoint ⇒ dispatched. Why: SSRF/egress guard (Lesson 4 + `assertLocalEndpointAllowed`).
6. **`non_2xx_status_redacted_failure`** — Asserts: a 4xx/5xx ⇒ a redacted typed failure; raw body + token absent from it. Why: fail-closed status mapping.
7. **`transport_throw_or_malformed_body_redacted`** — Asserts: a transport reject AND a non-JSON body each ⇒ a redacted failure; the raw cause/token never surfaces (Lesson 15 unconditional). Why: never leak on the fault path.
8. **`op_to_path_mapping_per_allowed_op`** — Asserts: search/graph/timeline/schema_read/health/contained_synthesis each map to the documented path/method. Why: §6 op surface.
