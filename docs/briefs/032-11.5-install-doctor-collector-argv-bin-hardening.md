# /tdd brief — install_doctor_collector_argv_bin_hardening (Phase 11, slice 11.5-e)

## Feature
A small defense-in-depth hardening of the install-doctor collectors' exec surface (closes two accepted LOW residuals from the 11.5 arc): (1) add a `--` end-of-options separator before every positional PATH argument so a `-`-leading config path can't be read as a flag; (2) pin the security-sensitive macOS SYSTEM probes to ABSOLUTE bins (`/usr/bin/fdesetup`, `/usr/bin/security` — joining the already-absolute `/bin/ls`·`/sbin/mount`·`/bin/ps` posture probes) so a hostile `PATH` can't shadow a security-state probe. The version-presence bins (`node`/`pnpm`/`temporal`/`gbrain`) stay BARE by design (the check IS "is it on `PATH` at the right version"). LOCAL-ONLY, deterministic. **This slice DELIBERATELY edits the 11.5-a/b/c collector files** (that's the point — the prior slices deferred this hardening).

## Acceptance criteria (what "done" means)
- [ ] Every collector command that passes a positional PATH arg uses a `--` end-of-options separator BEFORE the path (`/bin/ls -lde -- <vaultDir>`); a `-`-leading path is then read as a positional, never a flag.
- [ ] The security-sensitive macOS SYSTEM probes use absolute bins: `fdesetup` → `/usr/bin/fdesetup`, `security` → `/usr/bin/security` (join the already-absolute posture probes) — a hostile `PATH` cannot shadow a security-state probe.
- [ ] The version-PRESENCE bins stay BARE — `node`/`pnpm`/`temporal`/`gbrain` remain `PATH`-resolved (the prerequisite check IS `PATH`-presence + version; absolutizing them would defeat the check). Pinned so a later change can't over-absolutize them.
- [ ] Behavior is UNCHANGED for well-formed inputs (the hardening only closes the flag-confusion / PATH-shadow edges); all existing 11.5-a/b/c/d tests stay green after the fixed-argv assertions are updated to include `--` / the absolute bins.
- [ ] Fast-unit assertions on the exact argv/bin per command + a `-`-leading path case; the gated `SOW_DOCTOR_REAL=1` run still yields well-formed probes with the hardened argv. Repo-wide `turbo typecheck test` green; `/preflight` clean.

## Things to flag at Step 2.5
1. **`--` separator scope.** Default vote: **add `--` before the positional path in `/bin/ls -lde`; audit every other collector command — `mount`/`ps`/`git remote -v`/`<bin> --version` take NO positional path (git uses `cwd`), so `ls` is the only one needing `--`.** Confirm the audit (any missed positional-path command gets `--` too).
2. **Absolute-bin set.** Default vote: **absolutize ONLY the macOS SYSTEM security tools with a fixed system location — `/usr/bin/fdesetup`, `/usr/bin/security`** (join `/bin/ls`·`/sbin/mount`·`/bin/ps`). These are the security-state probes where a PATH-shadow is a real lie-about-security-posture vector AND the tool has a fixed location. Confirm.
3. **`git` bin.** Default vote: **keep `git` BARE** (`PATH`-resolved) — it's a version/remote check (not a security-posture probe), and users legitimately run a newer non-`/usr/bin/git`; hardcoding `/usr/bin/git` could miss it or trip the Xcode-CLT prompt. (If you judge the remote check security-sensitive enough to warrant `/usr/bin/git`, flag it.)
4. **Absolute-path portability.** Default vote: **the absolute macOS system paths are fine for this Mac-first V1** (§13 is macOS). No cross-platform abstraction needed now; note it if you disagree.
