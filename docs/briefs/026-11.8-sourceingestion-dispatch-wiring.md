# /tdd brief — sourceingestion_dispatch_wiring (make-it-real arc, C3a)

## Feature
Build the production-side programmatic DISPATCH entry that STARTS a `sourceIngestion` workflow run on the worker's task queue via a Temporal Client — degraded-safe when Temporal is unavailable. C1 registered the workflow so it CAN execute; this slice adds the client-side path to trigger a run (which the C3b file-watcher will call). LOCAL-ONLY, TestWorkflowEnvironment-gated; the team does NOT stand up a real dev-server.

## Acceptance criteria (what "done" means)
- [ ] A programmatic `dispatchSourceIngestion(input, deps)` entry starts a `sourceIngestion` run on the proof-spine task queue via a Temporal Client, returning a typed `Result` (the started run's handle/id on success).
- [ ] **Idempotent dispatch:** the Temporal `workflowId` is a deterministic key derived from the source (the run's `idempotencyKey`), so re-dispatching the SAME source does NOT start a duplicate run (Temporal workflowId-reuse dedupe; mirrors the driver's `resolveRun`).
- [ ] **Degraded-safe:** when Temporal is unavailable (no client / connect failure), dispatch returns a typed `err` (fail-closed) and surfaces a §16 health item — it NEVER throws or crashes the boot; no silent drop.
- [ ] The dispatch carries a `trigger` (`"event"` | `"manual"`) through to the run — no invented closed enum (the field stays an open string per its arch_gap).
- [ ] **`SOW_TEMPORAL`-gated** integration test: `dispatchSourceIngestion` on a `TestWorkflowEnvironment` → the run executes to the deterministic terminal state (reuses C1's live harness); a second dispatch with the same key → ONE run (dedupe).
- [ ] Default suite unaffected (`describe.skipIf(!SOW_TEMPORAL)`); repo-wide `turbo typecheck test` green with no server; `/preflight` clean.
