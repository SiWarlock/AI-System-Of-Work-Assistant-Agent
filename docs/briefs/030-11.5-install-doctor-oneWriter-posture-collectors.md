## Acceptance criteria (what "done" means)
- [ ] `collectPostureProbes(input)` (pure over the injected `RunCommand` + injected config) returns the `vaultAcl` + `gbrainMount` + `strayGbrainProcess` partial `ProbeSnapshot` from the real check results.
- [ ] **vault-acl:** the worker is reported the SOLE write principal ONLY when the vault dir's owner is the worker principal AND no group/other write bit is set AND no extended-ACL entry grants write to any non-worker principal; ANY other/unparseable state → `{ workerIsSoleWritePrincipal: false }` (assume-worst → the engine's `vault_acl` finding).
- [ ] **gbrain-mount:** `{ readOnly: true, mountPointCanonical: true }` ONLY when the mount backing the canonical brain path is flagged read-only AND resolves to the canonical mount point; a writable OR mispointed OR absent/unparseable mount → the not-read-only / not-canonical shape (assume-worst → the `gbrain_readonly_mount` finding).
- [ ] **stray-gbrain-process:** a `ps` scan classifies each write-capable gbrain process (`serve` / `sync_install_cron` / `autopilot` / `jobs_work` / `dream`) bound to the canonical brain into ONE closed `StrayGbrainOp` label; the safe state (no write-capable gbrain writer) → `{ strayProcesses: [] }`; a `ps` fault/unparseable → the field is OMITTED (undefined) so the engine's `p == null` branch fails it closed to a `stray_gbrain_process` finding (we cannot confirm "no stray writer").
- [ ] **Redaction-safe by construction:** the collector emits ONLY the classified `StrayGbrainOp` labels — NEVER a raw `ps` line, argv, absolute path, or secret. Read-only gbrain ops (`query`/`search`/…) are NOT classified as stray.
- [ ] **Never-throws + fail-closed:** every sub-probe runs its injected check under a TOTAL guard; a missing binary / nonzero exit / timeout / malformed output / thrown port → the assume-worst shape (never a throw, never a fabricated ok). Reuses 11.5-a's `createLocalCommandRunner` (fixed argv array, `shell:false`, timeout+cap, errno-only redaction) — NO new exec surface, NO shell string.
- [ ] Composes into `runPrerequisiteDoctor` (the doctor composition) so the fed snapshot now carries all 10 fields; the pure `runDoctor` maps them unchanged.
- [ ] Fast-unit (injected `RunCommand` fakes: green / assume-worst / malformed / fault for each probe) + an OPTIONAL `SOW_DOCTOR_REAL=1`-gated real-adapter test; the default suite never shells out. Repo-wide `turbo typecheck test` green; `/preflight` clean.

## Things to flag at Step 2.5
1. **vault-ACL check mechanism (the crux).** Text-parse `/bin/ls -lde <vaultDir>` (captures owner + POSIX mode + EXTENDED ACL entries) vs `node:fs.stat` (uid + mode bits ONLY — misses extended ACLs). My default vote: **`/bin/ls -lde` via the injected `RunCommand`** — macOS extended ACLs are the actual "another principal can write" vector that `fs.stat` mode bits silently miss; keep ONE injected port; fail-closed on any unparseable/foreign-write state. Confirm the parse is conservative (unknown ACL grammar → assume-worst false).
2. **stray-process fail-closed representation.** On a `ps` fault, OMIT the `strayGbrainProcess` field (engine `p == null` → finding) vs emit a sentinel non-empty array. My default vote: **OMIT the field** — the engine already fail-closes an absent probe to a finding; a sentinel would need a fake `StrayGbrainOp`. Simpler + type-clean.
3. **"bound to a canonical brain" determination.** A write-capable gbrain process is stray IFF its command line references the canonical brain path; but what about a gbrain writer with NO resolvable brain arg (implicit default)? My default vote: **fail-closed — flag an unresolvable-brain write-capable gbrain process as stray** (safety rule 1: we cannot prove it is NOT bound to the canonical brain). Accept the possible false-positive over a missed hidden writer. Confirm the canonical-brain-path config is the authority (injected, no inference).
4. **`serve` bound to the canonical brain is flagged even for a loopback read path.** `serve` ∈ `STRAY_GBRAIN_OPS` (per the spec + the memory-noted `serve --http` read path exposes write MCP tools). My default vote: **follow the spec's closed set exactly — a `serve` bound to the canonical brain IS a finding**; the read-only-mount posture (probe 2) is what makes an exposed serve non-writable, and the operator adjudicates. Do NOT special-case `serve`. Confirm.
5. **Collector file placement.** New `posture-collectors.ts` (my default — safety-isolated, self-contained review diff) vs extend `probe-collectors.ts`. My default vote: **new `posture-collectors.ts`**.
6. **Injected config surface.** `PostureProbeInput` carries `{ run: RunCommand, vaultDir, canonicalBrainPath, workerPrincipal }` — all CONFIG (a policy decision), never probed (no inference, mirroring `repoDir`/`localBackupAccepted`). My default vote: **yes, config-injected**; confirm `workerPrincipal` source (e.g. the resolved worker uid/username from boot config, not `process.getuid()` inside the pure collector).
