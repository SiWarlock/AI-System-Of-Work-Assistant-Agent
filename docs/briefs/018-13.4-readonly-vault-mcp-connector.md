# /tdd brief — obsidian_vault_readonly_tool_surface (task 13.4 — a read-only Obsidian-vault MCP connector: a governed read-ONLY tool surface that structurally cannot write Markdown; the 3 write tools are NOT registered — KN-4/KN-9; dormant over a faked transport)

## Feature
Build task 13.4 — the **read-only Obsidian-vault MCP connector** as a governed read-ONLY tool surface in `packages/integrations`. It registers ONLY the 5 vault read tools (`obsidian_search` / `read_note` / `backlinks` / `vault_health` / `validate_note`) and does **NOT** register the 3 vault write tools (`save_note` / `update_note` / `capture`) — so no MCP path can write canonical Markdown (safety rule 1 / KN-4 / KN-9). A read-only descriptor set + a fail-safe registry (an unknown/unregistered tool id ⇒ rejected, mirroring `isMutatingCopilotTool`'s unknown⇒mutating default) + a read-only `invoke` that routes a REGISTERED read tool to a faked, injected transport and can never reach a write surface. Ships **DORMANT** (faked transport; no real MCP SDK, no real vault I/O, no per-workspace server process yet — that's injected config / a named wiring follow-up).

## Acceptance criteria (what "done" means)
- [ ] **Only the 5 read tools are registered:** `registeredToolIds()` === the 5 reads (`obsidian_search`/`read_note`/`backlinks`/`vault_health`/`validate_note`); `OBSIDIAN_VAULT_READ_TOOLS` every entry `mutating:false`.
- [ ] **The 3 write tools are NOT registered (KN-4/KN-9):** none of `save_note`/`update_note`/`capture` appears in `registeredToolIds()`; `isRegisteredReadTool('save_note') === false` (and the other two).
- [ ] **Fail-safe on unknown (mirror `isMutatingCopilotTool`):** an unknown/unregistered tool id ⇒ `isRegisteredReadTool` false AND `invoke` ⇒ a typed rejection err — never a silent pass, never a write.
- [ ] **`invoke` of a registered read tool routes to the injected transport** and returns its faked `VaultReadResult` (proves the read surface is live over the faked seam); `invoke` of a write/unknown tool ⇒ typed `VaultReadError` (no transport call for a write, no write surface reached).
- [ ] **Structurally cannot write (safety rule 1):** the adapter never imports `@sow/knowledge`, never touches `createFsVault`/`node:fs` writes/the Tool-Gateway write path — i.e. it passes the same write-surface denylist the 13.1-a anti-corruption guard enforces (Lesson 12). (A belt-and-suspenders assertion: no write-surface token in the module.)
- [ ] **Never-throws / typed Result (Lesson 11):** a transport that THROWS resolves to a typed err (never rejects); a faked `!ok`/malformed transport result ⇒ typed err; the whole transport-call-plus-map runs under one try.
- [ ] **No-inference / isolation:** `config.workspaceId`/`vaultScope` are passed through, never invented; the read scope is a READ scope (never write/mutate — reuse the `not.toMatch(/write|create|update|delete|admin|mutat/i)` assertion from `connector-adapters.test.ts`).
- [ ] `packages/integrations` `turbo typecheck test` green; repo-wide `pnpm -w turbo run typecheck test` green (additive integrations adapter + tests; no cross-package contract change — but run repo-wide belt-and-suspenders).
