# /tdd brief — reconcile_driver

## Feature
Build the **pure, trigger-agnostic reconcile DRIVER** (piece **D** of the reconcile-TRIGGER arc, task 13.10) — the end-to-end composition that, given a `workspaceId` + injected async collaborators, assembles one reconciliation pass and routes its outcome: `buildCanonicalFactSet` (piece C) → on `derived`, `buildReconcilerDbProjection` (piece B) → assemble the `ReconcileRequest` (**`rebuildOracle` omitted** — pre-delegated decision #2) → `runReconcilePass` (piece A) → return a typed 4-way **`ReconcileDriverOutcome`**. It **short-circuits** an absent/broken canonical reference (skips the gbrain read entirely) and **never throws** (a `runReconcilePass` rejection is caught into a typed `pass_faulted`). **REFRAME (orch17): this defers the pre-delegated workflow-weight decision** (full Temporal `reconcileParity` workflow vs a lighter worker-side scheduled pass) **to piece E** — the pure driver taking injected async collaborators works under EITHER trigger model (proxyActivities in a workflow, direct calls in a worker pass), so the choice manifests where the driver is *triggered*, not here. Ships **DORMANT + reachability-waivered**: no production caller (piece E/F), no boot, no Temporal, no real I/O — all collaborators injected (fakes in tests). The two OWNER-GATED real bindings (live `GbrainReadGrant` transport behind B; real `RebuildOracleSet`) stay unbound.

## Acceptance criteria (what "done" means)
- [ ] An exported driver (default name **`runReconcileForWorkspace`**) `runReconcileForWorkspace(workspaceId: string, deps: ReconcileDriverDeps): Promise<ReconcileDriverOutcome>`, where `ReconcileDriverDeps` injects: `getCanonicalFactSet: (ws) => Promise<CanonicalSnapshotOutcome>` (piece C, bound to the real reader at wiring), `getDbProjection: (ws) => Promise<ReconcilerDbProjection>` (piece B, bound to the real adapter at wiring), `origin: ReconcileTriggerOrigin`, and `passDeps: RunReconcilePassDeps` (piece A). Every collaborator a fake in tests.
- [ ] **Reconciled path:** on `getCanonicalFactSet → derived`, the driver awaits `getDbProjection`, assembles `req = { origin, canonicalSet: derived.set, dbProjection }` **without `rebuildOracle`** (decision #2), calls `runReconcilePass(req, passDeps)`, and returns `{ kind: "reconciled", disposition }`. Asserts: the assembled `req` carries the injected `origin` + the `derived.set` + the `dbProjection` (no re-derivation); `runReconcilePass` ran once; the disposition is forwarded.
- [ ] **Short-circuit on absent:** `getCanonicalFactSet → absent` ⇒ `{ kind: "skipped_absent" }` and **`getDbProjection` is NOT called** (no gbrain read without a canonical reference), `runReconcilePass` not called. Asserts: `skipped_absent` + 0 `getDbProjection` calls + 0 `runReconcilePass` calls.
- [ ] **Short-circuit on derive_error:** `getCanonicalFactSet → derive_error` ⇒ `{ kind: "skipped_derive_error", error }` (surfaced typed for the trigger caller to route to health — the driver does NOT itself materialize a HealthItem, decision #4), `getDbProjection`/`runReconcilePass` not called. Asserts: `skipped_derive_error` carrying the typed error + 0 downstream calls.
- [ ] **Fail-closed — pass fault caught, never thrown:** when `runReconcilePass` REJECTS (a store `record` fault or a health-sink fault, per piece A), the driver catches it into `{ kind: "pass_faulted", cause }` — never throws, never a false `reconciled`. Asserts: `pass_faulted` carrying the cause; captured + asserted UNCONDITIONALLY (Lesson 15). Why: §12 fault-visible-but-never-throws — the trigger model (Temporal or a bare worker pass) always gets a typed result.
- [ ] **`rebuildOracle` omitted (decision #2):** the assembled `req` has NO `rebuildOracle` ⇒ the reconciler's `coverageComplete` rests on `dbProjection.complete` (NOT forced false). Asserts: `req.rebuildOracle === undefined`. (The real `RebuildOracleSet` is OWNER-GATED — deferred.)
- [ ] Ships **DORMANT + reachability-waivered** — no production caller (piece E wires the trigger, F the boot gate), no boot, no Temporal, no real I/O; all collaborators injected. `/wired` reports test-only reachability; the waiver names pieces E/F.
- [ ] All unit tests pass; repo-wide `pnpm -w turbo run typecheck test` green; `/preflight` clean.
- [ ] **MANDATORY Step-8 dual review = security-reviewer + code-quality-reviewer** — the driver is the end-to-end trust-gate-coverage feeder; a mis-routed skip (reconciling on a broken canonical set) or a swallowed pass fault is a trust-gate defeat. Safety-critical; own commit.
