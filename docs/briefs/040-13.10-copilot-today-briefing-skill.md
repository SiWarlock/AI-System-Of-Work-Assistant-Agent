## Acceptance criteria (what "done" means)
- [ ] `answerCopilotBriefing(deps, input)` returns a `Result<UiSafeCopilotAnswer, FailureVariant>` for a KNOWN workspace: assembles the workspace-scoped Today read-model into a candidate context, synthesizes, and gates the candidate → UI-safe.
- [ ] **WS-8 fail-closed** — an unknown workspace fails closed (`WORKSPACE_NOT_FOUND`); the assembled context is RE-GUARDED (`enforceRetrievalScope`, defense-in-depth over any adapter) so a foreign-workspace read-model item can NEVER be synthesized into a brief. **No raw cross-workspace bodies** ever enter the briefing (only this workspace's already-UiSafe read-model items).
- [ ] **Governed core is single-sourced** — the post-retrieval governed sequence (WS-8 re-guard → authoritative posture resolve → route select → `decideCopilotEgress` veto BEFORE synthesis → synthesize on the veto-CLEARED route → `toUiSafeCopilotAnswer`) is REUSED by both `answerCopilotQuestion` and `answerCopilotBriefing` (extracted helper), so the safety machinery cannot drift between the two.
- [ ] **Egress veto governs the briefing** — the fail-closed Employer-Work egress veto runs BEFORE any synthesis (employer-work + cloud + ack OFF ⇒ fail closed, no provider call); the derived `egressProcessor` notice threads onto the answer for employer-work cloud egress. NO external fetch.
- [ ] **Read-only / propose OFF** — no side effect, no Markdown write, no ProposedAction/propose path; `copilotProposeMode`/`copilotProposeKnowledge` untouched (HARD LINE).
- [ ] **Candidate-data gate** — the synthesized brief is CANDIDATE data until it passes `UiSafeCopilotAnswerSchema` via `toUiSafeCopilotAnswer`; an over-cap / leak-shaped / empty candidate is REJECTED (never served).
- [ ] `query.copilotBriefing` is mounted in `buildQueryRouter` behind `authedResolver` (auth + WS-8) and reachable through `composeAppRouter` on the real API server (Step 7.5).
- [ ] All new unit tests pass; repo-wide `pnpm -w turbo run typecheck test` green; `/preflight` clean. **MANDATORY Step-8 review = security-reviewer + code-quality-reviewer** (WS-8 / candidate-gate / egress surface).

2. **Output contract — reuse `UiSafeCopilotAnswer` vs a new `UiSafeCopilotBriefing`.** My default: **reuse `UiSafeCopilotAnswer`** (`{ answer: string[], citations, egressProcessor? }`) — a brief is answer-shaped (synthesized lines + citations to Today items); keeps this a non-frozen-contract slice. A structured/sectioned briefing type is a later frozen-contract round if needed.
3. **Governed-core reuse mechanism.** My default: **extract** `answerCopilotQuestion`'s post-retrieval sequence into an exported helper both procedures call (single-sourced veto+gate). Alternative (parameterize `answerCopilotQuestion`'s retrieval) couples the two inputs; the extract is cleaner. Confirm the helper's signature keeps the veto-cleared-route → synthesis binding intact (the P1.2b invariant: synthesis MUST use `decision.value.route`, never re-select).
4. **Briefing retrieval — new port vs reuse `CopilotRetrievalPort`.** My default: a **new `CopilotBriefingRetrievalPort`** (its source is the read-model, not knowledge passages) with an interim fixture-backed adapter (like `createFixtureRetrieval`) + a real read-model-backed adapter in boot. 5. **Synthesis prose = eval, not unit.** My default: the DETERMINISTIC wiring (retrieval assembly, governed-core reuse, gate, WS-8) is TDD'd here; the brief PROSE quality is EVAL-tested — coordinate with the eval-security track (do NOT touch `../SoW-build-evalsec`; a governance-eval case joins `copilot-governance.test.ts` as a follow-on, flagged not built here).
