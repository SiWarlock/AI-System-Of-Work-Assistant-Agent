# /tdd brief ‚Äî gbrain_startup_verify_boot_wiring (Phase 11, slice 11.3-b)

## Feature
Make the GBrain version-pin startup check REACHABLE in production: wire `verifyGbrainStartup` (11.3-a) into `bootWorker`'s GBrain-startup sequence so the running gbrain is probed against the pin at boot and, on degrade, a distinct version-pin **System Health item** is surfaced (the ¬ß16 observability the 11.3 acceptance requires) ‚Äî reusing the existing persistent health-surface path. **Degraded-safe: never blocks/crashes boot** (a gbrain-unavailable / mismatch / PENDING-sentinel degrade is the EXPECTED, safe outcome ‚Äî surface it, don't fail boot). Closes the 11.3-a reachability waiver. **OUT of scope (stays HITL):** the write-through gating / `writeThroughEnabled` flip, any serving-oracle re-plumb, and the `config/gbrain.pin` re-capture. LOCAL-ONLY.

## Acceptance criteria (what "done" means)
- [ ] `bootWorker` runs the GBrain version-pin verify at startup: loads the `GbrainPin` (from `config/gbrain.pin`), constructs the real `createGbrainVersionProbe`, calls `verifyGbrainStartup`, and on the degrade branch surfaces the returned `HealthItem` via the persistent health surface (so System Health shows the distinct version-pin item).
- [ ] **Degraded-safe / never blocks boot:** a degrade (gbrain-unavailable / sha-mismatch / index-schema-mismatch / PENDING-sentinel) surfaces the HealthItem and boot CONTINUES (the brain stays read-only/index-only ‚Äî the safe default); a fault in the verify path (probe throw, pin-load failure) is caught + logged + boot continues (best-effort, like the vault-watcher/reconciler boot wiring). NEVER a boot crash.
- [ ] The serving branch (a matched, LIVE-validated pin) does NOT flip write-through or re-plumb the serving oracle ‚Äî those stay HITL; this slice only surfaces the version-pin health state at startup.
- [ ] **Reachable:** `verifyGbrainStartup` is now called from the production boot entry ‚Äî `/wired gbrain_version_pin` traces bootWorker ‚Üí verify. Closes the 11.3-a waiver.
- [ ] An integration test drives bootWorker (or the extracted boot-verify step) with an injected probe: a degrade ‚Üí the HealthItem is surfaced + boot completes; a fault ‚Üí boot completes (no crash). Repo-wide `turbo typecheck test` green; `/preflight` clean.
