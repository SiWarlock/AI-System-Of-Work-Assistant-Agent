# /tdd brief — onboarding_ui + preset_picker (14.1 + 14.5 desktop legs, BUNDLED)

## Feature
The **desktop onboarding surface** — the real first-run flow that lets a user EXIST in the system: create a workspace → choose a vault root → pick a Simple/Professional/Founder/Advanced preset → land in the app. It drives the already-shipped worker backends (`onboarding.createWorkspace`, `presetProfiles.preview`), replaces the `scope.ts` provisioning-placeholder ids with real onboarded workspace ids, and preserves the fail-closed empty-until-onboarded posture. Bundles **14.1 onboarding UI** + **14.5 preset picker** (the same surface — the preset pick is a step of onboarding), per the owner's bundling directive. Desktop-track.

## Acceptance criteria (what "done" means)
- [ ] A new onboarding surface (`apps/desktop/renderer/surfaces/onboarding/`) drives the flow: **create workspace (name + type) → choose vault root → pick preset**, calling `onboarding.createWorkspace` on submit.
- [ ] **Preset picker (14.5):** selecting a preset calls `presetProfiles.preview(preset)` and shows the resulting profile (the distinct connector/workflow set per tier) so the choice is differentiated, not cosmetic.
- [ ] **Onboarding round-trips to scope:** on a successful `createWorkspace`, the new workspace id enters the scope store (`scope.ts`), so the workspace becomes a selectable scope and the app leaves the first-run/empty state — driven by the real WS-8 registry, not the placeholder ids.
- [ ] **Fail-closed empty-until-onboarded (WS-8):** with NO workspace onboarded (no `provisionDev`), the app shows the first-run/empty state and scoped reads stay empty; a workspace absent from the registry gets zero read path (the scope store must not resurrect the removed placeholder ids as if populated).
- [ ] **Error handling:** a `createWorkspace` failure (e.g. a typed error) surfaces a user-visible error state, never a raw driver cause / secret (mirrors the worker's redaction-safe typed Results).
- [ ] **Reachable:** the onboarding surface is routed/mounted in AppShell (the first-run entry when no workspace exists) — a real production entry, not test-only.
- [ ] Deterministic-flow tests pass (component/store tests for the wiring, scope-store update, fail-closed empty state, error state); `/preflight` clean (desktop + repo-wide).

## RED test outline (Step 2) — deterministic flow (the visual layer is `/design-review`)
1. `onboarding_submit_calls_createWorkspace` — submitting the flow calls `onboarding.createWorkspace` with the entered name/type/vaultRoot/preset. Why: §19.1 real onboarding.

4. `no_workspace_shows_first_run_empty` — with no onboarded workspace, the first-run/empty state shows + scoped reads stay empty (fail-closed). Why: WS-8 empty-until-onboarded.
5. `createWorkspace_error_shows_safe_error_state` — a typed failure surfaces a user error state, no raw cause/secret. Why: §16 / rule 7 (renderer-safe).
