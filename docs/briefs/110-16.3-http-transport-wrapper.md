## LOAD-BEARING safety (the heart of this slice)
1. **SSRF/egress guard on the FINAL URL, BEFORE any token read.** The guard runs on the resolved final URL (after any redirect/templating) and BLOCKS private/loopback/link-local/internal-metadata targets (RFC1918, 127/8, ::1, 169.254/16, fc00::/7, `*.internal`, cloud metadata IPs) — a source URL that resolves to an internal host is an SSRF exfil vector. The token/credential read happens ONLY after the URL passes the guard (never leak a token to an attacker-controlled internal endpoint).
2. **ING-7 read-only at admission.** The transport admits ONLY read/GET-shaped requests; a mutating verb (POST/PUT/PATCH/DELETE — beyond any read-only exceptions the spec allows) is rejected at admission. No write path exists in this transport.
3. **Send seam UNBOUND (dormancy).** The real `send` (undici/https/fetch) is not bound in the shipped default — a factory-spy / grep pin that no real outbound socket is constructed. The wrapper is exercised via an INJECTED fake `send` (no real network) + the guard/admission logic tested directly.
4. **Redaction (rule 7).** A guard-rejection / transport fault carries a redaction-safe typed error (no token, no raw URL-with-embedded-credential, no response body) to any log sink.
