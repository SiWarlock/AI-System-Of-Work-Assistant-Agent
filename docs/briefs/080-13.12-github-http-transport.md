# /tdd brief — github_http_transport (connector round 5 — GitHub read transport + `mapPage` widening)

## Feature
Add **`createGithubHttpTransport(deps): ConnectorTransport`** — GitHub issues read-only HTTP transport — a thin per-vendor spec over the connector template, PLUS a minimal backward-compatible widening of the shared `ConnectorHttpSpec.mapPage` signature so a body-only mapper can see the request cursor.

**Why the template touch (lead-ruled Option A, 2026-07-15).** Unlike Granola/Drive/Calendar (body cursor / `nextPageToken` in the response body), **GitHub REST paginates by page number** (`?per_page&page`) and returns a **bare JSON array** — there is NO next-page token in the body. The template's `mapPage(json)` is body-only, so it can neither read a next-page token nor compute `page+1` (it never sees the current cursor). The fix is to widen `mapPage(json)` → **`mapPage(json, request)`** (the token-free `TransportRequest` already in scope at the call site). This gives GitHub full page-number pagination — parity with the other 4 connectors — and pre-pays Linear's cursor need (R6). Page-number pagination with short-page termination (`done = arr.length < per_page`) is a fully valid, correct GitHub strategy, not a hack.

**Static Bearer (no OAuth in this slice).** GitHub auth is a Bearer token (fine-grained/classic PAT) via `Authorization: Bearer` — the template's bearer-string `SecretsAccessor` VERBATIM (like Asana's PAT / Granola's `grn_` key). No OAuth manager / refresh here (Gmail's OAuth is R7).

**Dormant + no hard line.** The real `HttpTransport` + `SecretsAccessor` + PAT stay UNBOUND (byte-equivalent); binding a real transport + real token is the owner's arming crossing (real external network I/O = HARD LINE).

- [ ] **Widen `ConnectorHttpSpec.mapPage`** type: `(json: unknown) => ConnectorTransportResult` → `(json: unknown, request: TransportRequest) => ConnectorTransportResult` (`TransportRequest` is already imported). Update the call site (currently `spec.mapPage(json)` inside the wrapped try/catch) to `spec.mapPage(json, request)` — where `request` is the closure's existing `TransportRequest` param (`{cursor?, readScope}`).
- [ ] **GUARDRAIL 1 — backward-compat (purely additive):** the 4 existing mappers (`asanaMapPage`/`driveMapPage`/`calendarMapPage`/`granolaMapPage`) stay **byte-unchanged** and keep their `(json)` signature — a 1-arg fn is assignable to the 2-arg type and calling it with an extra arg is a runtime no-op. ALL 4 existing transport suites stay green (preflight-proven). No regression; call it out in the slice.
- [ ] **GUARDRAIL 2 — safety rule 7 (no secret to the mapper):** the object passed as `request` to `mapPage` is the `TransportRequest` (`{cursor?, readScope}`) — it MUST carry NO `Authorization` header / bearer token (the token lives only in the local `secret` + `httpRequest.headers.Authorization`, never in `request`). This holds BY CONSTRUCTION; pin it with a template-level test.

### GitHub spec (`connectors/adapters/github.ts` — add alongside the existing `createGithubConnector`; EXISTING file ⇒ no OSB count-pin trip)
- [ ] `createGithubHttpTransport(deps: ConnectorHttpTransportDeps): ConnectorTransport` returning `createConnectorHttpTransport(GITHUB_HTTP_SPEC, deps)`.
- [ ] `GITHUB_HTTP_SPEC: ConnectorHttpSpec = { baseUrl: "https://api.github.com", allowedHosts: ["api.github.com"], resourcePath: "/issues", buildQuery: githubBuildQuery, mapPage: githubMapPage }`. `GITHUB_PER_PAGE = 100` (vendor max).
- [ ] **`githubPageFromCursor(cursor?): number`** — parse the cursor as a positive integer; absent / non-numeric / `<1` ⇒ **1** (fail-safe: a tampered/corrupt cursor re-reads page 1, never injects a param, never loops). Shared by buildQuery + mapPage (single source of the page number).
- [ ] **`githubBuildQuery(request): string`** — `?per_page=100&state=all&sort=updated&direction=desc&page=<githubPageFromCursor(request.cursor)>`. `page` is a guaranteed positive integer (no injection surface). (`filter` left at its `assigned` default; `since`/`labels`/`filter` are arming-era refinements.)
- [ ] **`githubMapPage(json, request)` (arch_gap candidate):** if `json` is **not an array** ⇒ `transportFailure("unknown", …)` (bare-array contract — fail-closed on an envelope/null/non-array). For each entry: non-object ⇒ fail; `recordId = node_id` (string, non-empty) else fail-closed; `items.push({ id: node_id, hash: githubContentHash(entry, node_id), raw: entry })`. Then `page = githubPageFromCursor(request.cursor)`, **`done = json.length < GITHUB_PER_PAGE`**, `nextCursor = done ? undefined : String(page + 1)`.
- [ ] **`githubContentHash(issue, recordId)`** — `payloadHash({ id: recordId, updated_at })` when `updated_at` is a non-empty string, else `payloadHash(issue)` (raw fallback — mirrors Granola).
- [ ] Export `createGithubHttpTransport` from `connectors/adapters/index.ts` (alongside the other 4 `*HttpTransport` exports).
- [ ] **Fault map (inherited):** 401/403 → `auth_locked`; 429 → `rate_limited`; else `unreachable`. GitHub 403-can-mean-primary-rate-limit is diagnostic-only (the base collapses ALL faults to `unreachable`); `X-RateLimit`/`Retry-After` backoff is arming-era. Document as arch_gap.
- [ ] `/preflight` repo-wide green; DORMANT + reachability-waivered (transport/secrets/PAT UNBOUND; byte-equivalent).

## RED test outline
### GitHub spec (`connector-github-transport.test.ts`)
1. **`github_2xx_maps_to_page`** — bare array of 2 issues (`node_id` + `updated_at`) ⇒ `TransportPage` (recordId = `node_id`, stable `payloadHash({id: node_id, updated_at})`, raw preserved); short page (2 < 100) ⇒ `done=true`, no `nextCursor`.
2. **`github_pagination_advances_on_full_page`** — a full page (`length === GITHUB_PER_PAGE`): cursor `undefined` ⇒ `done=false`, `nextCursor="2"`; cursor `"2"` ⇒ `nextCursor="3"`. `buildQuery` emits `page=1` for undefined, `page=2` for `"2"` (host + fixed params unchanged).
3. **`github_cursor_failsafe`** — tampered/invalid cursor (`"abc"` / `"0"` / `"-1"` / `""`) ⇒ `buildQuery` emits `page=1` AND `mapPage` computes `nextCursor` from page 1 (⇒ `"2"` on a full page) — never injects a query param, never loops.
4. **`github_malformed_shapes_fail_closed`** — non-array body (envelope object / `null` / number / string) ⇒ `TransportFailure`; entry non-object / missing `node_id` / non-string `node_id` ⇒ `TransportFailure`.
5. **`github_ssrf_and_redaction_inherited`** — admit `api.github.com`; an off-allowlist host refused (zero dispatch); 401 ⇒ `TransportFailure(auth_locked)`, safe status only, token never leaked.
6. **`github_end_to_end_via_createGithubConnector`** — whole `ConnectorPort.fetch` chain (page → `ConnectorRecord[]` with recordId/contentHash/payload; `done`/`nextCursor` threaded).
