# /tdd brief — granola_http_transport (connector round 4 — Granola read transport)

## Feature
Add **`createGranolaHttpTransport(deps): ConnectorTransport`** — the Granola meeting-notes read-only HTTP transport — as a thin per-vendor spec over the connector template. Round 4; the SIMPLEST template-follow yet — a thin Asana-clone with a **static Bearer API key (no OAuth)**. Owner-corrected scope: Granola HAS a public HTTP API (`public-api.granola.ai`) — NOT a local reader. Supplies only the Granola `ConnectorHttpSpec` (base URL, allowlisted host, resource path, buildQuery, candidate mapPage) + wires it to the existing `createGranolaConnector`.

**Static Bearer (no code change to the template).** Granola auth is a static API key prefixed `grn_` via `Authorization: Bearer` — the template's bearer-string `SecretsAccessor` VERBATIM (like Asana's PAT; NO OAuth manager / NO refresh). Simpler than the Google connectors.

**Dormant + no hard line.** Real `HttpTransport` + `SecretsAccessor` + `grn_` key stay UNBOUND (byte-equivalent); binding a real transport + the real key is the owner's arming crossing.
