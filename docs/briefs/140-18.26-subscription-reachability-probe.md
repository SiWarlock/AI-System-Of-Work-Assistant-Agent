# /tdd brief — subscription_reachability_probe

## Feature
Build the real, SPEND-FREE reachability check for the Claude subscription arm's HEALTH gate — a `probeSubscriptionReachability` in `@sow/providers` producing the reachability signal the 18.22 `probeClaudeSubscriptionHealth({checkReachable})` consumes. The worker injects it at the owner ARM via `config.subscriptionArm.checkReachable`; without it the arm's health source stays `FAIL_CLOSED_REACHABILITY` ⇒ HEALTH UNAVAILABLE (CP2's "health available" leg cannot pass — arming today correctly HALTs at CP2). This unblocks CP2.

## Acceptance criteria (what "done" means)
- [ ] `probeSubscriptionReachability` returns EXACTLY the shape 18.22's `checkReachable` consumes — verify against `subscription-health-probe.ts` (it reads `.loginPresent === true` and `.sdkReachable === true`; a `null`/non-object ⇒ ambiguous). So: `{ loginPresent: boolean, sdkReachable: boolean }`.
- [ ] **SPEND-FREE — the load-bearing constraint.** NO `query()` / completion / billable call (a completion at CP2 would spend before the gated run). Checks: (a) the local `claude` login/credential is PRESENT; (b) the `@anthropic-ai/claude-agent-sdk` is resolvable/usable. Context7-ground WHICH spend-free signal proves each (credential/login-state presence + module resolvability, or `Query.accountInfo().tokenSource`/`subscriptionType` IF that is spend-free — verify).
- [ ] **Fail-closed + total (§16 / L52):** any fault / throw / ambiguous ⇒ a NON-reachable result (`loginPresent:false` and/or `sdkReachable:false`), NEVER a false-positive reachable (no false-green). A throw folds to non-reachable, never escapes.
- [ ] **Real I/O behind a mockable seam** (mirror 18.22 / `createClaudeSubscriptionCompletion`): the SDK/login check is injected/mockable so the build never touches the real login; the deterministic fold is TDD-tested; the real leg is exercised via an injected fake.
- [ ] **Rule 7:** never surface the credential VALUE — presence/boolean only, no token bytes in any result/log.
- [ ] Dormant: NO production caller until the worker injects it at the arm. Reachability-WAIVERED (L11).
- [ ] `/preflight` clean. No cross-doc invariant change (internal providers helper; no Appendix-A model touched).

## Things to flag at Step 2.5
1. **`checkReachable` SYNC vs ASYNC.** 18.22's probe calls `deps.checkReachable()` SYNCHRONOUSLY. Confirm the shape is sync (`() => {loginPresent, sdkReachable}`). If a true network reachability needs async, FLAG — it widens the probe seam (worker's `createSubscriptionHealthSources` + the memoize assume sync). My default vote: **SYNC readiness** (login-credential-present + SDK-resolvable), spend-free; a deeper async live-ping is a deferred enhancement (the RUN itself is the true reachability test). Don't widen the seam unless Context7 shows sync can't prove readiness.
2. **What is "login present" + "sdk reachable" spend-free.** Context7-ground on the Agent SDK. Default: credential/login-state presence (no network) + module resolvability; NOT a billable call. If `accountInfo()`/`tokenSource` is the SDK's readiness signal AND is spend-free, prefer it (it's the truest subscription-active check); if it costs/round-trips, use the presence path.
3. **security-reviewer.** Touches the login/credential domain. My default vote: **YES (invariant — credential-adjacent, rule 7).**
