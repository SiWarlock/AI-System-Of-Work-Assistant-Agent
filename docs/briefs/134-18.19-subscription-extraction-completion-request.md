# /tdd brief — subscription_extraction_completion_request

## Feature

## Things to flag at Step 2.5
1. **Assembler input shape — take the built `AgentExtractionRequest`, or take the `AgentJob` + resolve internally?** Options: (a) take the already-built `AgentExtractionRequest` (+ content + opts); (b) take the `AgentJob` + content and call `buildMeeting/SourceExtractionRequest` internally (folding the fail-closed `schema_unresolved` into a `Result`). My default vote: **(a)** — keeps this assembler pure + total (no `Result`), leaves the meeting-vs-source selection + the fail-closed schema resolution to the worker runner (18.20) which already knows the capability; smaller surface. If you pick (b), the return becomes `Result<CompletionRequest, ExtractionRequestFault>` and the worker passes the job — flag the ripple.
2. **`maxBudgetUsd` semantics under a subscription (the lead-flagged question).** The SDK `query()` option `maxBudgetUsd` is enforced by the SDK even on the subscription (it returns `error_max_budget_usd` → the `budget` kind). So threading `maxCostUsd` → `maxBudgetUsd` IS a real dollar ceiling, not a no-op. My default vote: **thread it through unconditionally** (present ⇒ set; absent ⇒ omit). No code change to `claude-subscription-completion.ts` is needed (it already forwards the field). **Escalate to the orchestrator only if** grounding the SDK `query()` option name against Context7 shows `maxBudgetUsd` is renamed/removed in the installed `@anthropic-ai/claude-agent-sdk` version — that would be a load-bearing arch call. Ground the option name on Context7 (`@anthropic-ai/claude-agent-sdk` `query()`), per the lead directive, before finalizing.
3. **Default extraction `betas` — reuse `DEFAULT_COPILOT_BETAS` or define an extraction-specific constant?** Default vote: **define an extraction-specific `DEFAULT_EXTRACTION_BETAS` constant** (even if its value equals the Copilot one today) so a later model/beta divergence between synthesis and extraction doesn't require touching the Copilot path. Value: the 1M-context beta (`context-1m-2025-08-07`) — verify against the current catalog at ENABLE, not now.
