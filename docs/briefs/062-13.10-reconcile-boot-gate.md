# /tdd brief — reconcile_boot_gate

## Feature
Build the **default-OFF reconcile boot gate** (piece **F** of the reconcile-TRIGGER arc, task 13.10 — the **LAST DORMANT piece**) — a pure `gateReconcile(opts, deps) → ReconcileWiring | undefined` helper (mirror `gateAutoIngest` `boot.ts:547` + Lesson 2/8/16) that, on the OFF path (owner opt-in unset — the default), returns `undefined` so the shipped boot stays **byte-equivalent**, and on the ON path (armed — owner-gated, never taken by default) assembles the reconcile machinery: `createReconcileScheduler` (piece E) bound to the driver `runReconcileForWorkspace` (piece D) over the **never-reject** `buildCanonicalFactSet` (C) + `buildReconcilerDbProjection` (B) + a **non-throwing redacted** log sink + the health surface — every real dep a **thunk invoked ONLY on the gated-on path**, so nothing is constructed on the OFF path. Plus the `bootWorker` call site (a single `gateReconcile(...)` line, byte-equivalent by default). This **closes the arc's reachability chain** (`bootWorker → gateReconcile → scheduler → driver → builders`) while shipping fully **DORMANT** — the arming (flip the flag + provision the owner-gated `GbrainReadGrant` transport + the signing key + real corpora + the governance eval) is the **HARD LINE**, and this slice does **NOT** cross it (no flag armed, no transport bound).

Asserts: a wiring is returned; the scheduler is present; the dep-thunks ran (on this path only).
- [ ] **Never-reject binding (constraint a):** the driver's `getCanonicalFactSet`/`getDbProjection` are bound ONLY to `buildCanonicalFactSet`/`buildReconcilerDbProjection` (never a raw transport) — the arc's never-throw guarantee holds. Asserts (structural / typed): the bound collaborators are the never-reject builders.
- [ ] **Redacted non-throwing log + health from the safe summary (constraints b/c):** the scheduler's `log` is bound to a non-throwing sink; the health materializer builds the `skipped_derive_error`/ws-mismatch HealthItem from the safe `detail` code only (never the raw error). Asserts: the log sink receives only the redacted summary; a HealthItem is minted from the code.
- [ ] **Owner-gated transport stays UNBOUND (arming precondition, NOT crossed):** on the ON path the `GbrainReadAdapter` is bound over the **unbound** transport ⇒ the db-projection degrades (`complete=false`) ⇒ the recorded report is `coverageComplete=false` (never a false-green).
