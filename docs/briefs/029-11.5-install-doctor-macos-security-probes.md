# /tdd brief — install_doctor_macos_security_probes (Phase 11, slice 11.5-b)

## Feature
Extend the install-doctor real collectors (11.5-a, `5171f2a`) with the two macOS-security prerequisite probes — **FileVault** (full-disk encryption status) and **Keychain** (availability) — filling the `filevault` + `keychain` `ProbeSnapshot` fields the built pure `runDoctor` already diagnoses. Same shape as 11.5-a: real LOCAL macOS checks over the existing injected `RunCommand` port, never-throw, fail-closed. LOCAL-ONLY.

## Use case + traceability
- **Task ID:** 11.5 (install doctor — the macOS-security probe collectors; continues 11.5-a)
- **Architecture sections it implements:** `ARCHITECTURE.md §13` (Deployment / Install / doctor), with `§16` (Keychain-locked degraded posture) supporting. (§13 ∈ Phase 11 Spec anchors — no widen.)
- **Related context:** 11.5-a (`5171f2a`) built `collectPrerequisiteProbes` + `createLocalCommandRunner` (real `execFile`, `shell:false`, fixed argv-array, timeout+cap, errno-only redaction) + the pure `runDoctor` engine (`diagnoseFilevault`/`diagnoseKeychain` already exist, fail-closed). This slice REUSES the 11.5-a adapter + pattern; it only adds two collectors. **macOS TCC gotcha (known):** a security probe can hit "Operation not permitted" (Full-Disk-Access / permission-denied); the collector must fail-CLOSED to the assume-worst probe shape on a permission error, never throw or fabricate an `ok`. **HARD LINE:** LOCAL-ONLY (`fdesetup status` / a `security` keychain query — local macOS tooling, no network); nothing crosses the hard line.

## Acceptance criteria (what "done" means)
- [ ] `collectSecurityProbes(deps)` (pure over the existing injected `RunCommand`) returns the `filevault` + `keychain` `ProbeSnapshot` fields from the real check results (extend the 11.5-a collector or a sibling; compose into `runPrerequisiteDoctor`/`runDoctor`).
- [ ] **FileVault:** a real `fdesetup status` result → the `filevault` probe (`enabled` true/false); a non-`on`/unknown output → the not-enabled shape.
- [ ] **Keychain:** a real availability query → the `keychain` probe (available/locked); the assume-worst on absent.
- [ ] **Never-throws + fail-closed on TCC/permission errors:** a permission-denied / ENOENT / nonzero / timeout / malformed result → the assume-worst probe shape (which `runDoctor` maps to a `finding`), NEVER a throw or a fabricated `ok`.
