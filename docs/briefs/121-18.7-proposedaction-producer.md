# /tdd brief — proposedaction_producer

## Feature
Make a **deterministic** ProposedAction producer: when a meeting/source extraction implies an external action, emit a real `ProposedAction` with a **concrete `targetSystem`** + the required external-write keys — populating the KMP's `externalActionProposals` (empty `actions:[]` today) — which lands as a **PENDING §9 Approval**. **NO real dispatch** (Phase 21/22 arms the Tool Gateway). **SAFE-BUILD: no external write, no dispatch, no hard line — a deterministic producer over the 18.3/18.4 extraction.**

## Acceptance criteria (what "done" means)
- [ ] A deterministic producer: a meeting/source extraction implying an external action emits a `ProposedAction` with a **concrete `targetSystem`** (not empty `actions:[]`) into the KMP's `externalActionProposals`.
- [ ] The ProposedAction carries the **required external-write keys** — `canonicalObjectKey` (existence key) + `idempotencyKey` (dedupe key) + `actionId` + `payload` + `approvalPolicy` — so it passes the candidate-data gate (`ProposedActionSchema` + `ruleExternalWriteKeys`).
- [ ] It lands as a **PENDING §9 Approval** (`approvalPolicy` = requires-approval) — **NO dispatch**, no Tool Gateway call, no external write.
- [ ] **No implied action ⇒ `actions:[]`** (empty) — a ProposedAction is emitted ONLY when the extraction implies a concrete action AND a `targetSystem` is resolvable.
- [ ] **WS-8 / no-inference:** `targetSystem` + the keys derive from the workspace binding/config + the action identity (traversal-safe, LESSON 5), NEVER smuggled from extraction/source CONTENT.
- [ ] **SAFE-BUILD:** deterministic over the 18.3/18.4 extraction; no dispatch/external write/hard line.
- [ ] All unit tests pass; `/preflight` clean.

## Things to flag at Step 2.5
1. **What implies an action** — a structured action-intent field in the extraction (NOT inferred from prose). My default vote: **an explicit action-intent field implies the ProposedAction; prose is never parsed into an action** (no-inference). Trace the extraction shape at Step 1.
2. **`targetSystem` source** — the workspace binding/config (the configured connector), NOT extraction content. My default vote: **binding/config-sourced `targetSystem`**; if no configured target, no action (empty) — never guess a target from content.
3. **Key derivation** — `canonicalObjectKey` + `idempotencyKey` from the action identity (workspace + extraction identity + action type), traversal-safe. My default vote: **derive deterministically from the identity, traversal-safe (LESSON 5)** — a hostile content string can't escape the key.
4. **No-dispatch boundary** — the ProposedAction lands PENDING; the producer does NOT call the Tool Gateway (Phase 21/22 arms dispatch). My default vote: **pin no-dispatch** (assert no Tool Gateway/external-write call site is added).
