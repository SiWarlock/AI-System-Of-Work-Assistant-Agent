# /tdd brief — worker_checkreachable_bind_seam

## Feature
Close the last unbound §ARM-18 arm seam: bind the real **spend-free** subscription reachability probe (`probeSubscriptionReachability({ detectLogin: detectClaudeKeychainLogin })`) into `bootWorker`'s `checkReachable` — but ONLY behind a **new, independent reachability-enable OFF-lock**, so an env-only subscription arm (`SOW_SUBSCRIPTION_ARM=1` alone) **stays `FAIL_CLOSED_REACHABILITY`-denied by design**. Today `boot.ts:1327` hard-defaults `checkReachable → FAIL_CLOSED_REACHABILITY` and the real probe is bound in NO production path, so the ENABLE currently needs a **code patch**, not a config flip. This slice makes the eventual arming a clean one-step crossing. Shipped default byte-equivalent (the probe thunk is never constructed). Spend-free (Keychain PRESENCE `-s`, no `-w`, + SDK module-resolvability, rule 7); no egress; no key provisioning. The FLIP stays owner+lead-gated — this slice arms NOTHING.

## Acceptance criteria (what "done" means)
- [ ] A **pure** worker helper resolves the effective `checkReachable` for the arm: it returns the REAL probe thunk (`() => probeSubscriptionReachability({ detectLogin: detectClaudeKeychainLogin })`) **iff** the subscription arm is enabled **AND** the reachability-enable signal is present; otherwise it returns `FAIL_CLOSED_REACHABILITY` (byte-equivalent to today).
- [ ] **⛔ Env-only arm STAYS denied (load-bearing, by design):** arm enabled + reachability-enable **absent** ⇒ the resolver yields `FAIL_CLOSED_REACHABILITY` (HEALTH UNAVAILABLE) — the shipped/default arm path is unchanged.
- [ ] **Truthy-not-`true` guard (L28):** the reachability-enable signal arms ONLY on the strict accepted value; `"false"`, `"0"`, `""`, an arbitrary truthy string, `{}` ⇒ NOT enabled ⇒ `FAIL_CLOSED_REACHABILITY`. (Match the accepted-value convention of the sibling `SOW_SUBSCRIPTION_ARM` read — strict `"1"|"true"`.)
- [ ] **AND-lock (L57):** the reachability-enable signal present but the arm DISABLED ⇒ still `FAIL_CLOSED_REACHABILITY` (a reachability signal can never arm a disabled gate; the OFF guard precedes the signal read).
- [ ] **Shipped default byte-equivalent:** with neither signal set, `bootWorker` constructs the arm seam EXACTLY as today (the real-probe thunk is never constructed — factory-spy zero-invocation on the OFF path, per L23/L27).
- [ ] When the resolver DOES bind the real probe, the wiring is correct end-to-end: the bound thunk, when invoked, calls `probeSubscriptionReachability` with the `detectClaudeKeychainLogin` detector (assert via injected spies — do NOT spawn real `security` / resolve real modules in the test).
- [ ] `boot.ts:1327`'s `checkReachable: …` argument is sourced from the new resolver (the `?? FAIL_CLOSED_REACHABILITY` default is preserved as the resolver's own fail-closed floor).
- [ ] All unit tests in the new test file pass; `/preflight` clean.
- [ ] No new egress / spend / key provisioning; the real probe stays reachability-WAIVERED (no production run until the owner flip) — the boot bind line itself covered by typecheck + the resolver unit tests (the `bootWorker` integration path is `SOW_API`-gated/skipped in preflight, L58).
