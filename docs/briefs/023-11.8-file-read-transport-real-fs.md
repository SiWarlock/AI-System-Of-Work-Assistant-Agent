## Acceptance criteria (what "done" means)
- [ ] `createFileReadTransport(root)` returns a `FileExtractTransport` that, given `{path}` under `root`, reads the file via `node:fs/promises.readFile(..., "utf8")` and resolves `{ok:true, file:{path, filename, mime, text}}` with the real file bytes as `text`.
- [ ] **ROOT-CONFINEMENT (the safety crux):** a `path` that resolves OUTSIDE `root` — `../` traversal, an absolute path outside `root`, or a symlink whose realpath escapes `root` — is REJECTED as `{ok:false, code:"unreachable"}` and NO bytes are read/returned. (Resolve to a real absolute path and assert containment under the resolved `root` BEFORE opening.)
- [ ] A missing file (ENOENT) or a non-file (directory) → `{ok:false, code:"unreachable", message}` (never throws).
- [ ] A binary / non-UTF8 file (NUL-byte sniff) → `{ok:false, code:"unknown", message}` — the transport does NOT emit garbage text as content (text/Markdown only; PDF/doc parsing stays deferred).
- [ ] An empty / whitespace-only file → the transport returns its (empty) text and `extractFileSource` fails it closed at the adapter as `empty_content` (existing behavior — assert it end-to-end, no contentless candidate registered).
- [ ] **End-to-end (`SOW_TEMPORAL`-gated):** a REAL temp file under a temp `root` → `createFileReadTransport` → `extractFileSource` → a valid `RegisterSourceInput` → fed into the live C1 `sourceIngestion` workflow → the real `registerSource()` gate returns `registered` → the workflow rests at `applied`. Proves a real local file flows end-to-end.
- [ ] `§16`: every transport fault is a typed `Result`/closed `FileExtractResult` — the transport NEVER throws across the seam (mirrors the adapter's Lesson-11 posture).
- [ ] Default suite unaffected: the transport unit tests run in the default suite (real temp files, no Temporal); the end-to-end case is `SOW_TEMPORAL`-gated. Repo-wide `turbo typecheck test` stays green.
- [ ] `/preflight` clean.

## Things to flag at Step 2.5
1. **ROOT-confinement mechanism.** My default vote: **resolve `path` to a real absolute path (`fs.realpath` on the parent / `path.resolve` + `realpath`), then assert it is contained under the resolved `root` (prefix check on the resolved real paths, not the raw strings)** — reject escapes as `unreachable`. Rationale: raw-string prefix checks are bypassable by `..`/symlinks; realpath containment is the robust form. This is the safety crux — do NOT ship a string-only check.
2. **Binary detection.** My default vote: **NUL-byte sniff on the read buffer → `unknown`** (read as buffer, check for `0x00`, else decode utf8). Simpler than a full charset detector; honest "text-only." Alternative: trust the caller `mime` hint — rejected (content-derived safety shouldn't depend on an optional caller hint).
3. **Symlink policy.** My default vote: **follow symlinks but require the REALPATH stay within `root`** (a symlink to an in-root file is fine; one escaping `root` is rejected). Alternative: reject all symlinks — stricter but breaks legit in-vault symlinks; defer that hardening.
4. **Where `extractFileSource` is driven in the live flow.** My default vote: **capture UPSTREAM of the workflow** (the transport+adapter produce `ctx.source`, the workflow's `registerSource` leg re-gates) — matches the connector/ingestion architecture; the test feeds the live workflow. Production auto-capture (file-watcher) is C3/later, NOT this slice.
5. **Sandbox safety.** Confirm `file-read-transport.ts` (uses `node:fs`) is imported ONLY activity/composition-side and never pulled into the Temporal workflow sandbox bundle (the barrels already neutralize `node:fs` via the bundler plugin — don't rely on that; keep the import off the sandbox path). Flag if the integrations barrel forces it in.
