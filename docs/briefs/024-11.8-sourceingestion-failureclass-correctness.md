# /tdd brief — sourceingestion_failureclass_correctness (make-it-real arc, C-fix)

## Feature
Fix the §16 failure-class mislabel the C1 live test surfaced: the `sourceIngestion` driver's `failureClassFor` maps every `failed_terminal` resting state to `worker_down`, but `failed_terminal` conflates several distinct causes — so a data-validation (`malformed_source`) reject is labeled as if the Temporal worker is down, violating the §16 "distinct health item per failure class" invariant. Make the terminal failure-class CAUSE-aware.

## Use case + traceability
- **Task ID:** 11.8 (make-it-real arc — the C-fix follow-up bullet; drains the C1 Finding)
- **Architecture sections it implements:** `ARCHITECTURE.md §16` (Observability / System Health — inv-5 "distinct §16 health item per failure class"; the FailureClass taxonomy).
- **Related context:** the mislabel is in `packages/workflows/src/workflows/sourceIngestion.ts` `failureClassFor` (L154) — a STATE→class map: `failed_terminal`→`worker_down`. But `failed_terminal` is reached by: (i) a register-`MALFORMED` schema reject (step 2, `captured`→`failed_terminal` — there is NO `captured→rejected` machine edge, so a schema reject at REGISTER can't rest at `rejected`); (ii) `agentFailureState`: `admission_rejected`/`injection_detected`/`unsupported_type`/`egress_vetoed`; (iii) `commitFailureState`: `ownership_violation`/`secret_found`/`commit_failed`. The frozen `FailureClass` enum (`packages/contracts/src/models/shared-enums.ts`) members: `connector_unreachable · write_through_failed · budget_breach · missed_or_late_schedule · schema_rejection · worker_down · parity_defect · conflict_review · sync_lagging · rebuild_divergence`. The C1 Finding (session-f2673cd5 round 7) flagged this; C1 test (b) currently PINS the wrong `worker_down` value.
- **Implements:** REQ (via §16 inv-5)

## Acceptance criteria (what "done" means)
- [ ] A register-`MALFORMED` source (schema reject at step 2) surfaces `failureClass === "schema_rejection"` (NOT `worker_down`) while still resting at the `failed_terminal` machine state (the machine is unchanged — no `captured→rejected` edge is added).
- [ ] A genuine infra/worker terminal failure (e.g. `commit_failed`) maps to a defensible class (`worker_down` reserved for actual worker/infra failure, or `write_through_failed` for a commit-write failure — decided at Step 2.5), NOT the same bucket as a schema reject.
- [ ] Each other `failed_terminal` cause (`injection_detected`/`admission_rejected`/`unsupported_type`/`egress_vetoed`/`ownership_violation`/`secret_found`) maps to the least-wrong EXISTING enum member, with an `arch_gap` note where the frozen enum lacks a dedicated member (consistent with the existing `policy_denial`/`egress_status` named-constant precedent) — NO frozen-contract enum change in this slice.
- [ ] The C1 live test (b) assertion is updated (`worker_down` → `schema_rejection`) and stays green.
- [ ] `failureClassFor`'s non-terminal mappings are unchanged (`queued_for_review`→`conflict_review`, `rejected`→`schema_rejection`, `failed_retryable`→`write_through_failed`).

## Things to flag at Step 2.5
1. **Cause-threading mechanism.** Default vote: **thread an explicit `FailureClass` (or a cause discriminant) from each `surface()` call site** (the site knows the cause), keeping `failureClassFor(state)` as the state-based DEFAULT fallback for non-terminal states — minimal, local, no new type ceremony. Alt: a `failureClassFor(state, cause)` overload.
2. **The classes for the causes lacking a dedicated enum member** (`injection_detected`/`egress_vetoed`/`ownership_violation`/`secret_found`). Default vote: **map candidate-gate-shaped causes (`admission_rejected`/`unsupported_type`) → `schema_rejection`; keep genuine worker/infra (`commit_failed`) → `worker_down` or `write_through_failed`; for injection/egress/ownership/secret use the least-wrong existing member + an `arch_gap` code-comment** flagging the frozen enum lacks a security/isolation member (the `policy_denial`/`egress_status` precedent). **ESCALATE (frozen-contract) ONLY IF** no existing member is defensible for a SECURITY failure (labeling an injection/secret breach as anything misleading is worse than an honest enum expansion) — Step-9 `Cross-doc invariant` / `Finding` if so.
3. **Machine unchanged?** Default vote: **yes — do NOT add a `captured→rejected` edge**; a register-malformed schema reject stays at `failed_terminal` (its only legal early-exit) with the corrected CLASS. Confirm the `sourceMachine` has no `captured→rejected` edge.
