# /tdd brief — reconciler_db_projection_builder

## Feature
Build the **`ReconcilerDbProjection` builder** (piece **B** of the reconcile-TRIGGER arc, task 13.10) — a worker-side composition helper that reads the injected **read-only `GbrainReadAdapter`** (the grant-verified, op-gated gbrain read surface), maps the gbrain read response → **`DbFact[]`**, determines **`complete`**, and returns the `ReconcilerDbProjection` that piece A's `runReconcilePass` consumes as `req.dbProjection`. **Fail-closed:** any read `err`, a truncation/paging signal, or a malformed row makes the projection `complete=false` (coverage-incomplete ⇒ the reconciler cannot claim full coverage ⇒ serving degrades), never a throw and never a false-complete. Ships **DORMANT + reachability-waivered**: no production caller (piece D wires it) and the **REAL live `GbrainReadGrant` HTTP transport stays OWNER-GATED/unbound** — the builder binds only an injected (fake-in-tests) adapter, so no real gbrain I/O runs the whole arc.

## Acceptance criteria (what "done" means)
- [ ] An exported builder (default name **`buildReconcilerDbProjection`**) that takes the injected `GbrainReadAdapter` + the `workspaceId` (+ any revision/paging inputs per Step-2.5 #1) and returns a `Promise<Result<ReconcilerDbProjection, …>>` **or** `Promise<ReconcilerDbProjection>` (error-channel shape per Step-2.5 #5) — every dep a fake in tests.
- [ ] **Happy path:** a clean, fully-consumed gbrain read maps each response row → a `DbFact` (`factIdentity`/`factKind`/`dbContentHash`/`stamped`/`revisionId`) and returns `{ workspaceId, gbrainSchemaVersion, facts, complete: true }`. Asserts: the facts round-trip the response rows faithfully; `complete=true` only here.
- [ ] **Fail-closed — read fault ⇒ incomplete:** a read `err` (`transport_fault` / `op_not_allowed`) makes the projection **`complete=false`** (never a throw, never a false-complete); the coverage leg degrades. Asserts: `complete=false` on any read `err`.
- [ ] **Fail-closed — truncation/paging ⇒ incomplete:** a response that signals truncation / a short page / an unconsumed cursor (per the expected shape, Step-2.5 #1) ⇒ **`complete=false`** even though the rows read are well-formed. Asserts: `complete=false` on a truncated read.
- [ ] **Fail-closed — malformed row ⇒ incomplete (or typed err, Step-2.5 #5):** a response row that does not parse into a `DbFact` ⇒ `complete=false` (degrade), collecting the parseable facts — never a throw, never a silently-dropped-but-complete claim. Asserts: `complete=false` on a malformed row.
- [ ] **`stamped` fail-closed default:** a row without an explicit signed-provenance signal maps to **`stamped=false`** (a false `stamped=true` would hide a `db_only`-unstamped HARD parity defect — the reconciler must SEE the unstamped fact to quarantine it). Asserts: absent stamp signal ⇒ `stamped=false`.
- [ ] **`gbrainSchemaVersion` sourced, not invented:** the schema version comes from the read (per Step-2.5 #1's op choice) — not a hardcoded constant; an absent/unreadable schema version ⇒ `complete=false` (can't corroborate the index version). Asserts: the version reflects the read.
- [ ] Ships **DORMANT + reachability-waivered** — no production caller (piece D), no boot, no Temporal; the real `GbrainReadGrant` HTTP transport stays unbound (the injected adapter is a fake). `/wired` reports test-only reachability; the waiver names piece D as the future caller.
- [ ] All unit tests pass; repo-wide `pnpm -w turbo run typecheck test` green; `/preflight` clean.
- [ ] **MANDATORY Step-8 dual review = security-reviewer + code-quality-reviewer** — this projection feeds the serve-time trust gate's coverage source; a false-complete projection or a mis-mapped `stamped` flag is a trust-gate defeat. Safety-critical; own commit.

## Things to flag at Step 2.5
1. **Which read op + expected response shape yields the fact set?** The `GbrainReadAdapter` payloads are typed `unknown`, so the builder DEFINES + parses the expected read-response contract (the shape the real transport must satisfy — mirror gbrain's actual read-op output, not a speculative shape). **My default vote:** source the semantic-fact set from **`graph`** (the typed semantic graph — `factIdentity`/`factKind`/edges) and the `gbrainSchemaVersion` from **`schemaRead`** (or `health`); define a minimal parsed contract `{ facts: [{ factIdentity, factKind, contentHash, stamped, revisionId }...], schemaVersion, truncated/cursor }`. Flag if a single op (or a different op) is the right minimal fact source — keep the parsed contract as small as the reconciler needs.
2. **`complete` determination — the fail-closed rule.** **Default vote:** `complete=true` **only** on a fully-consumed, clean, well-formed read; ANY of {read `err`, a truncation/short-page/open-cursor signal, a malformed row, an absent schema version} ⇒ `complete=false`. Never throw. Rationale: `complete` gates the reconciler's `coverageComplete` — a false-complete is a trust-gate defeat; incomplete fails SAFE (degrade).
3. **`stamped` mapping fidelity.** **Default vote:** map `stamped=true` ONLY when the row explicitly carries the signed-provenance signal; absent/ambiguous ⇒ `stamped=false` (fail-closed — the reconciler must see an unstamped `db_only` fact to quarantine it; a false `stamped=true` hides the hidden-brain defect, safety rule 1).
4. **File placement — worker-composition vs knowledge-package.** **Default vote: worker-composition** (`apps/worker/src/composition/reconcilerDbProjection.ts`) — consistent with piece A, single-track worker, `apps/worker → @sow/knowledge` allowed, and it keeps the arc cohesive in one track. (The alternative — `packages/knowledge/src/gbrain/parity/` next to the reconciler — is arguably "closer to gbrain's response shape" but is knowledge-track territory with no knowledge implementer this cycle.) Flag if you'd rather site it in knowledge.
5. **Malformed-row / partial-read error channel — degrade vs typed err.** **Default vote:** return a `ReconcilerDbProjection` with `complete=false` (degrade — collect the parseable facts) rather than a typed builder `err`, so the reconcile pass still runs against what WAS read and degrades honestly (matches the `complete` contract's intent). Flag if you'd rather return a typed err on any malformed row (piece A/D would then route it to health).
